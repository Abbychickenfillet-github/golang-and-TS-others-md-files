🩺 claude 指令在終端機跑不出來 — 排查筆記

Windows / PowerShell ｜ 建立日期 2026-06-21 ｜ 作者：Abby + Claude

前情提要

你執行 Get-Command claude 得到的是：

CommandType     Name          Source
-----------     ----          ------
ExternalScript  claude.ps1    C:\Users\...\claude.ps1

解讀：claude 有裝、而且在 PATH 裡。npm 在 Windows 全域安裝 CLI 時會放三個「殼(shim)」： claude（給 Git Bash）、claude.cmd（給 CMD）、 claude.ps1（給 PowerShell）。PowerShell 命中的就是那支 .ps1， 所以 ExternalScript 是正常的、不是壞掉。

👉 結論：你的問題不是「找不到」，而是「殼有了、但執行起來沒反應或報錯」。往下用決策樹定位。

① 互動決策樹：先跑這行,看症狀點按鈕

claude --version

你看到的是哪一種?(點選)

原因：PowerShell 執行原則(ExecutionPolicy)擋掉 .ps1 腳本。這是 Windows 最常見的兇手。

Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

跳出確認按 Y → Enter。只改「目前使用者」範圍最安全,不需要系統管理員。改完重開終端機再試 claude --version。

想看目前狀態：Get-ExecutionPolicy -List。RemoteSigned = 本機自己寫的腳本可跑、網路下載的要簽章。

原因：Node.js 不見了或不在 PATH(claude 是 Node 程式,殼會去呼叫 node)。

node -v
npm -v

若這兩行也報錯 → 重裝 Node(建議 LTS,或用 nvm-windows 管理),裝完重開終端機。Node 回來後 claude 通常就活了。

原因：殼還在,但它指向的安裝內容被刪/壞了(殼是空轉)。重裝即可。

npm install -g @anthropic-ai/claude-code

裝完確認落點有在 PATH：

npm config get prefix
# 印出的資料夾(例 C:\Users\你\AppData\Roaming\npm)要在系統 PATH 裡

版本印得出來 = 程式本體正常,卡住多半是「終端機相容性」或「登入/設定」問題。

試試這幾招(由易到難)：

# 2) 直接看它說什麼
claude
# 3) 設定/登入快取壞掉時,備份後清掉 .claude 再重登
#    設定通常在 C:\Users\你\.claude

⑤ 申論題（先想,再展開參考答案）

Q：為什麼同樣一支 claude,在 CMD 跑得動、在 PowerShell 卻說「停用指令碼執行」?請從「殼的種類」與「執行原則」兩個角度說明,並寫出你的解法。

▸ 顯示參考答案

npm 全域安裝時對同一個指令產生多種殼:CMD 走 claude.cmd(批次檔)、PowerShell 走 claude.ps1(PowerShell 腳本)、Git Bash 走無副檔名的 claude。

CMD 執行 .cmd 不受 ExecutionPolicy 管,所以能跑;但 PowerShell 對 .ps1 有「執行原則」把關,預設常是 Restricted/受限,於是擋下腳本,丟出「停用指令碼執行」。

解法:把目前使用者的執行原則放寬到 RemoteSigned—— Set-ExecutionPolicy -Scope CurrentUser RemoteSigned, 本機自己的腳本就能跑,網路下載的仍需簽章,安全與方便兼顧。改完重開終端機即可。
