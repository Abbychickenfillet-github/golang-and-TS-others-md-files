# C 槽空間清理 SOP（Windows）

> 觸發原因：C:\ 只剩 ~432MB，Adobe（Photoshop/Illustrator）跳出「暫存磁碟已滿」警告。
> 目標：先快速清出幾 GB 讓系統與 Adobe 能正常運作，再處理長期肥大來源。

## 🧠 先懂兩個名詞
- **暫存磁碟 (scratch disk)**：Adobe 軟體在 RAM 不夠時，拿硬碟當臨時工作區（類似虛擬記憶體），預設在 C:。C: 滿了它就無法運作。
- **暫存檔 (temp files)**：系統與各程式產生的臨時檔，通常可安全刪除，刪了程式會自己重建。

> 看到那個「暫存磁碟偏好設定」視窗：因為清單裡只有 C: 一顆且已滿，改設定沒用，先按 **取消**。真正要做的是把 C: 空間清出來。

---

## ✅ 第一波：最快見效、最安全（建議先做這些）

### 1. Windows 儲存空間 → 暫存檔（通常一次清幾 GB）
設定 → 系統 → 儲存空間 → **暫存檔案** → 勾選後「移除檔案」。可安全勾：
- Windows Update 清理 / 傳遞最佳化檔案
- 縮圖、暫存檔、資源回收筒
- 之前的 Windows 安裝（`Windows.old`，若有，常常好幾 GB）

⚠️ 不要勾「下載」除非你確定 Downloads 資料夾沒有要留的東西。

### 2. 磁碟清理工具（磁碟清理 / cleanmgr）
開「磁碟清理」→ 選 C: → 「清理系統檔案」→ 勾暫存、Update 快取等 → 確定。

### 3. 清空資源回收筒
桌面資源回收筒 → 右鍵 → 清空。

---

## ⚙️ 第二波：開發者常見的肥大來源（你最可能在這裡省最多）

| 來源 | 怎麼清 | 備註 |
|---|---|---|
| **Docker 映像/volume** | `docker system prune -a --volumes` | 常常數 GB~數十 GB；會刪掉沒在用的映像與 volume，要確認沒在用 |
| **node_modules** | 刪掉沒在開發的舊專案的 `node_modules` | 之後 `npm install` 可重建，安全 |
| **npm 快取** | `npm cache clean --force` | |
| **pip 快取** | `pip cache purge` | |
| **Adobe 媒體快取** | Photoshop/AE 偏好設定 → 媒體快取 → 清除 | Illustrator 較少，但 Creative Cloud 快取也可清 |
| **瀏覽器快取** | 瀏覽器設定清除快取 | |

> 想知道「到底是誰吃掉空間」：用免費工具 **WizTree** 或 **WinDirStat** 掃 C:，一眼看出最大的資料夾。

---

## 🛟 安全原則（別誤刪）
- 不要刪 `C:\Windows`、`Program Files`、不認識的系統資料夾。
- `node_modules`、各種 `cache`、`Temp`、`資源回收筒` → 安全。
- Docker prune 前先確認沒有正在用的容器/映像。
- 不確定的東西先別刪，先用 WizTree 看清楚再說。

## 🎯 這次的處理順序（建議）
1. Adobe 視窗按「取消」。
2. 設定 → 儲存空間 → 暫存檔案 → 移除（第一波）。
3. 清空資源回收筒。
4. 若還不夠 → Docker prune + 清舊 node_modules（第二波）。
5. 空間夠了，重開 Illustrator 就不會再跳警告。

## 📖 名詞解釋（設定裡會看到的東西）

- **新增儲存集區 / 儲存空間 (Storage Spaces)**：把「多顆實體硬碟」合併成虛擬大碟做備援（類似軟體 RAID）。**不能清空間**，單碟也用不到 → 忽略。
- **儲存空間感知器 (Storage Sense)**：自動定期清暫存/回收筒。**建議打開**，以後不會再爆。
- **磁碟清理建議**：Windows 自動列出可刪的大檔/暫存讓你勾，**實用，建議用**。
- **傳遞最佳化設定**：Windows Update 的 P2P 快取，可清。
- **管理磁碟及磁碟區**：看/管理分割區（進階，別亂動）。
- **Beyond Compare**：檔案/資料夾「比對與合併」diff 工具。已用 VS Code/Git diff 的話常多餘；但它很小，**解除安裝省不到空間**，不是清理重點。

## 🎬 各類別刪除注意
- **影片**：多半是 OBS/Movavi/威力導演 的螢幕錄影，肥且常一次性 → 好目標，但刪前確認沒有要留的作品。
- **已安裝的應用程式**：移除「大型且沒在用」的才有感（小工具省不到）。
- **Docker / node_modules**：開發機真正的大魔王，往往一清數十 GB。
