### 1. 解释 INNER JOIN 的作用

在 `025_improve_order_item_table.sql` 中添加了说明：

**为什么使用 INNER JOIN？**
- INNER JOIN 只更新在 `order` 表中存在的 `order_item` 记录
- 如果 `order_item` 的 `order_id` 指向不存在的 `order`，则不会更新（避免数据不一致）
- 这是数据完整性保护机制

**如果使用 LEFT JOIN：**
- 会更新所有 `order_item` 记录，即使对应的 `order` 不存在
- 可能导致 `event_id` 被设为 NULL 或错误的值

**如果使用 RIGHT JOIN：**
- 会更新所有 `order` 记录对应的 `order_item`（即使 `order_item` 不存在）
- 不符合需求

### 2. 创建订单费用明细表

创建了 `026_create_order_cost_details_tables.sql`，包含两个新表：

#### **order_booth_detail（订单摊位费用明细表）**

记录每个订单的摊位费用明细：
- `booth_id`：摊位 ID
- `booth_name`：摊位名称（快照，避免摊位名称变更影响历史记录）
- `booth_number`：摊位编号（如：A01, B02 等）
- `quantity`：摊位数量

- `total_price`：摊位总价（quantity * unit_price）
- `currency`：币别

**支持场景：**
- 一个订单可以包含多个摊位
- 每个摊位可以有不同的编号和价格
- 记录快照，避免历史数据因价格变更而改变

#### **order_electricity_detail（订单电力费用明细表）**

记录每个订单的电力费用计算明细：
- `total_wattage`：总需求瓦数
- `base_wattage`：基本瓦数（从规则中取得）
- `excess_wattage`：超出瓦数（total_wattage - base_wattage）
- `wattage_unit`：计价比数单位（如：每500W为一单位）
- `units`：计算单位数（向上取整：excess_wattage / wattage_unit）
- `price_per_unit`：每单位价格（快照，如：750 NTD）
- `wattage_cost`：瓦数费用（units * price_per_unit）
- `cable_extension_fee`：拉线基本费
- `cable_length_meters`：延长线长度（公尺）
- `cable_fee_per_meter`：每公尺拉线费（快照）
- `cable_length_cost`：延长线费用
- `total_electricity_cost`：总电力费用

**计算示例：**
```
总需求：1500W
基本瓦数：500W
超出瓦数：1000W
单位：500W
单位数：1000 / 500 = 2
每单位价格：750 NTD
瓦数费用：2 * 750 = 1500 NTD
拉线基本费：750 NTD
延长线：5 公尺 * 50 NTD/公尺 = 250 NTD
总电力费用：1500 + 750 + 250 = 2500 NTD
```

## 完成的保护机制

### 1. 数据库层面保护

创建了 SQL 迁移脚本 `027_protect_order_items_from_product_deletion.sql`：

- 修改外键约束：从 `ON DELETE CASCADE` 改为 `ON DELETE SET NULL`
  - 商品被删除时，`order_item.product_id` 会被设为 `NULL`
  - 订单项目不会被删除，其他信息（价格、数量等）保留
- 添加商品快照字段：`product_name`
  - 保存下单时的商品名称
  - 即使商品被删除，订单仍可显示商品名称

### 2. 后端模型更新

- 更新 `OrderItem` 模型：
  - `product_id` 改为可空（`str | None`）
  - 添加 `product_name` 字段（快照）
- 更新 `OrderItemService.create_item`：
  - 创建订单项目时自动从 `product` 表获取并保存商品名称
  - 确保订单记录包含商品快照

### 3. 前端显示优化

- 更新订单详情页面：
  - 优先显示 `product_name`（快照）
  - 如果 `product_name` 为空，显示 `product_id` 或“商品已删除”

### 4. 保障机制说明

当商品被删除时：
- `order_item.product_id` → `NULL`（外键 `ON DELETE SET NULL`）
- `order_item.product_name` → 保留（快照）
- `order_item.price` → 保留
- `order_item.amount` → 保留
- 订单记录完整保留

当商品被下架（软删除）时：
- 如果使用 `deleted_at` 软删除，商品记录仍然存在
- `order_item.product_id` 仍然有效
- 可以正常关联查询
