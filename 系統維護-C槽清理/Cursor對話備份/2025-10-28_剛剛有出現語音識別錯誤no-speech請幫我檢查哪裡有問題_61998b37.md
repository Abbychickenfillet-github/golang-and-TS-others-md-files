functional updater。是什麼

## Cursor
### 簡單說
- 在 React 中，`useState` 的 setter（例如 `setX`）如果接收到「函式」，React 會把它當成「functional updater」：也就是用這個函式去計算新狀態，並把「前一個狀態」當參數傳進去。

### 兩種用法
- 直接指定值：`setCount(3)` → 新狀態就是 3
- functional updater：`setCount(prev => prev + 1)` → 用前一個狀態算出新值

### 為什麼我們要包一層
- 我們其實想把「一個可呼叫的函式」存進 state，而不是要 React「用它計算新狀態」。
- 如果直接 `setVoiceToggleFn(toggleFn)`，React 看到你傳的是函式，會把它當成 functional updater 來執行，導致型別/行為錯誤。
- 正確做法：用「函式回傳值」的形式，讓 React 知道你要存的是「這個函式本身」。
```ts
// 錯（會被當作 functional updater）
setVoiceToggleFn(toggleFn)

// 對（把函式本身存進 state）
setVoiceToggleFn(() => toggleFn)

// 清空時也要對應處理
setVoiceToggleFn(null)
```

### 一句話記
- 只要你要「把函式當作值存進 state」，就用 `setState(() => thatFunction)`，避免被 React 誤當成 functional updater。

- 1) 直接操作 DOM：

- 3) 效能瓶頸：

## 修復說明

### 問題根源

   - `data-input-type="title"` 或 `data-input-type="desc"`（標識輸入框類型）
   - `data-input-id="lapTitleInput"` 或 `data-input-id="lapDescInput"`（輸入框的 id）

   ```tsx
   let currentInputType: string | null = null  // 閉包變數
   
   // 點擊時保存 inputType
   currentInputType = inputType
   
   // 識別結果出來時，帶著 inputType 一起回傳
   onResult(transcript, currentInputType || undefined)
   ```

   - 透過 `inputId` 找到輸入框並聚焦

   - 優先使用傳入的 `inputType`
   - 如果沒有則使用 `focusedInputRef.current` 作為備用

5. 更新對應輸入框 → `handleVoiceResult` 根據 `inputType` 更新 `title` 或 `desc`
