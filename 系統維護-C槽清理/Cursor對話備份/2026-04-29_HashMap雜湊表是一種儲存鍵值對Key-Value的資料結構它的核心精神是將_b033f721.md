HashMap（雜湊表）是一種儲存鍵值對（Key-Value）的資料結構。它的核心精神是將鍵（Key）透過雜湊函數（Hash Function）轉換成整數，對應到「陣列（Array）」中的特定索引位置，藉此達成 \(O(1)\) 常數時間的超高速查詢效能。內層結構解析（以 Java 為例）HashMap 的基礎結構是一個結合陣列與串列/樹狀結構的複合體：主體（陣列 / Bucket）


：結構的底層是一個陣列。陣列中的每一個格子稱為一個「桶（Bucket）」，用來存放資料。節點（Node / Entry）：陣列裡儲存的不是單純的 Value，而是包含 Key、Value、HashCode 及指向下一個節點的 指標 的節點物件。運作原理計算位置：當你存入一個鍵值對時，系統會調用 Key 的 hashCode() 方法，並進一步計算出雜湊值，最後對陣列長度進行取模運算（或位元運算），決定該筆資料要放入陣列的哪一個索引中。雜湊碰撞（Collision）：不同的 Key 有可能計算出相同的陣列索引，這稱為「碰撞」。鏈結串列（Linked List）：若發生碰撞，新舊節點會以「鏈結串列」的形式串接在同一個桶（Bucket）底下。紅黑樹（Red-Black Tree）：為了避免單一個桶內的串列過長導致查詢退化成 \(O(n)\)，當串列長度超過特定限制（如 Java 預設的 8 個節點）時，該串列會自動轉換為「紅黑樹」，大幅優化搜尋效率。核心特性無序性：資料儲存的位置取決於雜湊值，因此輸出的順序與插入順序無關。鍵唯一性：Key 是唯一的（不可重複），如果存入已存在的 Key，舊的 Value 會被覆蓋。效率極高：在理想狀態下，新增、刪除、查詢的時間複雜度皆為 \(O(1)\)
