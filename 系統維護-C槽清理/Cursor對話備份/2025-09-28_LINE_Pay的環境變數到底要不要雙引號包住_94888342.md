## LINE Pay 環境變數是否需要雙引號

**結論：不需要雙引號。**

### 1. 專案中的實際使用情況

在 `config/server.config.js` 中，環境變數是這樣使用的：

```javascript
linePay: {
  development: {
    channelId: process.env.LINE_PAY_CHANNEL_ID || '12008177891',
    channelSecret: process.env.LINE_PAY_CHANNEL_SECRET || '9a2f9cbdf8058762307491a2da9ab15c',
  },
  production: {
    channelId: process.env.LINE_PAY_CHANNEL_ID || '12008177891',
    channelSecret: process.env.LINE_PAY_CHANNEL_SECRET || '9a2f9cbdf8058762307491a2da9ab15c',
  },
}
```

### 2. 環境變數檔案中的設定

- `.env.development`：使用雙引號
- `.env.production`：未使用雙引號

### 3. 建議

**不要使用雙引號**，原因：
1. `process.env` 會自動去除引號
2. 專案中 production 環境未使用引號
3. 避免多餘引號造成混淆

### 4. 正確的設定方式

```bash
# 正確 - 不使用引號
LINE_PAY_CHANNEL_ID=12008177891
LINE_PAY_CHANNEL_SECRET=T9a2f9cbdf8058762307491a2da9ab15c

# 錯誤 - 不需要引號
LINE_PAY_CHANNEL_ID="12008177891"
LINE_PAY_CHANNEL_SECRET="T9a2f9cbdf8058762307491a2da9ab15c"
```

### 5. 何時需要引號

僅在值包含空格或特殊字元時才需要引號：

```bash
# 需要引號的情況
DATABASE_URL="postgresql://user:pass@host:port/db"
```

LINE Pay 的 Channel ID 和 Secret 通常不含空格或特殊字元，因此不需要引號。

## 正確答案：`lineUid: lineUid`

**原因：**

### 1. Prisma Schema 定義
```prisma
model User {
  lineUid         String?    @unique @map("line_uid")
  // ...
}
```

### 2. Prisma 查詢規則
- 在 Prisma 查詢中使用 **Prisma 模型欄位名稱**（`lineUid`）
- 不是資料庫欄位名稱（`line_uid`）
- `@map("line_uid")` 只影響資料庫層面

### 3. 程式碼一致性
```javascript
// 第 45 行：查詢使用者
where: {
  lineUid: sub,  // ✅ 使用 Prisma 欄位名稱
}

// 第 55 行：更新使用者
where: {
  lineUid: sub,  // ✅ 使用 Prisma 欄位名稱
}

// 第 80 行：建立使用者
lineUid: String(sub),  // ✅ 使用 Prisma 欄位名稱
```

### 4. 修正後的正確寫法
```javascript
// ✅ 正確
where: {
  lineUid: lineUid,  // Prisma 欄位名稱
}

// ❌ 錯誤
where: {
  line_uid: lineUid,  // 資料庫欄位名稱
}
```

**總結：**
- 使用 `lineUid: lineUid`（Prisma 欄位名稱）
- 不要使用 `line_uid: lineUid`（資料庫欄位名稱）
- 與現有程式碼一致
