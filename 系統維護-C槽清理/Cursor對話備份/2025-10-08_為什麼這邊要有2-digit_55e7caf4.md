## 為什麼需要 `2-digit`？

`2-digit` 是 JavaScript 的 `Intl.DateTimeFormat` 選項，用於確保數字始終以兩位數格式顯示。具體原因如下：

### 1. **一致性顯示**
- **沒有 `2-digit`**：月份可能顯示為 `1`, `2`, `3`... `12`
- **有 `2-digit`**：月份會顯示為 `01`, `02`, `03`... `12`

### 2. **視覺對齊**
```javascript
// 沒有 2-digit 的結果：
2024/1/5 14:3:25    // 不整齊
2024/12/25 14:30:25 // 長度不一致

// 有 2-digit 的結果：
2024/01/05 14:03:25  // 整齊對齊
2024/12/25 14:30:25  // 長度一致
```

### 3. **用戶體驗**
- 在訂單摘要中，時間格式的一致性很重要
- 用戶可以更容易地掃視和比較時間
- 符合台灣常見的日期時間顯示習慣

### 4. **實際效果對比**
```javascript
// 當前時間假設為 2024年1月5日 14:3:25

// 不使用 2-digit：
currentTime.toLocaleString('zh-TW', {
  year: 'numeric',
  month: 'numeric',    // 顯示 "1"
  day: 'numeric',      // 顯示 "5" 
  hour: 'numeric',     // 顯示 "14"
  minute: 'numeric'    // 顯示 "3"
})
// 結果：2024/1/5 14:3

// 使用 2-digit：
currentTime.toLocaleString('zh-TW', {
  year: 'numeric',
  month: '2-digit',    // 顯示 "01"
  day: '2-digit',      // 顯示 "05"
  hour: '2-digit',     // 顯示 "14" 
  minute: '2-digit'    // 顯示 "03"
})
// 結果：2024/01/05 14:03
```

在您的訂單摘要組件中，使用 `2-digit` 確保了時間顯示的專業性和一致性，這對於支付相關的介面來說是很重要的細節。

## 為什麼可以這樣用？

### 1. **React State Setter 的特性**
`setCurrentTime` 是 React 的 state setter 函數，它有以下特性：
- **函數式更新**：可以接受一個函數作為參數
- **自動重新渲染**：當 state 改變時，組件會自動重新渲染
- **異步執行**：React 會批量處理 state 更新

### 2. **setInterval 的工作原理**
```javascript
const timer = setInterval(() => {
  setCurrentTime(new Date())  // 每 60000ms (1分鐘) 執行一次
}, 60000)
```

- `setInterval` 會定期執行回調函數
- 每次執行時，都會創建一個新的 `Date` 物件
- 調用 `setCurrentTime` 更新 state

### 3. **實際執行流程**
```javascript
// 初始狀態
const [currentTime, setCurrentTime] = useState(new Date()) // 第一次渲染

// 1分鐘後
setInterval(() => {
  setCurrentTime(new Date()) // 創建新的 Date 物件，觸發重新渲染
}, 60000)

// 2分鐘後
setInterval(() => {
  setCurrentTime(new Date()) // 再次創建新的 Date 物件，觸發重新渲染
}, 60000)

// ... 持續下去
```

### 4. **為什麼這樣設計？**

**問題**：如果不用 `setInterval` 更新時間會怎樣？
```javascript
// ❌ 錯誤做法 - 時間不會更新
const [currentTime, setCurrentTime] = useState(new Date())

// 時間永遠是組件初始化時的時間
return <div>{currentTime.toLocaleString()}</div>
```

**解決方案**：使用 `setInterval` 定期更新
```javascript
// ✅ 正確做法 - 時間會每分鐘更新
useEffect(() => {
  const timer = setInterval(() => {
    setCurrentTime(new Date()) // 每次都創建新的時間
  }, 60000)

  return () => clearInterval(timer) // 清理定時器
}, [])
```

### 5. **記憶體管理**
注意代碼中的清理機制：
```javascript
useEffect(() => {
  const timer = setInterval(() => {
    setCurrentTime(new Date())
  }, 60000)

  return () => clearInterval(timer) // 組件卸載時清理定時器
}, [])
```

這很重要，因為：
- 防止記憶體洩漏
- 避免組件卸載後定時器仍在運行
- 確保只有一個定時器在運行

### 6. **實際效果**
- 頁面載入時顯示當前時間
- 每分鐘自動更新一次時間顯示
- 用戶可以看到實時的訂單建立時間
- 組件卸載時自動清理定時器

這種模式在 React 中很常見，特別是用於顯示實時數據（如時間、計數器、狀態更新等）的場景。

function signLinePay(path, body = '') {
  const nonce = nanoid();
  const raw = `${LINE_PAY_CHANNEL_SECRET}${path}${body}${nonce}`;
  const signature = crypto.createHmac('sha256', LINE_PAY_CHANNEL_SECRET).update(raw).digest('base64');

async function linePayRequestApi(requestBody) {
  const path = '/v3/payments/request';
  const body = JSON.stringify(requestBody);
  const { signature, nonce } = signLinePay(path, body);

## 🔍 **Axios 錯誤結構說明**

### 📋 **Axios 錯誤物件的標準結構**

當 Axios 請求失敗時，錯誤物件的結構是：

```javascript
const error = {
  message: "Request failed with status code 404",
  name: "AxiosError",
  code: "ERR_BAD_REQUEST",
  config: { /* 請求配置 */ },
  request: { /* 請求物件 */ },
  response: {  // ← 這是 HTTP 回應
    data: {    // ← 這是後端回傳的資料
      status: "error",
      message: "缺少必要參數",
      payload: { /* 實際資料 */ }
    },
    status: 404,
    statusText: "Not Found",
    headers: { /* HTTP 標頭 */ }
  }
}
```

### 🎯 **為什麼用 `err.response`？**

1. **`err.response`** - 這是 Axios 的標準錯誤結構
   - 包含完整的 HTTP 回應資訊
   - 包含 `status`、`statusText`、`headers` 等

2. **`err.payload`** - 這不存在！
   - Axios 錯誤物件沒有 `payload` 屬性
   - 這會導致 `undefined` 錯誤

### 📊 **資料層級關係**

```
err (Axios 錯誤物件)
├── message: "Request failed..."
├── response (HTTP 回應)
│   ├── data (後端回傳的 JSON)
│   │   ├── status: "error"
│   │   ├── message: "錯誤訊息"
│   │   └── payload: { /* 實際資料 */ }  ← 這裡才是 payload
│   ├── status: 404
│   └── statusText: "Not Found"
└── config: { /* 請求配置 */ }
```

### 🔧 **正確的錯誤處理方式**

```javascript
try {
  const response = await axios.post('/api/payment', data)
  // 成功時：response.data.payload
  console.log(response.data.payload)
} catch (err) {
  // 錯誤時：err.response.data
  console.log(err.response?.data?.message)  // 後端的錯誤訊息
  console.log(err.response?.status)         // HTTP 狀態碼
  console.log(err.message)                  // Axios 的錯誤訊息
}
```

  ```javascript
  Math.random().toString(36)
  ```

  - `2`：起始位置（跳過 "0."）

  ```javascript
  '0.abc123def456'.substr(2, 9)
  ```

## 📋 **CSS 優先級說明**

### **Bootstrap 類別優先級：**
```css
/* Bootstrap 的 CSS 順序 */
.text-white { color: white !important; }
.text-success { color: #198754 !important; }
```

### **為什麼 text-success 沒有生效：**

1. **同時存在時**：`text-white` 會覆蓋 `text-success`
2. **解決方法**：
   - 移除 `text-white`
   - 或使用條件式類別
   - 或使用 `!important` 強制覆蓋

## 🎨 **現在的邏輯**

```javascript
// 選中時：顯示綠色 ✓
```
