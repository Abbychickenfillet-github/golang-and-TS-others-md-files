## 100vh 的機制

**`100vh` 代表什麼？**
- `vh` = viewport height（視窗高度）
- `100vh` = 100% 的視窗高度
- 這是 CSS 的相對單位，會根據**瀏覽器視窗**的高度來計算

## 不同螢幕尺寸的影響

**是的，`100vh` 確實會因為不同螢幕尺寸而改變！**

### 1. **螢幕尺寸影響**
- **14吋筆電**：通常解析度 1366×768 或 1920×1080
- **16吋筆電**：通常解析度 1920×1080 或更高
- **27吋螢幕**：通常解析度 2560×1440 或 3840×2160

### 2. **實際計算範例**
```css
/* 14吋筆電 (1366×768) */
100vh = 768px

/* 16吋筆電 (1920×1080) */
100vh = 1080px

/* 27吋螢幕 (2560×1440) */
100vh = 1440px
```

## 你的專案中的使用情況

從程式碼中可以看到，你的專案大量使用了 `100vh`：

```scss
// styles/globals.scss
main {
  min-height: 100vh; // 使用完整視窗高度
}

.gradient-bg {
  min-height: 100vh; // 確保至少覆蓋整個視窗高度
}
```

## 潛在問題與解決方案

### 1. **行動裝置問題**
- 行動瀏覽器的地址欄會影響 `100vh` 的計算
- 當用戶滾動時，地址欄隱藏/顯示會改變視窗高度

### 2. **更好的替代方案**
```css
/* 使用 min-height 而不是 height */
min-height: 100vh;

/* 或者使用 CSS 自定義屬性 */
height: 100dvh; /* 動態視窗高度 (較新的瀏覽器支援) */
```

### 3. **響應式設計建議**
```scss
// 針對不同螢幕尺寸的調整
@media (max-width: 768px) {
  .container {
    min-height: 100vh;
    min-height: 100dvh; // 支援動態視窗高度的瀏覽器
  }
}

@media (min-width: 1200px) {
  .container {
    min-height: 100vh;
    max-height: 1200px; // 限制最大高度
  }
}
```

## 總結

1. **`100vh` 會根據螢幕尺寸改變** - 這是正常的行為
2. **14吋和16吋電腦會有不同的高度** - 因為解析度不同
3. **你的專案使用 `min-height: 100vh`** - 這是好的做法，確保內容至少填滿視窗
4. **建議考慮響應式設計** - 針對不同螢幕尺寸做優化

這種設計讓你的網站能夠適應各種螢幕尺寸，提供一致的使用體驗。
