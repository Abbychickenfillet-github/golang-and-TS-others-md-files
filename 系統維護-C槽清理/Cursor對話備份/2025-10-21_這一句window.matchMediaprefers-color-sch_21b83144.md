## 關於 `window.matchMedia('(prefers-color-scheme: dark)')` 的解釋：

### 1. **是否必要？**
在這個範例中，**不是絕對必要的**，因為：
- CSS 的 `@media (prefers-color-scheme)` 已經可以自動處理主題切換
- 頁面的樣式會根據系統主題自動變化

### 2. **為什麼還是要用？**
JavaScript 版本的主要目的是：
- **顯示狀態**：在頁面上顯示當前系統主題狀態（讓用戶知道現在是什麼主題）
- **即時監聽**：當用戶切換系統主題時，頁面能立即反映變化
- **擴展性**：為後續可能的手動主題控制功能做準備

### 3. **`mediaQuery.matches` 是什麼？**
- `matches` 是一個 **boolean 值**
- `true`：表示當前系統設定為深色主題
- `false`：表示當前系統設定為淺色主題
- 這個值會根據用戶的系統設定自動變化

### 4. **`window.matchMedia()` 的作用**
- 這是瀏覽器提供的 API，用來檢測 CSS media query 是否匹配
- 返回一個 `MediaQueryList` 對象
- 包含 `matches` 屬性（boolean）和 `change` 事件

簡單來說，如果你只需要純 CSS 的主題切換，可以不用 JavaScript。但如果你想要顯示主題狀態或提供更豐富的交互體驗，就需要使用 `window.matchMedia` 來監聽和響應主題變化。

## 為什麼 `currentTime` 的初始值要設為空字符串？

### 1. **Hydration 錯誤的根本原因**
```javascript
// ❌ 錯誤的做法
const [currentTime, setCurrentTime] = useState(new Date())
```

**問題**：
- **服務端渲染**：`new Date()` 在服務端執行時會產生一個時間（例如：14:30:25）
- **客戶端渲染**：當 JavaScript 在瀏覽器中執行時，`new Date()` 會產生另一個時間（例如：14:30:26）
- **時間差**：服務端和客戶端的時間不同，導致 React 無法正確匹配 DOM 結構

### 2. **正確的解決方案**
```javascript
// ✅ 正確的做法
const [currentTime, setCurrentTime] = useState('')
const [isClient, setIsClient] = useState(false)
```

**為什麼這樣做**：
- **服務端**：`currentTime` 是空字符串 `''`
- **客戶端**：`currentTime` 也是空字符串 `''`
- **一致性**：服務端和客戶端渲染的內容完全一致

### 3. **完整的修復流程**
```javascript
// 1. 初始狀態：服務端和客戶端都是空字符串
const [currentTime, setCurrentTime] = useState('')
const [isClient, setIsClient] = useState(false)

// 2. 在 useEffect 中設置實際時間（只在客戶端執行）
useEffect(() => {
  setIsClient(true)           // 標記為客戶端
  setCurrentTime(new Date())  // 設置實際時間
  
  const timer = setInterval(() => {
    setCurrentTime(new Date())
  }, 1000)
  
  return () => clearInterval(timer)
}, [])

// 3. 條件渲染：避免 Hydration 錯誤
{isClient ? currentTime.toLocaleTimeString('zh-TW') : '載入中...'}
```

### 4. **為什麼需要 `isClient` 狀態？**
- **服務端**：`isClient` 是 `false`，顯示 "載入中..."
- **客戶端**：`isClient` 是 `true`，顯示實際時間
- **避免閃爍**：確保用戶看到的是預期的內容

### 5. **其他可能的解決方案**
```javascript
// 方案 1：使用 null 作為初始值
const [currentTime, setCurrentTime] = useState(null)

// 方案 2：使用 undefined
const [currentTime, setCurrentTime] = useState(undefined)

// 方案 3：使用固定的初始時間
const [currentTime, setCurrentTime] = useState('00:00:00')
```

**總結**：空字符串是最安全的選擇，因為它確保服務端和客戶端渲染的內容完全一致，避免 Hydration 錯誤。
