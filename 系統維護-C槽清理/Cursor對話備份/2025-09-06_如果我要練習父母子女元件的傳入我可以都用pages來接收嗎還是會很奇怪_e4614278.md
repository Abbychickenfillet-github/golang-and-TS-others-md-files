- **paths 是做什麼**: 在 `jsconfig.json/tsconfig.json` 中建立「路徑別名」。你這行
  ```json

  ```
  代表把 `@/` 對應到專案根目錄（`my-app/`）。所以
  - `import Btn from '@/components/Button'` → 解析為 `my-app/components/Button`.

- **baseUrl 是做什麼**: 設定「非相對路徑」的基準起點。常見設定：
  ```json
  "baseUrl": "."
  ```
  代表把目前設定檔所在資料夾（`my-app/`）當成起點。這樣可以寫：
  - `import Btn from 'components/Button'`（不加 `@` 也能指到 `my-app/components/Button`）
  
- **paths vs baseUrl 的差異與關係**
  - **baseUrl**: 啟用從某個根開始的「裸匯入」路徑，例如 `components/...`。
  - **paths**: 自訂前綴別名（如 `@/`、`~` 等），可指向一個或多個實際路徑。

  - 使用 `@/` 別名可讀性最好，搬移檔案不容易壞掉。
  - 設定或修改 `jsconfig.json` 後，如解析異常，重啟開發伺服器一次。

- **小範例**

- 父元件「完全沒傳」某個 prop

## Cursor
- 核心規則：在 JSX，屬性值若不用花括號，會被當成「字串常值」。要放「任意 JS 表達式」就必須用花括號。
  - 字串常值可直接寫：title="Hello"
  - 任何 JS 表達式（物件、陣列、布林、數字、函式、變數、運算）都要：prop={...}

- 為什麼物件/布林/陣列/函式要花括號
  - 它們是 JS 值或表達式，不是字串字面量，所以要用 `{}` 告訴 JSX「把這個當 JavaScript 評估」：
    - objProp={{ a: 1 }}
    - listProp={[1, 2, 3]}
    - flagProp={true}（或簡寫 `<Comp flagProp />`）
    - onClickProp={() => {...}}
    - countProp={42}

- 字串為什麼不用
  - HTML 習慣直接寫字串：label="OK"
  - 但你也可以寫成字串表達式：label={'OK'}（兩者等價）
  - 若是變數或運算，還是要用花括號：label={name}、label={`Hi ${user}`}

- 布林特殊規則
  - `<Comp disabled />` 等同 `disabled={true}`
  - 要傳 false 必須寫 `disabled={false}`（不能寫 disabled）

- 小提醒
  - 不加引號也不加花括號像 `title=abc` 會被當成識別字而報錯；要用 `title="abc"` 或 `title={'abc'}` 或 `title={abc}`（變數）。

## Cursor
### 範例：資料容器型 ref 追蹤上一個使用者 id（不綁 DOM）

```jsx
import { useEffect, useRef, useState } from 'react';

function Child({ onSwitch }) {
  return (
    <button onClick={() => onSwitch()}>
      切換使用者
    </button>
  );
}

export default function UserSwitcher() {
  const [userId, setUserId] = useState(1);

  // 資料容器型 ref：用來保存「上一個 userId」
  const prevUserIdRef = useRef(userId);

  // 每次 userId 改變後，比對上一個值，再把 ref 更新為目前值
  useEffect(() => {
    if (prevUserIdRef.current !== userId) {
      console.log('帳號有變更：', prevUserIdRef.current, '→', userId);
      // 在這裡可以做：清除快取、重新拉資料、或顯示提示等
      prevUserIdRef.current = userId; // 更新 ref，供下一次比較
    }
  }, [userId]);

  // 傳遞函式給子元件（和你 parent.js 的 onClick props 類似）
  const handleSwitch = () => {
    setUserId((id) => (id === 1 ? 2 : 1));
  };

  return (
    <>
      <h3>目前使用者：{userId}</h3>
      <Child onSwitch={handleSwitch} />
    </>
  );
}
```

- 重點
  - `prevUserIdRef` 不綁任何 JSX；只用 `prevUserIdRef.current` 存「上一個值」。
  - `useEffect` 裡對比 `prevUserIdRef.current` 和現在的 `userId`，偵測變更；最後把 ref 更新。
  - 這種 ref 改變不會觸發 re-render，非常適合「跨 render 維持可變資料」。

- 練習建議
  - 在切換時若偵測到變更，觸發重新 fetch 該使用者資料（模擬 API）。
  - 新增一個 DOM ref（例如輸入框聚焦）對比兩者用途：DOM ref 綁 `ref={inputRef}`，資料 ref 完全不綁 JSX。

這邊的.current是什麼

## Cursor
- `.current` 是 `useRef` 回傳物件上的唯一欄位。`useRef(initial)` 會回傳一個穩定物件 `{ current: initial }`。
- 這個物件會在每次 render 間保留，同一個引用；你用 `ref.current` 存/取值。
- 更新 `ref.current = newValue` 不會觸發 re-render；適合存「上一個值、計時器 id、任意暫存資料」。
- DOM ref 時，React 會把對應的 DOM 節點放進 `ref.current`，例如 `inputRef.current.focus()`。
- 資料容器型 ref 時，你自己賦值與讀取，例如：
  ```js
  const prevUserIdRef = useRef(userId);   // 初值
  // ...
  if (prevUserIdRef.current !== userId) {
    prevUserIdRef.current = userId;       // 更新但不重渲染
  }
  ```
- 小提醒：不要解構 `const { current } = useRef(...)`，要保留 `ref.current` 的存取方式，確保引用穩定。

為什麼這邊的props還有冒號

## Cursor
因為這是 TypeScript 的型別註記。冒號後面的是參數的型別。

- 解構版
  - `({ onResult }: { onResult: (_text: string) => void })`
  - 左邊是解構的參數，右邊用型別文字描述此參數的結構：必須有 `onResult`，且它是一個函式，參數是 `string`，回傳 `void`。

- 不解構版
  - `(props: { onResult: (_text: string) => void })`
  - `props` 這個參數的型別是物件，含有 `onResult` 這樣的函式屬性。

等價寫法還有：
- 先定義型別
  ```ts
  type VoiceInputProps = { onResult: (text: string) => void };
  function VoiceInput({ onResult }: VoiceInputProps) { ... }
  ```
- 或用 interface
  ```ts
  interface VoiceInputProps { onResult: (text: string) => void }
  ```
冒號只是在宣告「這個參數/屬性是什麼型別」，與執行時行為無關。

## 你
但是這邊我們並沒有連接資料庫所以也沒有所謂的userId

## Cursor
沒問題，範例的 userId 只是「模擬用的狀態值」，不需要資料庫。

- 你可以把它當成任何可變狀態來練習資料容器型 ref：
  - 計數器 counter
  - 目前頁碼 page
  - 目前選取的項目 selectedId（從假資料陣列取）
  - 隨機生成的 sessionId（`crypto.randomUUID()`）

小改版示例（不依賴資料庫）：
```jsx
const [counter, setCounter] = useState(0);
const prevCounterRef = useRef(counter);

useEffect(() => {
  if (prevCounterRef.current !== counter) {
    console.log('counter 變更：', prevCounterRef.current, '→', counter);
    prevCounterRef.current = counter;
  }
}, [counter]);

<button onClick={() => setCounter(c => c + 1)}>+1</button>
```
重點是用 ref 保存「上一個值」，和現在值比較；值本身可來自任何本地 state。
