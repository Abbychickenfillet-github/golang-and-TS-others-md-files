## 1. useState vs Zustand 的寫法對比

**useState 寫法：**
```javascript
const [cart, setCart] = useState([]);
```

**Zustand 寫法：**
```javascript
// 在 store 中定義
const useCartStore = create((set) => ({
  cart: [],
  setCart: (newCart) => set({ cart: newCart }),
}));

// 在組件中使用
const { cart, setCart } = useCartStore();
```

這裡 `cart` 和 `setCart` 是 props，不是直接使用 useState 或 Zustand。

## 2. cart 和 productItem 的關係

**cart 是購物車陣列：**
```javascript
cart = [
  { id: 1, name: "手錶", price: 1000 },
  { id: 2, name: "滑鼠", price: 500 }
]
```

**productItem 是 filter 的回調參數：**
```javascript
cart.filter(productItem => productItem.id !== product.id)
//              ↑
//        這是 cart 陣列中的每一個項目
```

**關係說明：**
- `cart`：購物車陣列
- `productItem`：`filter` 遍歷 `cart` 時的單一項目
- `product`：當前渲染的商品（來自 `map`）

**流程：**
1. `cart.map()` 渲染每個商品
2. 點擊移除時，`cart.filter()` 過濾掉該商品
3. `setCart()` 更新購物車狀態

## 3. setCart 包裝 filter 的語意

**當前寫法：**
```javascript

```

**語意：**
- 從購物車移除指定商品
- 用 `filter` 產生新陣列
- 用 `setCart` 更新狀態


**語意：**
- 更新當前時間
- 用 `set` 更新 store 狀態

**對比：**
- `setCart`：更新購物車陣列
- `set`：更新 store 的任意狀態

## 4. 完整的狀態更新模式

**Zustand 模式：**

```javascript
const useStore = create((set) => ({
  cart: [],

    cart: state.cart.filter(item => item.id !== productId)
```

### **JavaScript 模組系統**

在 JavaScript 中，有兩種匯出方式：

#### **1. Default Export（預設匯出）**
```javascript
// App.js
function App() { ... }
export default App;  // 預設匯出

// index.js
import App from '../App';  // 預設匯入
```

#### **2. Named Export（命名匯出）**
```javascript
// App.js
export function App() { ... }  // 命名匯出

// index.js
import { App } from '../App';  // 命名匯入
```

### **為什麼使用 `import App from '../App'`？**

1. `index.js` 是 Next.js 的首頁入口
2. 需要匯入主應用組件 `App`
3. 使用預設匯入語法 `import App from '../App'`
4. 因此 `App.js` 必須有 `export default`

```
zustand-cart/src/

│   └── index.js  ← Next.js 首頁路由
└── App.js        ← 主應用組件
```

### **匯入流程**

```
index.js
├── import App from '../App'  ← 匯入 App 組件
└── return <App />            ← 渲染 App 組件
```

- 每個檔案只能有一個 `export default`
