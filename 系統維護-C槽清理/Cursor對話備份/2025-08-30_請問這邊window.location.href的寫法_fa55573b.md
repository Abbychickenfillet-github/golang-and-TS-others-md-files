根據我的分析，我發現了一個重要的問題。在你的代碼中，`window.location.href` 是用來導航到前端頁面的，而不是 API 端點。讓我解釋一下：

## 問題分析

1. **當前代碼的作用**：
   ```javascript
   window.location.href = `http://localhost:3000/product/${product_id}`
   ```
   這行代碼是用來導航到前端的產品詳情頁面，不是 API 調用。

2. **環境變數的用途**：
   `NEXT_PUBLIC_API_BASE_URL` 是用來指向後端 API 服務器的地址（通常是 `http://localhost:3005`），而不是前端頁面的地址。

3. **正確的修改方式**：
   如果你想要使用環境變數來配置前端頁面的 URL，應該使用不同的環境變數，比如 `NEXT_PUBLIC_FRONTEND_BASE_URL`。

## 建議的解決方案

你有幾個選擇：

### 選項 1：保持當前寫法（推薦）
如果你想要導航到前端頁面，建議保持當前的寫法，但使用環境變數來配置前端 URL：

```javascript
onClick={() =>
  (window.location.href = `${process.env.NEXT_PUBLIC_FRONTEND_BASE_URL || 'http://localhost:3000'}/product/${product_id}`)
}
```

### 選項 2：使用 Next.js 的 Router（更推薦）
更好的做法是使用 Next.js 的 Router 來進行客戶端導航：

```javascript
import { useRouter } from 'next/router'

// 在組件內部
const router = useRouter()

// 然後在 onClick 中
onClick={() => router.push(`/product/${product_id}`)}
```

### 選項 3：如果你確實想要使用 API 地址
如果你想要導航到 API 端點（這通常不是你想要的效果），那麼才使用：

```javascript
onClick={() =>
  (window.location.href = `${process.env.NEXT_PUBLIC_API_BASE_URL}/product/${product_id}`)
}
```

## 我的建議

我建議使用**選項 2**，因為：
1. 它更符合 Next.js 的最佳實踐
2. 避免了硬編碼的 URL
3. 提供了更好的用戶體驗（不會重新加載整個頁面）
4. 自動處理不同環境的 URL

- 只能讀取 `httpOnly: false` 的 Cookie

**`document.cookie`** (前端):
- 瀏覽器端讀取 Cookie
- JavaScript 可以存取
- 只能讀取 `httpOnly: false` 的 Cookie
