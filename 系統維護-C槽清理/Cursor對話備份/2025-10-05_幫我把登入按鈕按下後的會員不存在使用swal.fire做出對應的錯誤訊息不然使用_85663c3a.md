## 🔍 動態路由參數的運作原理

### 📍 **什麼是動態路由參數？**

動態路由參數是 Next.js 的一個功能，讓你可以創建**一個 API 路由來處理多個不同的 ID**，而不需要為每個 ID 都創建一個單獨的檔案。

### 🗂️ **檔案結構說明**

```
app/(api)/api/timelog/
├── route.js          ← 處理 /api/timelog (GET, POST)
└── [id]/
    └── route.js      ← 處理 /api/timelog/1, /api/timelog/2, /api/timelog/123...
```

### 🔄 **實際運作流程**

1. **用戶在 Dashboard 看到時間記錄列表**
   ```
   網址：http://localhost:3001/dashboard
   顯示：所有時間記錄（ID: 1, 2, 3, 4, 5...）
   ```

2. **用戶點擊刪除按鈕**
   ```javascript
   // 前端代碼知道這個記錄的 ID 是 5
   const logId = 5  // 從資料中取得
   
   // 發送 DELETE 請求到動態路由
   fetch(`/api/timelog/${logId}`)  // 實際請求：/api/timelog/5
   ```

3. **後端動態路由處理**
   ```javascript
   // /api/timelog/[id]/route.js
   export async function DELETE(request, { params }) {
     const { id } = await params  // id = "5" (字串)
     // 用這個 ID 去資料庫刪除對應的記錄
   }
   ```
