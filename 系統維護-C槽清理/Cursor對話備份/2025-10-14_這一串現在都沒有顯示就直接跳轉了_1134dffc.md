### 2. **數據流向 (Data Flow)**
```
AuthProvider (認證狀態)
    ↓ pass down
UnifiedNavbar (導航欄)
    ↓ pass down  
Suspense (非同步載入邊界)
    ↓ pass down

```

## 🎯 各 Provider 的語意

### **AuthProvider**
- **語意**：管理全域認證狀態
- **職責**：提供 `isAuth`, `user`, `login`, `logout` 等認證相關狀態
- **範圍**：整個應用都需要知道用戶是否登入

### **LoaderProvider** 
- **語意**：管理全域載入動畫
- **職責**：提供 `showLoader`, `hideLoader` 等載入狀態
- **範圍**：任何組件都可能需要顯示載入動畫

### **CartProvider**
- **語意**：管理購物車狀態
- **職責**：提供購物車商品、數量、總價等狀態
- **範圍**：電商相關功能

### **ProductProvider**
- **語意**：管理商品搜尋和狀態
- **職責**：提供商品列表、搜尋條件等狀態
- **範圍**：商品展示和搜尋功能

## 🔄 為什麼這樣設計？

### **1. 認證狀態需要全域訪問**
```jsx
// UnifiedNavbar 需要知道用戶是否登入
const { isAuth, user } = useAuth()

// 根據登入狀態顯示不同內容
```

### **2. Suspense 處理非同步載入**
```jsx
<Suspense fallback={<Loading />}>
  {/* 這裡的組件可能需要載入數據 */}
  <DashboardPage />  // 可能需要載入用戶數據
  <ProductPage />    // 可能需要載入商品數據

```

### **3. 數據依賴關係**
```
AuthProvider (認證) 
    ↓ 用戶必須先登入
DashboardPage (儀表板)
    ↓ 需要載入用戶的時間記錄
Suspense (處理載入狀態)
```

- **AuthProvider**：使用 `isLoading` 狀態管理自己的載入
- **Suspense**：處理頁面組件的非同步載入
- **兩者分工明確**：認證載入 vs 頁面載入
- **避免衝突**：不會有載入狀態管理的重複
