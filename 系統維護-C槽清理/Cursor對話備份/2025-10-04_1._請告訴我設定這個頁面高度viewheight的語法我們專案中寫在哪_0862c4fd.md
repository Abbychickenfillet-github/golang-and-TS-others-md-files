`react/no-unescaped-entities` 是 ESLint 的一個規則，專門用於檢查 React JSX 中的**未轉義實體字符**。

## 什麼是未轉義實體字符？

在 JSX 中，某些特殊字符需要被轉義（escape），否則可能會導致語法錯誤或渲染問題。

### 常見需要轉義的字符：

1. **引號**：
   - `"` → `&quot;` 或 `&ldquo;` / `&rdquo;`
   - `'` → `&apos;` 或 `&#39;`

2. **其他特殊字符**：
   - `<` → `&lt;`
   - `>` → `&gt;`
   - `&` → `&amp;`

## 為什麼需要這個規則？

### 問題範例：
```jsx
// ❌ 錯誤 - 未轉義的引號
<p>點擊"測試調試API"使用專門的調試端點</p>

// ✅ 正確 - 轉義後的引號
<p>點擊&ldquo;測試調試API&rdquo;使用專門的調試端點</p>
```

### 原因：
1. **語法衝突**：JSX 中的引號可能與 JavaScript 字符串的引號衝突
2. **渲染問題**：某些字符可能無法正確顯示
3. **一致性**：確保代碼風格統一

## 問題分析：

1. **`d-flex` 與 `col-md-6` 衝突**：
   - `d-flex` 讓元素變成 flex 容器
   - `col-md-6` 是 Bootstrap Grid 系統的類別
   - 兩者不能同時使用，會互相干擾

2. **Flex 容器內的子元素**：
   - 在 `d-flex` 容器內，子元素會變成 flex items
   - `col-md-6` 的寬度設定會被 flex 行為覆蓋

## 解決方案：

### 方案 1：移除 Grid 類別，用 flex 控制寬度
```tsx
<input
  type="text"
  id="stepDescription"
  className="form-control"
  style={{ flex: '0 0 50%' }} // 或 width: '50%'
  // ... 其他屬性
/>
```

### 方案 2：改用 Bootstrap Grid 系統
```tsx
<div className="row">
  <div className="col-md-6">
    <input
      type="text"
      id="stepDescription"
      className="form-control"
      // ... 其他屬性
    />
  </div>
</div>
```

### 方案 3：用 Bootstrap 的 flex 工具類別
```tsx
<input
  type="text"
  id="stepDescription"
  className="form-control flex-fill" // flex-fill 讓它佔滿剩餘空間
  // ... 其他屬性
/>
```

**建議用方案 1**，因為你已經在用 `d-flex` 了，保持一致性比較好。
