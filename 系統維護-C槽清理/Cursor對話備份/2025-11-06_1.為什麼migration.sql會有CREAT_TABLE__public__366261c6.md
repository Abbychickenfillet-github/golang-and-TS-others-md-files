## "public" 的原因

**"public" 是 PostgreSQL 的預設 Schema（命名空間）**

### 簡單理解：

1. **Schema = 命名空間**：用來組織表的一個容器
2. **每個資料庫都有 public schema**：創建資料庫時自動建立
3. **所有表預設在 public**：除非特別指定其他 schema

### 可以省略嗎？

**技術上可以，但 Prisma 會明確寫出：**

```sql
-- ✅ 寫上 public（Prisma 標準）
CREATE TABLE "public"."user" (...);

-- ✅ 不寫也可以（PostgreSQL 會自動使用 public）
CREATE TABLE "user" (...);
```

### 為什麼 Prisma 要寫 "public"？

- **明確性**：清楚表明表在 public schema
- **避免歧義**：避免因 search_path 設定造成的問題
- **標準做法**：Prisma 的遷移檔案總是明確寫出 schema

**總結**：`public` 是 PostgreSQL 的預設命名空間，每一張表都會在 `public` schema 中（除非特別指定其他 schema）。Prisma 明確寫出 `"public"` 是為了讓 SQL 更清楚明瞭。

如果你想要簡化版的 migration-zeabur.sql（不寫 "public"），我可以幫你修改。這在功能上是一樣的，只是 Prisma 的標準做法是明確寫出。

## `DO $$` 語句說明

**`DO $$`** 是 PostgreSQL 的**匿名代碼塊（Anonymous Code Block）**語法，可執行一段 PL/pgSQL 代碼，無需創建函數。

### 為什麼需要它？

PostgreSQL 不支持某些 SQL 語句的條件執行：

```sql
-- ❌ 錯誤：PostgreSQL 不支持 IF EXISTS
TRUNCATE TABLE IF EXISTS "public"."user";
```

使用 `DO $$` 可以實現條件判斷：

```sql
-- ✅ 正確：使用 DO $$ 檢查表是否存在後再執行
DO $$
BEGIN
    IF EXISTS (SELECT FROM information_schema.tables 
               WHERE table_schema = 'public' AND table_name = 'user') THEN
        TRUNCATE TABLE "public"."user" CASCADE;
    END IF;
END $$;
```

### 語法結構

```sql
DO $$                    -- 開始匿名代碼塊
BEGIN                    -- 開始代碼塊
    -- 你的 PL/pgSQL 代碼
    -- 可以寫條件判斷、迴圈等
END $$;                  -- 結束代碼塊
```

### 為什麼用 `$$` 而不是單引號？

**`$$`** 是 PostgreSQL 的**美元符號標記（Dollar-quoting）**，好處是不需要轉義：

```sql
-- ❌ 使用單引號：需要轉義（將 ' 變成 ''）
DO '
BEGIN
    IF EXISTS (SELECT FROM information_schema.tables 
               WHERE table_schema = ''public'') THEN ...
END ';

-- ✅ 使用 $$：不需要轉義，更清晰
DO $$
BEGIN
    IF EXISTS (SELECT FROM information_schema.tables 
               WHERE table_schema = 'public') THEN ...
END $$;
```

### 在你的 migration-zeabur.sql 中的用途

```sql
-- 檢查表是否存在，如果存在就清空數據
DO $$
BEGIN
    IF EXISTS (SELECT FROM information_schema.tables 
               WHERE table_schema = 'public' AND table_name = 'user') THEN
        TRUNCATE TABLE "public"."user" CASCADE;
    END IF;
END $$;
```

這段代碼會：
1. 檢查 `public.user` 表是否存在
2. 如果存在，清空表中的所有數據
3. 如果不存在，跳過（不會報錯）

總結：`DO $$` 讓你在 SQL 腳本中執行條件邏輯，例如「如果表存在就清空數據」，而不需要創建正式函數。

## 1. CASCADE 作用说明 (`docs/postgresql-cascade-explanation.md`)

要点：
- CASCADE 不会删除子表本身，只会删除依赖关系或相关数据
- 三种使用场景：
  - `DROP TABLE CASCADE`：删除表及其依赖对象（不删除子表）
  - `TRUNCATE TABLE CASCADE`：清空表及其相关表的数据（不删除表结构）
  - `ON DELETE CASCADE`：删除父表记录时，自动删除子表相关记录（不删除子表）

## 2. Prisma vs PostgreSQL 语法差异 (`docs/prisma-vs-postgresql-syntax-differences.md`)

包含：
- 语法对比（声明式 vs 命令式）
- 数据类型对应表
- 主键、外键、索引的定义方式
- 字段映射和表名映射
- 级联操作的写法
- 完整语法对照表

两个文档都包含实际例子和你的 `migration-zeabur.sql` 中的相关代码说明。

**关键澄清**：CASCADE 不会删除子表本身，只会：
- 删除依赖关系（DROP TABLE CASCADE）
- 清空相关数据（TRUNCATE TABLE CASCADE）
- 删除相关记录（ON DELETE CASCADE）

子表结构仍然存在，只是数据或依赖关系被删除。

## 核心解释

**为什么需要两个反斜线 `\\`？**

1. **一个反斜线的问题**：
   ```javascript
   const path = 'C:\Users\Documents';
   // JavaScript 看到 \U，尝试解释为转义序列
   // \U 不是有效的转义序列，可能会报错或产生意外行为
   ```

2. **两个反斜线的解决方案**：
   ```javascript
   const path = 'C:\\Users\\Documents';
   // 第一个 \：告诉 JavaScript「下一个字符是字面量」
   // 第二个 \：实际的反斜线字符
   // 结果：字符串中存储的是 C:\Users\Documents（一个反斜线）
   ```

- **一个反斜线 `\`**：JavaScript 认为「这是转义序列的开始」

## 核心答案

### 使用 `DO $$`（推荐，不需要转义）

```sql
DO $$
BEGIN
    IF EXISTS (SELECT FROM information_schema.tables 
               WHERE table_schema = 'public' AND table_name = 'user') THEN
        TRUNCATE TABLE "public"."user" CASCADE;
    END IF;
END $$;
```

### 使用 `DO '`（需要转义）

```sql
DO '
BEGIN
    IF EXISTS (SELECT FROM information_schema.tables 
               WHERE table_schema = ''public'' AND table_name = ''user'') THEN
        TRUNCATE TABLE "public"."user" CASCADE;
    END IF;
END ';
```

**转义规则**：
- 所有单引号 `'` 都需要转义成两个单引号 `''`
- `'public'` → `''public''`
- `'user'` → `''user''`

## 为什么需要转义？

在 SQL 字符串中，单引号 `'` 用来标记字符串边界。如果字符串内也有单引号，需要用两个单引号 `''` 来表示一个单引号字符。

## Dollar-quoting 如何避免转义？

`$$` 使用不同的标记来标记字符串边界，所以：
- 字符串边界是 `$$`，不是 `'`
- 内部的 `'` 不会被误认为字符串边界
- 因此不需要转义
