## Cursor
是的，`form-control` 应该用在输入框上。这是 Bootstrap 的标准用法。

**Bootstrap 表单类的正确用法：**

1. **`form-control`** - 应该直接用在输入框上
   ```jsx
   <input className="form-control" />
   <textarea className="form-control" />
   <select className="form-control" />
   ```

2. **`form-control-lg`** / **`form-control-sm`** - 可以组合使用来调整大小
   ```jsx
   <input className="form-control form-control-lg" />  // 大尺寸
   <input className="form-control form-control-sm" />  // 小尺寸
   ```

3. **`input-group`** - 这是容器类，用于包裹输入框和按钮，让它们组合在一起
   ```jsx
   <div className="input-group">
     <input className="form-control" />
     <button className="btn">按钮</button>
   </div>
   ```

所以你的活动名称输入框现在使用 `form-control form-control-lg` 是正确的，会显示为大尺寸输入框。

之前的 `input-group input-group-large` 是错误的，因为：
- `input-group` 不应该直接用在 input 上（它是容器类）
- `input-group-large` 不是有效的 Bootstrap 类名

现在的代码是正确的。

## 详细解释

### `...` 是展开运算符
在这里用于展开数组。

### 逗号前后的意思：

1. **逗号前面**：`...state.savedActivities`
   - 展开 `state.savedActivities` 中的所有元素
   - 如果 `state.savedActivities = [活动1, 活动2]`，展开后得到元素 `活动1, 活动2`

2. **逗号后面**：`activityRecord`
   - 新添加的活动记录
   - 会被追加到数组末尾

### 完整含义：

```javascript
// 假设原来的数组是：
state.savedActivities = [活动A, 活动B]

// 执行这行代码后：
savedActivities = [...state.savedActivities, activityRecord]
// 等价于：
savedActivities = [活动A, 活动B, activityRecord]
//                     ↑           ↑        ↑
//                  原有元素1   原有元素2   新元素
```

### 为什么这样写？

这是不可变更新（Immutable Update）的写法，创建一个新数组而不是修改原数组，符合 React/Zustand 的最佳实践。

### 对比：

```javascript
// ❌ 错误方式（直接修改原数组，违反不可变性）
state.savedActivities.push(activityRecord)

// ✅ 正确方式（创建新数组）
savedActivities: [...state.savedActivities, activityRecord]
```

总结：这是在数组末尾添加新元素的不可变写法。

### 定义

- `{ disabled: true }`：对象，包含 disabled 属性
- `{}`：空对象，展开后不添加任何属性
- 原写法是想只在需要时添加 `disabled` 属性
- 新写法更安全：始终设置 `disabled`（true 或 false），避免 hydration 不一致

这样既避免了 hydration 错误，也让代码更清晰。
