## 宣告式 vs 陳述式 vs 表達式

### 1. **宣告式 (Declaration)**
宣告式是用來建立變數、函數或類別的語句：

```typescript
// 變數宣告
const name = 'John'
let age = 25
var isActive = true

// 函數宣告
function greet(name: string) {
  return `Hello, ${name}!`
}

// 類別宣告
class Person {
  constructor(public name: string) {}
}
```

### 2. **陳述式 (Statement)**
陳述式是執行動作的語句，不產生值：

```typescript
// 條件陳述式
if (condition) {
  // 執行某些動作
}

// 迴圈陳述式
for (let i = 0; i < 10; i++) {
  console.log(i)
}

// 賦值陳述式
voiceBtn.disabled = true  // 這就是你程式碼中的例子
```

### 3. **表達式 (Expression)**
表達式會產生一個值：

```typescript
// 算術表達式
2 + 3  // 結果: 5

// 函數呼叫表達式
Math.max(1, 2, 3)  // 結果: 3

// 條件表達式
age > 18 ? 'adult' : 'minor'  // 結果: 'adult' 或 'minor'

// 型別轉換表達式
(voiceBtn as HTMLButtonElement)  // 結果: 轉換後的元素
```


## 解釋大括號 `{ onResult }` 的用法

### 1. **ESLint 註解**
```typescript
// eslint-disable-next-line @typescript-eslint/no-unused-vars
```
這是正確的 ESLint 註解寫法，用來禁用下一行的 TypeScript 未使用變數警告。

### 2. **大括號 `{ onResult }` 是解構賦值 (Destructuring)**

```typescript
// 有解構賦值
function VoiceInput({ onResult }: { onResult: (_text: string) => void }) {
  // 可以直接使用 onResult
  onResult("語音結果")
}

// 沒有解構賦值
function VoiceInput(props: { onResult: (_text: string) => void }) {
  // 需要使用 props.onResult
  props.onResult("語音結果")
}
```

### 3. **為什麼要用解構賦值？**

**優點：**
- **簡潔性**: 直接使用 `onResult` 而不是 `props.onResult`
- **可讀性**: 一眼就能看出這個組件需要什麼 props
- **TypeScript 支援**: 更好的型別推斷

**比較：**
```typescript
// 解構賦值版本 (推薦)
function VoiceInput({ onResult, onError }: { 
  onResult: (text: string) => void
  onError: (error: string) => void 
}) {
  onResult("結果")
  onError("錯誤")
}

// 非解構版本
function VoiceInput(props: { 
  onResult: (text: string) => void
  onError: (error: string) => void 
}) {
  props.onResult("結果")
  props.onError("錯誤")
}
```

### 4. **其他 ESLint 註解寫法**

```typescript
// 禁用特定規則
// eslint-disable-next-line @typescript-eslint/no-unused-vars

// 禁用所有規則
// eslint-disable-next-line

// 禁用多個規則
// eslint-disable-next-line @typescript-eslint/no-unused-vars, @typescript-eslint/no-explicit-any

// 整個檔案禁用
/* eslint-disable @typescript-eslint/no-unused-vars */
```

所以大括號 `{ onResult }` 是 React 組件中非常常見的解構賦值語法，讓你可以直接使用 props 而不需要 `props.` 前綴！

## 為什麼是 `void` 而不是有回傳值？

### 1. **`onResult` 是回調函數 (Callback Function)**

```typescript
// 語音識別結果的流向
VoiceInput 組件 → 語音識別 → 得到文字 → 通知父組件

// 不是這樣：
VoiceInput 組件 → 語音識別 → 得到文字 → 回傳給父組件
```

### 2. **實際的數據流向**

```typescript
// 子組件 (VoiceInput)
recognizer.onresult = (event: any) => {
  const text = event.results?.[0]?.[0]?.transcript
  if (text) {
    onResult(text) // 把結果"推送"給父組件
  }
}

// 父組件 (TimeLogClient)
const handleVoiceResultWrapper = (text: string) => {
  handleVoiceResult(text) // 接收結果並處理
}

// 使用
<VoiceInput onResult={handleVoiceResultWrapper} />
```

### 3. **為什麼不是有回傳值？**

```typescript
// 如果是有回傳值的設計 (不推薦)
function VoiceInput(): string {
  // 問題：語音識別是異步的！
  // 我們無法立即回傳結果
  return "???" // 不知道要回傳什麼
}

// 正確的設計 (回調函數)
function VoiceInput({ onResult }: { onResult: (text: string) => void }) {
  // 當語音識別完成時，調用回調函數
  onResult("識別到的文字")
}
```

### 4. **React 組件的通則**

```typescript
// React 組件通常不直接回傳數據給父組件
// 而是通過 props 回調函數來"通知"父組件

// 子組件
function Child({ onData }: { onData: (data: string) => void }) {
  const handleClick = () => {
    onData("一些數據") // 通知父組件
  }
  return <button onClick={handleClick}>點擊</button>
}

// 父組件
function Parent() {
  const handleData = (data: string) => {
    console.log("收到數據:", data) // 處理數據
  }
  
  return <Child onData={handleData} />
}
```

### 5. **語音識別的特殊性**

語音識別是**異步操作**，需要時間來處理：
- 用戶說話 → 麥克風錄音 → 網路傳輸 → 語音識別服務 → 回傳文字

這個過程可能需要幾秒鐘，所以無法同步回傳結果。

## 總結

`onResult: (text: string) => void` 的意思是：
- **輸入**: `text: string` (語音識別的結果)
- **輸出**: `void` (沒有回傳值，只是通知父組件)
- **目的**: 把語音識別的結果"推送"給父組件處理

這是一個**事件驅動**的設計模式，而不是**請求-回應**模式！
