- 默认参数为 `"ALL"`
- 未匹配到指定方式时回退为 `"ALL"`
- 电子钱包（EasyWallet、LinePay 等）会强制设为 `"ALL"`

### 2. 什么是 ECPay 的全方位金流付款？

根据 [ECPay 官方文档](https://developers.ecpay.com.tw/?p=2864)，全方位金流付款（ChoosePayment = "ALL"）的特点：

- 应用场景：特店将 `ChoosePayment` 设为 `"ALL"`，由绿界在付款页面显示所有已开通的付款方式，让消费者自行选择。
- 支持的付款方式（取决于后台开通）：
  - 信用卡一次付清
  - 信用卡分期付款
  - 信用卡定期定额
  - Apple Pay
  - ATM 虚拟账号
  - CVS 超商代码
  - BARCODE 超商条码
  - WebATM 网络转账
  - TWQR 欧付宝
  - BNPL 无卡分期
  - 微信支付
  - 绿界 PAY 付款
  - 其他已开通的方式

- 与信用卡一次付清的区别：
  - `ChoosePayment = "Credit"`：仅显示信用卡一次付清
  - `ChoosePayment = "ALL"`：显示所有已开通的付款方式，由消费者选择

### 3. 项目中的实际行为

根据代码逻辑：
1. 默认使用 `"ALL"`，让消费者选择付款方式
2. 对于电子钱包（EasyWallet、LinePay 等），强制使用 `"ALL"`，因为这些方式在 ECPay 中通常需要通过选择页面触发
3. 代码注释也提到：`"如果我們想讓 ECPay 顯示所有選項，使用 ALL 是最安全的"`

总结：项目使用全方位金流付款（ALL），而非仅限信用卡一次付清。这样可提供更多付款选项，提升用户体验。

## Python 的 ECPay SDK

Python 的 ECPay SDK 是 **`ECPayAIO_Python`**，对应 Node.js 的 `ecpay_aio_nodejs`。

### 官方 Python SDK 信息

- GitHub 仓库：https://github.com/ECPay/ECPayAIO_Python
- 语言：Python 3
- 许可证：GPLv2 or later
- 状态：活跃维护中（最后更新：2025-07-23）

### 项目当前状态

项目目前没有使用官方 SDK，而是自行实现了相关功能：

1. 没有安装 SDK：`requirements.txt` 中没有 ECPay 相关包
2. 没有 SDK 文件：`backend/app/utils/` 下没有 `payment` 目录或 `ecpay_payment_sdk.py`
3. 自行实现：在 `backend/app/utils/database/crud_helpers.py` 中实现了 `generate_check_mac_value` 函数

### 如果要使用官方 SDK

根据文档 `flow&docs/ECPAY_INTEGRATION_GUIDE.md`，安装方式：

```bash
# 方式 1: Clone 并安装
git clone https://github.com/ECPay/ECPayAIO_Python.git
cd ECPayAIO_Python
pip install -e .

# 方式 2: 复制 SDK 文件到项目（推荐）
mkdir -p backend/app/utils/payment
# 复制 ecpay_payment_sdk.py 到 backend/app/utils/payment/
```

### 对比

| 语言 | SDK 名称 | GitHub 仓库 |
|------|---------|-------------|
| **Node.js** | `ecpay_aio_nodejs` | https://github.com/ECPay/ECPayAIO_Node.js |
| **Python** | `ECPayAIO_Python` | https://github.com/ECPay/ECPayAIO_Python |

总结：Python 的 SDK 是 `ECPayAIO_Python`，但项目目前未使用，而是自行实现了 CheckMacValue 生成和验证逻辑。
