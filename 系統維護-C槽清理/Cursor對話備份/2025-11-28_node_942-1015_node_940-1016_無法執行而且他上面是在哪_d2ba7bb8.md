`DEALLOCATE PREPARE stmt;` 是 MySQL 中為了釋放先前 `PREPARE stmt FROM ...` 建立的預備語句所用的指令。簡單說：

1. `PREPARE stmt FROM 'SQL語句';` 會把那段 SQL 編譯成一個預備語句，名稱叫 `stmt`。
2. `EXECUTE stmt;` 會實際執行這個預備語句。
3. `DEALLOCATE PREPARE stmt;` 則是在執行完後，把這個預備語句從記憶體中釋放掉，避免占用資源。

所以那一行就是在說：「我已經用完名為 `stmt` 的預備語句了，現在把它清掉。」

## 檔案用途

這個 `nginx.conf` 是前端 Docker 容器中 Nginx 的配置檔，用於提供 React 編譯後的靜態檔案。

## 逐行解釋

```nginx
server {
  listen 80;                    # Nginx 監聽 80 端口（HTTP）
  
  location / {                  # 處理所有路徑（根路徑及子路徑）
    root /usr/share/nginx/html; # 靜態檔案的位置（React 編譯後的檔案）
    index index.html index.htm; # 預設首頁檔案
    
    # ⭐ 最重要：SPA 路由配置
    try_files $uri /index.html =404;
    # 意思：
    # 1. 先嘗試找對應的檔案（$uri）
    # 2. 如果找不到，就返回 index.html（讓 React Router 處理）
    # 3. 如果 index.html 也找不到，返回 404
  }
  
  include /etc/nginx/extra-conf.d/*.conf;  # 包含額外配置檔案
```

## 2. 怎樣指定 URI？

`try_files` 會依序嘗試檔案，直到找到為止：

```nginx
try_files $uri /index.html =404;
```

- `$uri`：用戶請求的 URI（例如 `/dashboard` → 嘗試找 `/usr/share/nginx/html/dashboard`）
- `/index.html`：如果找不到，回退到 `index.html`（讓 React Router 處理）
- `=404`：如果 `index.html` 也不存在，返回 404

### 範例

用戶訪問 `https://future-sign1.zeabur.app/dashboard`：

1. Nginx 收到請求：`/dashboard`
2. `try_files` 嘗試：
   - 先找 `/usr/share/nginx/html/dashboard`（不存在）
   - 回退到 `/usr/share/nginx/html/index.html`（存在）
   - 返回 `index.html`
3. 瀏覽器載入 `index.html`，React Router 接管，顯示 `/dashboard` 頁面

### 如果想自訂路徑

可以修改 `root` 指令：

```nginx
location / {
  root /custom/path/to/files;  # 改成你想要的路徑
  index index.html;
  try_files $uri /index.html =404;
}
```

但通常不需要改，`/usr/share/nginx/html` 是 Nginx 的標準路徑。
