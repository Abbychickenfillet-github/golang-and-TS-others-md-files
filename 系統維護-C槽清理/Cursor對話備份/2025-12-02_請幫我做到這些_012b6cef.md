## CIDR 数字越小，涵盖的 IP 越多

### 原理说明：

CIDR 中的数字表示“网络位的数量”，剩余的是“主机位”。

- `/28` = 前 28 位是网络位，后 4 位是主机位
  - 主机位：2^4 = 16 个 IP
  - 范围：211.249.40.0 到 211.249.40.15（只涵盖 16 个 IP）

- `/27` = 前 27 位是网络位，后 5 位是主机位
  - 主机位：2^5 = 32 个 IP
  - 范围：211.249.40.0 到 211.249.40.31（涵盖 32 个 IP）

### 直观理解：

```
IP 地址是 32 位（4 个字节）

/28: [网络位 28 位][主机位 4 位]  → 只有 4 位可以变化 → 2^4 = 16 个 IP
/27: [网络位 27 位][主机位 5 位]  → 有 5 位可以变化 → 2^5 = 32 个 IP
/26: [网络位 26 位][主机位 6 位]  → 有 6 位可以变化 → 2^6 = 64 个 IP
/24: [网络位 24 位][主机位 8 位]  → 有 8 位可以变化 → 2^8 = 256 个 IP
```

### 为什么需要从 /28 改成 /27？

您需要涵盖 211.249.40.1 到 211.249.40.30（30 个 IP）

- `/28` 只能涵盖 16 个 IP（211.249.40.0-15）→ 不够
- `/27` 可以涵盖 32 个 IP（211.249.40.0-31）→ 足够

### 记忆口诀：

数字越小 = 网络位越少 = 主机位越多 = 涵盖的 IP 越多

就像：
- `/24`（常见）涵盖 256 个 IP（一个完整的 C 类网段）
- `/32` 只涵盖 1 个 IP（精确匹配）

所以从 `/28` 改成 `/27` 是正确的，因为需要涵盖更多 IP 地址。
