## MySQL 命令參數說明

- `-p` 或 `--password`：提示輸入密碼（小寫 p）
- `-P` 或 `--port`：指定埠號（大寫 P）

## 正確的命令格式

```bash
# 基本格式（使用預設埠 3306，會提示輸入密碼）
mysql -u root -p future_sign < backend/sql/130_add_currency_symbol_fields.sql

# 如果需要指定埠號（例如 3307）
mysql -u root -p -P 3307 future_sign < backend/sql/130_add_currency_symbol_fields.sql

# 或者使用完整格式
mysql -u root -p --port=3307 future_sign < backend/sql/130_add_currency_symbol_fields.sql
```

## 其他常用選項

```bash
# 指定主機和埠號
mysql -u root -p -h localhost -P 3306 future_sign < backend/sql/130_add_currency_symbol_fields.sql

# 如果密碼直接寫在命令中（不建議，有安全風險）
mysql -u root -pYourPassword future_sign < backend/sql/130_add_currency_symbol_fields.sql
```

## 執行流程

當您執行 `mysql -u root -p` 時：
1. 系統會提示：`Enter password:`
2. 輸入密碼（不會顯示）
3. 按 Enter 執行

## 解決的問題

### 為什麼 `.py` 文件會有問題？

1. 可能被 Python 誤導入
   - 如果目錄在 Python 路徑中，Python 可能會嘗試導入這些文件
   - 可能導致 `ImportError` 或命名衝突

2. 代碼檢查工具會處理
   - mypy、pylint 等工具會檢查 `.py` 文件
   - 可能產生不必要的錯誤報告

3. IDE 可能誤認為是可執行代碼
   - 某些 IDE 可能會嘗試執行或檢查這些文件

### 使用 `.py.txt` 的好處

1. 不會被 Python 導入
   - `.txt` 擴展名確保 Python 不會將其視為模組

2. 編輯器仍可提供語法高亮
   - 許多編輯器可以識別 `.py.txt` 中的 Python 語法

3. 明確表示這是文檔
   - 文件名清楚表明這是文檔文件，不是可執行代碼

4. 避免代碼檢查工具處理
   - 代碼檢查工具通常不會處理 `.txt` 文件

現在文件已安全，不會造成任何問題。
