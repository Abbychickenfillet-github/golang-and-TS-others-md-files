## 為什麼 `prebuild` 可以設定為 `|| true`

### 原本的問題
```json
"prebuild": "npm run lint"
```
- 若 ESLint 失敗，整個建置會中斷
- 在 Zeabur 等平台，`devDependencies` 可能未安裝
- 導致 `eslint not found` 錯誤

### 解決方案
```json
"prebuild": "npm run lint || true"
```

### `|| true` 的作用
1. 允許 ESLint 失敗而不中斷建置
2. 確保建置流程繼續
3. 在生產環境中避免因 lint 失敗導致部署失敗

### 其他可選方案

#### 方案 1：條件式執行
```json
"prebuild": "if [ \"$NODE_ENV\" = \"production\" ]; then npm run lint || true; else npm run lint; fi"
```

#### 方案 2：分離建置腳本
```json
"prebuild": "echo 'Pre-build checks...'",
"build": "npm run lint && next build"
```

#### 方案 3：使用 cross-env
```json
"prebuild": "cross-env-shell \"npm run lint || echo 'Lint failed but continuing...'\""
```

### 最佳實踐建議
1. 本地開發：保留嚴格 lint 檢查
2. 生產部署：使用 `|| true` 避免中斷
3. CI/CD：在 CI 中單獨執行 lint 檢查

### 你的專案設定
```json
"prebuild": "npm run lint || true"
```
- 開發環境：lint 失敗會顯示錯誤但不中斷
- 生產環境：lint 失敗不會阻止部署
- 程式碼品質：仍會執行 lint，只是不強制要求通過

此設定在部署環境中更穩定。
