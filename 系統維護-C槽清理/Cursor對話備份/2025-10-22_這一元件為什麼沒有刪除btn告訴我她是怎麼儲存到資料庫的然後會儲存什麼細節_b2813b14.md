
## MutationObserver 是什麼？

`MutationObserver` 是瀏覽器原生的 JavaScript API，用來監聽 DOM 樹的變化。它不依賴任何第三方庫，包括 SWR。

### 主要用途：
- 監聽 DOM 節點的添加、刪除、修改
- 監聽屬性變化
- 監聽文本內容變化

### 在我們的 VoiceInput 組件中的使用：

```typescript
const observer = new MutationObserver(() => {
  // 當 DOM 發生變化時執行
  voiceBtn = bindButtonEvent()
  if (voiceBtn) {
    observer.disconnect() // 找到按鈕後停止監聽
  }
})

observer.observe(document.body, {
  childList: true,    // 監聽子節點的添加/刪除
  subtree: true      // 監聽整個子樹的變化
})
```

## 為什麼需要 MutationObserver？

在我們的案例中，問題是：
1. `VoiceInput` 組件在 `useEffect` 中嘗試找到按鈕
2. 但按鈕可能還沒有渲染到 DOM 中
3. 所以 `document.getElementById(buttonId)` 返回 `null`
4. 事件監聽器無法綁定

`MutationObserver` 解決了這個時序問題，它會等待按鈕被添加到 DOM 後再綁定事件。

## 其他解決方案

除了 `MutationObserver`，還有其他方法：

1. **使用 `setTimeout`**：
```javascript
setTimeout(() => {
  const btn = document.getElementById(buttonId)
  // 綁定事件
}, 100)
```

2. **使用 `requestAnimationFrame`**：
```javascript
requestAnimationFrame(() => {
  const btn = document.getElementById(buttonId)
  // 綁定事件
})
```

3. **重新設計組件結構**：
讓 `VoiceInput` 組件直接渲染按鈕，而不是依賴外部按鈕。

`MutationObserver` 是最可靠的解決方案，因為它能準確地知道 DOM 何時發生變化，而不需要猜測延遲時間。

```typescript
  buttonRef: React.RefObject<HTMLButtonElement>
}) {
  // 直接使用 ref
  if (buttonRef.current) {
    buttonRef.current.addEventListener('click', handleButtonClick)
  }
}
```

### 3. **父組件控制**

## 🎯 這個方案的優勢：

1. **性能更好**：不需要 DOM 查詢
2. **更可靠**：直接引用，不會有找不到元素的問題
3. **更清晰**：父組件明確控制按鈕的引用
4. **更安全**：避免了 `getElementById` 可能返回 `null` 的問題
5. **更 React 化**：使用 React 的 ref 機制而不是原生 DOM API

## 🔧 工作原理：

1. **父組件創建 ref**：`const titleVoiceButtonRef = useRef(null)`
2. **傳遞給子組件**：`<VoiceInput buttonRef={titleVoiceButtonRef} />`
3. **綁定到按鈕**：`<button ref={titleVoiceButtonRef}>🎤 語音</button>`
4. **子組件使用 ref**：`buttonRef.current.addEventListener('click', handleButtonClick)`
