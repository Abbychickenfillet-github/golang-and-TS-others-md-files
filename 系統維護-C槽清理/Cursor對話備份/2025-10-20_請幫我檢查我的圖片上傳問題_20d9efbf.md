## 📋 問題回答總結

### 1. **file.arrayBuffer() vs Buffer.from(bytes) 差異**

| 特性 | ArrayBuffer | Buffer |
|------|-------------|--------|
| **環境** | 瀏覽器標準 API | Node.js 專用 |
| **可變性** | 不可變 (immutable) | 可變 (mutable) |
| **操作方式** | 透過 TypedArray | 直接操作 |
| **方法** | 有限 | 豐富 (.pipe(), .toString() 等) |
| **用途** | 原始二進位資料 | 二進位資料處理 |

**轉換流程**：
```
File → ArrayBuffer → Buffer → Stream → Cloudinary
```

### 2. **lib/cloudinary.js 執行順序**

```
模組載入時：
1. import cloudinary
2. cloudinary.config() ← 最先執行
3. 定義函數 (uploadImage, deleteImage)
4. 匯出模組

API 請求時：
1. 呼叫 uploadImage()
2. 建立 uploadStream
3. Buffer → Stream
4. 上傳到 Cloudinary
```

### 3. **config vs lib 執行順序**

**config 先執行**：
- **時機**: 模組載入時 (應用啟動)
- **頻率**: 只執行一次
- **目的**: 設定 Cloudinary 認證資訊

**lib 函數後執行**：
- **時機**: API 請求時
- **頻率**: 每次上傳都執行
- **目的**: 處理具體上傳邏輯

- **config 在模組載入時執行**，比 API 路由更早

## 問題回答總結

### 1. 兩組上傳的圖片會到同一個資料夾嗎？

不會，它們會到不同的資料夾：
- 頭貼上傳 → `avatars` 資料夾（第 76 行）
- 一般圖片上傳 → `general` 資料夾（第 31 行）

換密鑰不會影響資料夾：
- 資料夾名稱在程式碼中設定，不是由密鑰決定
- 只要使用相同的 `cloud_name`，資料夾結構就會保持相同

後台看不到資料夾的原因：
- Media Library 預設顯示所有圖片（不分資料夾）
- 需要點擊左側的資料夾選單或使用搜尋 `folder:avatars` 才能看到

### 2. My Default Preset 是什麼意思？

Cloudinary 的預設轉換設定（Transformation Preset）：
- 預先定義一組圖片轉換參數
- 可在上傳時自動套用
- 目前專案使用程式碼中的轉換設定，而非後台的 Preset

### 3. Dynamic Folder 在 Cloudinary 是什麼意思？

根據動態條件決定資料夾名稱的功能：
- 例如：`uploads/2024/1`（根據日期）
- 例如：`users/123/avatars`（根據用戶ID）
- 目前專案使用固定資料夾名稱（`avatars` 和 `general`）

### 4. 資料夾名稱在哪裡設定？

- `avatars` 資料夾：`app/(api)/api/upload/avatar/route.js` 第 76 行
- `general` 資料夾：`app/(api)/api/upload/image/route.js` 第 31 行

### 5. uploadImage 和 GENERAL_IMAGE_TRANSFORMATIONS 介紹

#### uploadImage 函數
- 位置：`lib/cloudinary.js` 第 43 行
- 功能：Cloudinary 圖片上傳的核心函數
- 用途：將 Buffer 轉為 Stream 並上傳，支援自訂資料夾、公開ID、轉換設定

#### GENERAL_IMAGE_TRANSFORMATIONS
- 位置：`lib/cloudinary.js` 第 162 行
- 功能：一般圖片的轉換設定
- 參數：
  - `width: 800, height: 600` - 限制最大尺寸
  - `crop: 'limit'` - 保持比例，不裁切
  - `format: 'webp'` - 轉為 WebP 格式
  - `quality: 'auto'` - 自動調整品質
