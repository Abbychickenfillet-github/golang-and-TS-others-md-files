## 🐛 問題分析

**錯誤原因**：
- 在 Next.js 中，組件會在服務端（Server-side）先渲染一次
- 服務端沒有 `document` 對象，所以 `document.createElement` 會報錯
- 這就是為什麼你看到 `ReferenceError: document is not defined` 的原因

## ✅ 解決方案

我採用了兩種方法來修正這個問題：

### 1. **移除有問題的程式碼**
```javascript
// 移除了這段會導致服務端錯誤的程式碼
const style = document.createElement('style');
document.head.appendChild(style);
```

### 2. **將動畫樣式移到全局 CSS**
在 `styles/globals.css` 中添加了：
```css
@keyframes bounce {
  0%, 20%, 50%, 80%, 100% {
    transform: translateY(0);
  }
  40% {
    transform: translateY(-30px);
  }
  60% {
    transform: translateY(-15px);
  }
}
```

## 📝 `useRef(null)` 的詳細解釋

### 為什麼使用 `null` 作為初始值？

```javascript
const inputRef = useRef(null);  // ← 這裡為什麼是 null？
```

**原因說明：**

1. **表示「還沒有 DOM 元素」**
   - `null` 表示 ref 還沒有連接到任何 DOM 元素
   - 這是 React 的慣例，表示「空」或「未初始化」的狀態

2. **類型安全檢查**
   ```javascript
   if (inputRef.current) {  // ← 檢查是否為 null
     inputRef.current.focus();  // 只有不是 null 才執行
   }
   ```

3. **React 的標準做法**
   - React 官方文檔推薦使用 `null` 作為 DOM ref 的初始值
   - 這樣可以清楚地區分「有元素」和「沒有元素」的狀態

### 其他可能的初始值對比：

```javascript
// ❌ 不推薦 - 會造成混淆
const inputRef = useRef(undefined);
const inputRef = useRef('');

// ✅ 推薦 - 清楚表示「沒有元素」
const inputRef = useRef(null);

// ✅ 也可以 - 但通常用於非 DOM 用途
const countRef = useRef(0);
const timerRef = useRef();
```

### 實際運作流程：

```javascript
// 1. 初始狀態
const inputRef = useRef(null);
console.log(inputRef.current); // null

// 2. React 渲染後，將 DOM 元素賦值給 ref
<input ref={inputRef} />
// 此時 inputRef.current 變成實際的 DOM 元素

// 3. 使用時檢查
if (inputRef.current) {  // 檢查不是 null
  inputRef.current.focus();  // 安全地使用 DOM 元素
}
```

### 為什麼需要檢查 `if (inputRef.current)`？

```javascript
// ❌ 危險 - 如果元素還沒渲染會報錯
inputRef.current.focus();  // 可能報錯：Cannot read property 'focus' of null

// ✅ 安全 - 先檢查再使用
if (inputRef.current) {
  inputRef.current.focus();  // 安全
}
```

## 🎯 總結

`useRef(null)` 中的 `null` 是：
- **初始狀態**：表示 ref 還沒有連接到 DOM 元素
- **安全檢查**：讓我們可以檢查元素是否存在
- **React 慣例**：官方推薦的做法
- **類型安全**：避免在元素不存在時出錯

這就是為什麼我們總是看到 `useRef(null)` 和 `if (inputRef.current)` 這樣的組合！
