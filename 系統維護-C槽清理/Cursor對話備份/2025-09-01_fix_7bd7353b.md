## 🔍 什麼時候需要把函式定義在 `useEffect` 內部？

### 1. **當函式依賴於組件的 props 或 state 時**
```javascript
// ❌ 錯誤：函式會導致無限循環

  const response = await fetch(`/api/data/${userId}`)
  setData(response.data)
}

  fetchData()
}, [userId, fetchData]) // fetchData 每次都會重新創建！

// ✅ 正確：函式定義在 useEffect 內部
useEffect(() => {
  const fetchData = async () => {
    const response = await fetch(`/api/data/${userId}`)
    setData(response.data)
  }
  fetchData()
}, [userId]) // 只依賴 userId
```

### 2. **當函式會導致依賴數組變化時**
```javascript
// ❌ 錯誤：每次渲染都會創建新的函式
const handleClick = () => {
  console.log('clicked')
}

useEffect(() => {
  document.addEventListener('click', handleClick)
  return () => document.removeEventListener('click', handleClick)
}, [handleClick]) // handleClick 每次都會變化！

// ✅ 正確：函式定義在 useEffect 內部
useEffect(() => {
  const handleClick = () => {
    console.log('clicked')
  }
  document.addEventListener('click', handleClick)
  return () => document.removeEventListener('click', handleClick)
}, []) // 空依賴數組
```

## ✅ 什麼時候可以把函式拉出來？

### 1. **當函式不依賴於組件的 props 或 state 時**
```javascript
// ✅ 可以拉出來：函式是純函式，不依賴組件狀態
const formatDate = (date) => {
  return new Date(date).toLocaleDateString()
}

useEffect(() => {
  const formattedDate = formatDate(someDate)
  setDisplayDate(formattedDate)
}, [someDate])
```

### 2. **當函式使用 `useCallback` 包裝時**

// ✅ 可以拉出來：使用 useCallback 避免重新創建
const fetchData = useCallback(async () => {

- **用途**：帶 token 的請求（但現在不需要了，因為有攔截器）

### **`fetcherWithObject`**：
- **用途**：將對象參數轉換為 URL 查詢字符串
- **使用場景**：`fetcherWithObject({ url: '/api/data', args: { page: 1, limit: 10 } })`

## 🔄 4. fetch vs axios 的後端路由寫法

### **使用 axios（推薦）**：
```javascript
// 前端
import axiosInstance from '@/services/axios-instance'

const response = await axiosInstance.get('/api/dashboard/123')
```

### **使用原生 fetch**：
```javascript
// 前端
const response = await fetch('/api/dashboard/123', {
  credentials: 'include', // 重要：發送 cookies
  headers: {
    'Authorization': `Bearer ${token}` // 手動添加 token
  }
})
```

### **後端路由寫法（兩者都適用）**：
```javascript
// 後端
router.get('/:user_id', authenticate, async (req, res) => {
  // authenticate 中間件會處理認證
  // 無論是 axios 還是 fetch 都能正常工作
})
```
