## 你
fabric namespace具體是什麼意思

## Cursor
在 Fabric.js v5 以前，整個函式庫對外只有一個全域物件 `fabric`，所有類別都掛在這個“命名空間”底下，所以寫法是：

```ts
import { fabric } from 'fabric';

const canvas = new fabric.Canvas('canvas-id');
const rect = new fabric.Rect({ width: 100, height: 50 });
canvas.add(rect);
```

這裡的 `fabric` 就是一個 namespace（類似一個物件），裡面包含 `Canvas`、`Rect`、`Circle` 等建構子。TypeScript 的 `@types/fabric` 也是依照這種設計把所有類別放在 `fabric` 這個 namespace 之下。

從 Fabric v6 開始，官方把這種 namespace 風格改成 ES 模組輸出，每個類別都獨立 export，因此只能寫：

```ts
import { Canvas, Rect } from 'fabric';

const canvas = new Canvas('canvas-id');
const rect = new Rect({ width: 100, height: 50 });
canvas.add(rect);
```

不再提供 `fabric.Canvas` 這樣的 namespace；所以如果安裝 v6，但程式還用 `fabric.Canvas`，就會出現「fabric has no exported member 'fabric'」的錯誤。

總結：**Fabric namespace** 就是舊版 Fabric.js 中對應元件的全域物件 `fabric`。目前的程式與型別定義都依賴這個 namespace，因此必須使用 v5 或維持舊版寫法；若想升級到 v6，需要改為新的模組化 API。
