### 🔧 **問題原因**

#### **CSS 層疊上下文（Stacking Context）**
- `.geometricShape` 有 `position: relative`，這會創建一個新的層疊上下文
- 在層疊上下文內，子元素的 z-index 只在其父層疊上下文內有效
- 即使表單卡片的 z-index 是 4，也無法超越父層疊上下文的限制

### 🛠️ **修復方案**

#### **1. 移除 position: relative**
```scss
.geometricShape {
  min-height: 100vh; // 移除了 position: relative
}
```

## 🔧 **Autoprefixer 是什麼？**

### 📋 **基本概念**

**Autoprefixer** 是一個 CSS 後處理工具，它會自動為你的 CSS 添加瀏覽器前綴（vendor prefixes）。

### 🎯 **主要功能**

#### **1. 自動添加瀏覽器前綴**
```css
/* 你寫的 CSS */
.example {
  display: flex;
  align-items: center;
  justify-content: center;
}

/* Autoprefixer 自動轉換為 */
.example {
  display: -webkit-box;
  display: -ms-flexbox;
  display: flex;
  -webkit-box-align: center;
  -ms-flex-align: center;
  align-items: center;
  -webkit-box-pack: center;
  -ms-flex-pack: center;
  justify-content: center;
}
```

#### **2. 跨瀏覽器兼容性**
- **WebKit** 前綴：`-webkit-`（Chrome, Safari, Edge）
- **Mozilla** 前綴：`-moz-`（Firefox）
- **Microsoft** 前綴：`-ms-`（舊版 IE/Edge）
- **Opera** 前綴：`-o-`（舊版 Opera）

### 🛠️ **在 Next.js 中的應用**

#### **自動整合**
Next.js 預設就整合了 Autoprefixer，所以：
- 你不需要手動添加前綴
- 編譯時自動處理
- 根據目標瀏覽器自動添加必要的前綴

#### **配置範例**
```javascript
// next.config.js
module.exports = {
  // Autoprefixer 會根據 browserslist 配置自動工作
}
```

### ⚠️ **你遇到的警告**

#### **警告內容**
```
autoprefixer: start value has mixed support, consider using flex-start instead
```

#### **原因**
- `align-items: start` 和 `justify-content: start` 是較新的 CSS 值
- 某些瀏覽器支援度不一致
- Autoprefixer 建議使用更廣泛支援的 `flex-start`

#### **解決方案**
```css
/* 不建議 */
align-items: start;
justify-content: start;

/* 建議 */
align-items: flex-start;
justify-content: flex-start;
```

### 🎯 **實際好處**

#### **1. 開發效率**
- 不需要記住哪些屬性需要前綴
- 不需要手動添加前綴
- 專注於 CSS 邏輯，不用擔心兼容性

#### **2. 維護性**
- 自動更新前綴需求
- 根據瀏覽器使用統計調整
- 減少手動維護工作

#### **3. 性能優化**
- 只添加必要的前綴
- 避免過度前綴化
- 根據目標瀏覽器精確添加

### 💡 **總結**

Autoprefixer 是一個**自動化工具**，讓你可以：
- 寫現代 CSS 語法
- 自動獲得跨瀏覽器兼容性
- 專注於設計而不是兼容性問題

這就是為什麼我們剛才修復了 `start` → `flex-start` 的警告，讓代碼更符合最佳實踐！
