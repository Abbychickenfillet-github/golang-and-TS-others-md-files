## 🤔 **為什麼忽略而不是認識？**

### 1. **`styled-jsx` 不是標準 React**
- `jsx` 和 `global` 是 Next.js 的 `styled-jsx` CMS-in-JS 功能
- 它們**不是**標準的 HTML/JSX 屬性
- Standard ESLint React 規則不認識這些自定義屬性

### 2. **你需要安裝額外的插件**
要「認識」這些屬性，需要：
```bash
npm install --save-dev eslint-plugin-styled-jsx
```

然後在 `.eslintrc.json` 中添加：
```json
{
  "plugins": ["react", "jsx-a11y", "prettier", "styled-jsx"],
  "rules": {
    "styled-jsx/no-unused-styles": "error"
  }
}
```

### 3. **為什麼我選擇忽略？**
- **更簡單**：一行配置解決所有問題
- **更快**：不需要安裝新依賴
- **更穩定**：不會有版本兼容性問題
- **功能不缺**：你的 `styled-jsx` 功能完全正常運作

    {/* 這等同於把 props 裡的所有屬性加在這個 Tooltip 上 */}

// {...props} 等同於 color={props.color} size={props.size}
