## 什么是 Negated Condition？

**Negated condition**（否定条件）指使用 `!` 的条件判断，例如：
- `if (!condition)` ❌
- `if (condition === false)` ❌

**为什么建议避免？**
1. 可读性：正向条件更直观（“如果有什么”比“如果没有什么”更清晰）
2. 减少认知负担：正向逻辑更容易理解
3. 代码质量：符合最佳实践

**示例对比：**
```typescript
// ❌ 否定条件
if (!email.trim()) {
  // 错误处理
}

// ✅ 正向条件
const emailTrimmed = email.trim();
if (emailTrimmed) {
  // 正常流程
} else {
  // 错误处理
}
```

## Cursor
`chakra-ui` 裡的 **token** 可以理解成「設計變數」。它把顏色、字體、間距、陰影、動效等設計屬性統一命名與管理，讓你在整個專案中用語意化的 key 來引用，而不是硬寫 `#FF5733` 或 `16px`。好處：

- **一致性**：改 token 值即可同步更新所有使用該 token 的元件。
- **語意清晰**：像 `colors.primary`、`space.4` 這種語意名稱，比魔術數字好維護。
- **支援主題/暗色模式**：token 可以定義 `_light`、`_dark` 不同值，自動在模式切換時更新。
- **可自訂、可擴充**：透過 `defineTokens` 或 `semanticTokens` 自訂品牌顏色、字體、動效等。

若你看到文件範例 `defineTokens({ colors: { primary: { value: "#0FEE0F" }}})`，就是在建立一組 tokens，之後可以在元件裡用 `bg="primary"` 等 API 引用，Chakra 會幫你輸出對應的 CSS 值。

1. **在 theme.ts 定義語意 token**  
   例如：
   ```ts
   'surface.page': { value: { base: '{colors.cream.50}', _dark: '#121214' } },
   'surface.main': { value: { base: '{colors.white}', _dark: '#1c1c21' } },
   'surface.footer': { value: { base: '{colors.coffee.800}', _dark: '#0f0f15' } },
   'text.inverse': { value: { base: '{colors.coffee.900}', _dark: '{colors.cream.50}' } },
   ```
   注意 `_dark` 只在 dark mode 生效。只要 token 有這樣的定義，Chakra 會自動依目前的 color mode 套用對應的顏色。

2. **Layout 使用這些 token key**  
   ```tsx
   <Box minH="100vh" display="flex" flexDirection="column" bg="surface.page">
     <Header />
     <Box as="main" flex={1} py={8} w="100%" bg="surface.main">
       <Container maxW="1200px" w="100%">
         {children}
       </Container>
     </Box>
     <Box as="footer" bg="surface.footer" color="text.inverse" ...>
       ...
     </Box>
   </Box>
   ```

3. **CreateEvent（或其他頁面）也用同樣方式**
