## Cursor
## 1. `for...of` 的運作方式
`for (let x of list)` 會逐一取得 `list` 的每個元素，並把該元素賦值給 `x`。

## 2. `entries()` 回傳的內容
`fruits.entries()` 回傳一個迭代器，每次迭代得到 `[索引, 值]`：
- `[0, "Banana"]`
- `[1, "Orange"]`
- `[2, "Apple"]`
- `[3, "Mango"]`

## 3. `entries()` 不一定要搭配 `for...of`
可用其他方式處理：

```javascript
// 方法1: 使用 for...of (你目前用的)
for (let x of list) {
    console.log(x); // [0, "Banana"], [1, "Orange"]...
}

// 方法2: 使用 next() 方法
console.log(list.next().value); // [0, "Banana"]
console.log(list.next().value); // [1, "Orange"]

// 方法3: 轉成陣列再用 forEach
Array.from(list).forEach(x => {
    console.log(x); // [0, "Banana"], [1, "Orange"]...
});

// 方法4: 使用展開運算子
console.log([...list]); // [[0, "Banana"], [1, "Orange"], [2, "Apple"], [3, "Mango"]]
```

## 4. 為什麼常用 `for...of`
簡潔、易讀，且能直接迭代迭代器。

目前程式碼會顯示：
```
0,Banana
1,Orange
2,Apple
3,Mango
```

因為 `x` 是 `[索引, 值]` 陣列，轉成字串時會變成 `"0,Banana"`。

釐清 `+=` 與 `for...of` 的用法：

## `+=` 運算子的作用
`+=` 是「累加賦值」運算子：
```javascript
text += x + "<br>";
// 等同於
text = text + x + "<br>";
```

## 你的程式碼運作流程
```javascript
let text = " ";  // text 初始值是 " " (一個空格)

for (let x of list){ // x 是 [索引, 值] 陣列
    text += x + "<br>";  // 把 x 和 "<br>" 加到 text 後面
}
```

## 逐步執行過程
1. 初始：`text = " "`
2. 第一次：`x = [0, "Banana"]` → `text = " " + "0,Banana" + "<br>"` = `" 0,Banana<br>"`
3. 第二次：`x = [1, "Orange"]` → `text = " 0,Banana<br>" + "1,Orange" + "<br>"` = `" 0,Banana<br>1,Orange<br>"`
4. 依此類推

## 為什麼用 `text += x` 而不是 `x += text`？
- `text` 是累積結果的字串
- `x` 是每次迭代的 `[索引, 值]`
- 要把每次的 `x` 加到 `text` 後面，所以用 `text += x`

## 如果寫成 `x += text` 會怎樣？
```javascript
// 錯誤寫法
for (let x of list){
    x += text + "<br>";  // 這會嘗試修改 x (陣列)
}
```
這會出錯，因為 `x` 是陣列，不能這樣累加字串。

## 總結
- `text` 是累積結果
- `x` 是每次迭代的資料
- `text += x` 是把 `x` 加到 `text` 後面
- `for...of` 會自動處理迭代，不需要手動累加索引
