## 為什麼 `text-center` 能置中按鈕？

### 1. **按鈕的 `display` 屬性**
```css
/* 按鈕預設是 inline-block 或 inline-flex */
button {
  display: inline-block; /* 或 inline-flex */
}
```

### 2. **`text-center` 的工作原理**
```css
.text-center {
  text-align: center;
}
```

`text-align: center` 不僅影響文字，還會影響**行內元素**（inline elements）和**行內塊元素**（inline-block elements）的水平對齊。

### 3. **實際效果**
- 當按鈕是 `inline-block` 時，它會被當作一個「文字字符」來處理
- `text-align: center` 會將這個「字符」置中
- 就像文字會被置中一樣，按鈕也會被置中

## 更準確的置中方法

雖然 `text-center` 能工作，但更語義化的方法是：

### 方法1：使用 Flexbox
```jsx
<div className="d-flex justify-content-center">
  <Button>按鈕</Button>
</div>
```

### 方法2：使用 Bootstrap 的按鈕組
```jsx
<div className="btn-group" role="group">
  <Button>按鈕1</Button>
  <Button>按鈕2</Button>
</div>
```

### 方法3：使用 `mx-auto`（如果按鈕是 block）
```jsx
<Button className="d-block mx-auto">按鈕</Button>
```

### 2. **為什麼建議保留 `isListening` 檢查？**

#### **防止重複操作**
```javascript
// 沒有檢查的情況
voiceBtn.addEventListener('click', () => {
  recognizer.stop() // 如果已經停止，這會拋出錯誤

voiceBtn.addEventListener('click', () => {

})
```

#### **避免瀏覽器錯誤**
- 如果語音識別已經停止，再次調用 `stop()` 可能會拋出錯誤
- 如果語音識別正在進行，再次調用 `start()` 也可能會失敗

#### **提供更好的用戶體驗**
- 按鈕狀態會正確顯示（「🎤 語音」vs「🎤 聆聽中...」）

```javascript
voiceBtn.addEventListener('click', async () => {

      recognizer.stop()
    } else {
      recognizer.start()
    }
  } catch (err) {
    // 即使出錯也能優雅處理
    console.error('語音識別操作失敗:', err)
    setIsListening(false)
```

Web Speech API 的 `SpeechRecognition` 對象：
- `start()` 方法：如果已經在運行會拋出錯誤
- `stop()` 方法：如果已經停止也會拋出錯誤
- 狀態管理很重要，避免這些錯誤
