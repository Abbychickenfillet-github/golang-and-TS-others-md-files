## Cursor
關於你的問題，**`state` 參數名稱是可以自定義的**！

在 Zustand 的 `set()` 函數中，傳入的更新函數的參數名稱並沒有硬性規定一定要叫做 `state`。你可以使用任何你喜歡的名稱。

讓我展示幾個例子：

```javascript
// 使用 state（常見慣例）
set((state) => ({ todos: [...state.todos, newTodo] }))

// 使用 currentState
set((currentState) => ({ todos: [...currentState.todos, newTodo] }))

// 使用 prevState
set((prevState) => ({ todos: [...prevState.todos, newTodo] }))

// 使用 store
set((store) => ({ todos: [...store.todos, newTodo] }))

// 甚至使用中文（雖然不推薦）
set((當前狀態) => ({ todos: [...當前狀態.todos, newTodo] }))
```

**為什麼通常使用 `state`？**
1. **慣例**：這是 React 和狀態管理庫的常見慣例
2. **語義清晰**：`state` 明確表示這是當前的狀態
3. **一致性**：與其他開發者的代碼保持一致

**其他常見的命名方式：**
- `currentState` - 強調是當前狀態
- `prevState` - 強調是之前的狀態
- `store` - 表示整個 store 物件

所以答案是：**沒有硬性規定，你可以使用任何你覺得合適的名稱**，但建議遵循慣例使用 `state` 以保持代碼的一致性和可讀性。
