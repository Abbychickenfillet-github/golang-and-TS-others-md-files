## 主要差異總結：

**React 18 新寫法 vs 舊寫法：**
```javascript
// 舊寫法 (React 17 及之前)
import ReactDOM from 'react-dom'
ReactDOM.render(<App />, document.getElementById('root'))

// 新寫法 (React 18+)
import { createRoot } from 'react-dom/client'
createRoot(document.getElementById('root')).render(<App />)
```

**與 Next.js 的差異：**
- **Next.js**: 有自己的 `_app.js` 和 `_document.js`，自動處理 React 渲染
- **純 React + Vite**: 需要手動設定入口點，這就是為什麼需要 `createRoot`

**StrictMode 的限制：**
- 只在開發環境生效
- 會讓某些函數執行兩次（如 useEffect、useState 的初始化函數）
- 可能會讓你的 console.log 出現兩次，這是正常的
- 生產環境完全沒有影響

這種寫法在 Vite + React 專案中很常見，比 Next.js 更接近原生 React 的寫法！

## `prefers-color-scheme` 詳細說明：

**這是什麼？**
- CSS 媒體查詢，用來偵測使用者的系統主題偏好
- 不需要 JavaScript，純 CSS 就能實現自動主題切換

**常見用法：**
```css
/* 亮色主題 */
@media (prefers-color-scheme: light) {
  :root {
    background-color: white;
    color: black;
  }
}

/* 深色主題 */
@media (prefers-color-scheme: dark) {
  :root {
    background-color: black;
    color: white;
  }
}
```

**實際效果：**
- 當使用者在 Windows 設定中選擇「淺色」模式時，網站會自動顯示亮色主題
- 當選擇「深色」模式時，網站會自動顯示深色主題
- 完全跟隨系統設定，使用者不需要手動切換

**與 Next.js 的差異：**
- **Next.js**: 通常用 `next-themes` 套件，提供手動切換按鈕
- **純 CSS**: 自動跟隨系統，無法手動控制

**瀏覽器支援：**
- 現代瀏覽器都支援（Chrome 76+, Firefox 67+, Safari 12.1+）
- IE 不支援

這是一個很實用的功能，讓你的網站能自動適應使用者的使用習慣！

## `useRef` 詳細解釋：

**基本概念：**
```javascript
const inputRef = useRef(null);
// 等於：{ current: null }
```

**主要特點：**
1. **不會觸發重新渲染** - 與 `useState` 不同
2. **保存可變值** - 可以隨時修改 `current` 屬性
3. **直接操作 DOM** - 最常見的用法

**常見用法對比：**

**❌ 不用 useRef（需要 state）：**
```javascript
const [inputValue, setInputValue] = useState('');

// 每次輸入都會觸發重新渲染
<input 
  value={inputValue} 
  onChange={(e) => setInputValue(e.target.value)} 
/>
```

**✅ 使用 useRef（不需要 state）：**
```javascript
const inputRef = useRef(null);

// 不會觸發重新渲染，直接操作 DOM
<input ref={inputRef} />
// 取值：inputRef.current.value
// 設值：inputRef.current.value = 'new value'
```

**實際應用場景：**
- 表單提交後清空輸入框
- 自動聚焦到某個元素
- 滾動到特定位置
- 保存計時器 ID
- 保存前一個 state 值

**與 Next.js 的差異：**
- 完全沒有差異！`useRef` 是純 React Hook
- Next.js 中用法一模一樣

這就是為什麼 `useRef` 這麼好用 - 它讓我們可以直接操作 DOM，而不需要透過 state 來控制！

## 箭頭函數返回物件的語法差異：

**你的困惑很合理！讓我用例子說明：**

**❌ 錯誤寫法：**
```javascript
const useTodo = create((set) => {
    todos: [],  // 語法錯誤！JavaScript 不知道這是物件
    addNewTodo: () => {}
});
```

**✅ 正確寫法一（用圓括弧）：**
```javascript
const useTodo = create((set) => ({
    todos: [],  // 圓括弧告訴 JavaScript 這是物件
    addNewTodo: () => {}
}));
```

**✅ 正確寫法二（用大括弧 + return）：**
```javascript
const useTodo = create((set) => {
    return {
        todos: [],  // 明確返回物件
        addNewTodo: () => {}
    };
});
```

## 為什麼會有這個差異？

**JavaScript 的解析規則：**
- `=> {` → JavaScript 認為這是函數體的開始
- `=> ({` → JavaScript 認為這是表達式，會自動返回

**實際對比：**
```javascript
// 方法一：簡潔寫法
const func1 = () => ({ name: 'John' });

// 方法二：傳統寫法
const func2 = () => {
    return { name: 'John' };
};

// 兩者結果完全相同！
```

**在 Zustand 中：**
```javascript
// 常見的 Zustand 寫法
const useStore = create((set) => ({
    count: 0,
    increment: () => set((state) => ({ count: state.count + 1 }))
}));
```

這就是為什麼會看到 `=> ({` 而不是 `=> {` 的原因！圓括弧讓程式碼更簡潔，不需要寫 `return`。

## 詳細解釋表達式 vs 語句：

**1. 表達式 (Expression)：**
```javascript
// 這些都是表達式，會產生值
1 + 2                    // 產生 3
{ name: 'John' }        // 產生物件
function() { return 1 } // 產生函數
```

**2. 語句 (Statement)：**
```javascript
// 這些都是語句，執行動作但不產生值
if (true) { }           // 條件語句
for (let i = 0; i < 3; i++) { }  // 迴圈語句
return 1;               // 返回語句
let x = 1;              // 宣告語句
```

## 箭頭函數的關鍵差異：

**❌ 錯誤理解：**
```javascript
() => { name: 'John' }
// JavaScript 認為這是：
// () => { 
//   name: 'John'  // 這看起來像標籤語句！
// }
```

**✅ 正確寫法：**
```javascript
// 方法一：表達式形式
() => ({ name: 'John' })
// 圓括弧讓 JavaScript 知道這是表達式

// 方法二：語句形式  
() => { return { name: 'John' } }
// 明確的 return 語句
```

## 回答你的問題：

**Q: 為什麼要返回「表達式」？**
A: 因為箭頭函數的設計理念是簡潔，表達式會自動成為返回值

**Q: 大括弧+return也是返回表達式嗎？**
A: 不是！`return` 是語句，但 `return` 後面跟的是表達式：
```javascript
return { name: 'John' }
//      ↑
//   這是表達式
```

**實際對比：**
```javascript
// 兩種寫法結果完全相同
const func1 = () => ({ name: 'John' });           // 表達式形式
const func2 = () => { return { name: 'John' }; }; // 語句形式

console.log(func1()); // { name: 'John' }
console.log(func2()); // { name: 'John' }
```

所以圓括弧的作用就是告訴 JavaScript：「這是一個表達式，不是函數體」！

```javascript
// 第一個箭頭函數：外層函數
() => {
```

**詳細分解：**

## 3. StrictMode 雙重調用實際例子：

```javascript
function MyComponent() {
    console.log('組件渲染'); // StrictMode 下會印兩次！
    
    useEffect(() => {
        console.log('Effect 執行'); // StrictMode 下會印兩次！
        
        // 如果有副作用，會執行兩次
        fetch('/api/data'); // 會發送兩次請求！
    }, []);
    
    return <div>Hello</div>;
}
```

**為什麼要這樣做？**
- 確保你的函數是純函數
- 如果 `useEffect` 有副作用，雙重調用會讓問題更明顯
- 幫助你寫出更穩定的程式碼

**生產環境的差異：**
```javascript
// 開發環境（StrictMode 開啟）
console.log('組件渲染'); // 印兩次
console.log('組件渲染'); // 印兩次

// 生產環境（StrictMode 關閉）
console.log('組件渲染'); // 只印一次
```

這就是為什麼你在開發時可能會看到 console.log 印兩次，但生產環境只會印一次的原因！
