### 4. **送出處理函數 `analyzeSelected`**
```javascript
const analyzeSelected = async () => {
  // 1. 驗證是否有選中記錄
  if (selectedItems.size === 0) {
    setError('請至少選擇一個時間記錄')
    return
  }

  // 2. 組合自定義提示詞和手動輸入
  let combinedPrompt = ''
  if (customPrompt.trim()) {
    combinedPrompt += customPrompt.trim()
  }
  if (manualInput.trim()) {
    if (combinedPrompt) combinedPrompt += '\n\n'
    combinedPrompt += '用戶提供的額外資訊：\n' + manualInput.trim()
  }

  // 3. 如果有組合的提示詞，加入 payload
  if (combinedPrompt) {
    payload.customPrompt = combinedPrompt
  }

  // 4. 發送 API 請求
  const res = await fetch('/api/ai/analyze-activities', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    credentials: 'include',
    body: JSON.stringify(payload),
  })
}
```

### 5. **連動流程**
1. 輸入：使用者填寫 `customPrompt`、`manualInput`
2. 選擇：勾選要分析的記錄，更新 `selectedItems`
3. 送出：點擊按鈕觸發 `analyzeSelected`
4. 組合：將兩個輸入欄位合併為 `combinedPrompt`
5. 發送：連同選中記錄一併送出至 API

### 6. **按鈕禁用條件**
- `loading`：分析中
- `selectedItems.size === 0`：未選記錄
- `!isAuth`：未登入

此設計讓使用者可選記錄並搭配自訂提示詞與手動內容進行分析。

## 2. `toggleSelection` 函數解析

```javascript
const toggleSelection = (id) => {
  setSelectedItems((prev) => {
    const newSet = new Set(prev)  // 複製現有的 Set
    if (newSet.has(id))           // 如果已經選中
      newSet.delete(id)           // 就移除它 (取消選中)
    else                          // 如果沒選中
      newSet.add(id)              // 就加入它 (選中)
    return newSet                 // 回傳新的 Set
  })
}
```

### 步驟說明
1. `const newSet = new Set(prev)`：複製現有 Set，避免直接修改 state
2. `if (newSet.has(id))`：檢查是否已選中
3. `newSet.delete(id)`：已選中則移除
4. `else newSet.add(id)`：未選中則加入
5. `return newSet`：回傳新 Set 更新 state

### 使用範例

## Prisma 語法說明

### **`select` 語法**
```javascript
select: {
  // select: 指定要查詢的欄位，只返回需要的資料，提高效能
  user_id: true,        // 返回用戶ID
  email: true,          // 返回用戶信箱
  current_log_count: true, // 返回計數器
}
```

**作用**：
- 只查詢需要的欄位
- 減少網路傳輸
- 提升查詢效能

### **`_count` 語法**
```javascript
_count: {
  // _count: Prisma 的特殊語法，用於計算關聯記錄的數量
  select: {
    timeLogs: true,   // 計算 timeLogs 關聯的記錄數量
  },
}
```

**作用**：
- 計算關聯記錄數量
- 等同於 `COUNT(*)` 查詢
- 避免手動計算

### **實際應用**
```javascript
// 查詢結果
{
  user_id: 5,
  email: "yoki@gmail.com",
  current_log_count: 6,     // 我們維護的計數器
```

### **為什麼需要 `finally`？**

```javascript
try {
  // 執行程式碼
  console.log('開始執行...')
  // 可能成功或失敗
} catch (error) {
  // 如果出錯，執行這裡
  console.error('出錯了:', error)
} finally {
  // 無論成功或失敗，都會執行這裡
  await prisma.$disconnect()  // 關閉資料庫連線
}
```

### **執行流程**

1. 成功情況：
   ```
   try → finally → 結束
   ```

2. 失敗情況：
   ```
   try → catch → finally → 結束
   ```

### **為什麼要關閉資料庫連線？**

```javascript
// 問題：如果不關閉連線
const prisma = new PrismaClient()
await prisma.user.findMany()  // 執行查詢
// 程式結束，但連線還在記憶體中！
// 造成記憶體洩漏，連線池耗盡
```

```javascript
// 解決：使用 finally 確保關閉
try {
  await prisma.user.findMany()
} finally {
  await prisma.$disconnect()  // 確保連線被關閉
}
```

### **實際例子**

```javascript
// 沒有 finally 的危險情況
async function badExample() {
  const prisma = new PrismaClient()
  
  try {
    await prisma.user.findMany()
    console.log('成功')
    // 如果這裡出錯，連線不會被關閉！
  } catch (error) {
    console.error('失敗')
    // 連線仍然不會被關閉！
  }
  // 連線洩漏！
}

// 使用 finally 的安全情況

  const prisma = new PrismaClient()

    await prisma.user.findMany()
    console.log('成功')
  } catch (error) {
    console.error('失敗')
  } finally {
    await prisma.$disconnect()  // 無論如何都會關閉連線
  }
  // 連線安全關閉！
}
```

### **記憶體洩漏的後果**

- 連線池耗盡
- 記憶體使用量持續上升
- 資料庫連線數達到上限
- 應用程式變慢或崩潰

### **最佳實踐**

```javascript
  } finally {
    // 清理資源
    await prisma.$disconnect()
  }
}
```

## React Compiler vs Next.js 的資料庫連線管理

### **React Compiler 沒有共同管理機制**

React Compiler 是編譯時優化工具，不負責執行時連線管理。

### **React Compiler 的作用**

```javascript
// React Compiler 主要功能：
// 1. 自動 memoization
// 2. 減少不必要的重新渲染
// 3. 優化組件性能
// 4. 編譯時優化

// 例如：自動優化這個組件
function MyComponent({ data }) {
  // React Compiler 會自動 memoize 這個計算
  const expensiveValue = useMemo(() => {
    return data.map(item => item.value * 2)
  }, [data])
  
  return <div>{expensiveValue}</div>
}
```

### **React 應用程式的資料庫連線管理**

#### **純 React 應用程式**
```javascript
// 純 React 應用程式需要自己管理
// server.js (Express 後端)
const { PrismaClient } = require('@prisma/client')
const prisma = new PrismaClient()

// 手動管理連線
process.on('SIGINT', async () => {
  await prisma.$disconnect()  // 必須手動關閉
  process.exit(0)
})

// API 路由
app.get('/api/data', async (req, res) => {
  try {
    const data = await prisma.user.findMany()
    res.json(data)
  } catch (error) {
    res.status(500).json({ error: error.message })
  }
  // 每次請求都可能需要考慮連線管理
})
```

#### **React + Express 架構**
```javascript
// 需要手動管理 Prisma 連線
// lib/prisma.js
import { PrismaClient } from '@prisma/client'

const prisma = new PrismaClient()

// 手動關閉連線
export const disconnect = async () => {
  await prisma.$disconnect()
}

export default prisma

// 在應用程式關閉時
process.on('beforeExit', async () => {
  await disconnect()
})
```

### **Next.js 的優勢**

#### **Next.js 提供完整的後端管理**
```javascript
// lib/prisma.js - Next.js 自動管理
import { PrismaClient } from '@prisma/client'

const globalForPrisma = globalThis
export const prisma = globalForPrisma.prisma || new PrismaClient()

if (process.env.NODE_ENV !== 'production') {
  globalForPrisma.prisma = prisma
}

// Next.js 自動處理：
// 1. 連線池管理
// 2. 進程層級清理
// 3. 熱重載時的連線重用
```

#### **API 路由自動管理**
```javascript
// app/api/timelog/route.js
import prisma from '@/lib/prisma.js'

export async function POST() {
  try {
    const result = await prisma.timeLog.create({...})
    return NextResponse.json({ success: true, data: result })
  } catch (error) {
    return NextResponse.json({ error: error.message }, { status: 500 })
  }
  // 不需要 prisma.$disconnect()
  // Next.js 自動管理連線池
}
```
