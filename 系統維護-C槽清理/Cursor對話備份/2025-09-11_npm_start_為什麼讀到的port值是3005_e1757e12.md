## 📋 `normalizePort` 函數詳細介紹

### 🎯 **函數目的**
`normalizePort` 函數的主要目的是將各種格式的端口值轉換成 Node.js HTTP 伺服器可以使用的有效端口格式。

### 📥 **輸入參數**
- `val`: 可以是字串或數字類型的端口值
- 例如：`"3005"`, `"8080"`, `3005`, `8080`, `"abc"`, `-1`

### 🔄 **處理流程**

#### 1. **數字轉換**
```javascript
var port = parseInt(val, 10)
```
- 使用 `parseInt(val, 10)` 將輸入值轉換為十進制整數
- 如果輸入是字串 `"3005"`，會轉換為數字 `3005`
- 如果輸入是字串 `"abc"`，會轉換為 `NaN`

#### 2. **有效性檢查**
```javascript
if (isNaN(port)) {
  return val  // 返回原始值
}
```
- 檢查轉換後的結果是否為 `NaN`（非數字）
- 如果是 `NaN`，表示輸入值無法轉換為有效數字
- 返回原始輸入值（通常是字串）

#### 3. **端口範圍檢查**
```javascript
if (port >= 0) {
  return port  // 返回數字端口
}
return false  // 返回 false
```
- 檢查端口是否為非負數（≥ 0）
- 有效端口範圍：0-65535
- 如果端口 ≥ 0，返回數字格式的端口
- 如果端口 < 0，返回 `false`

### �� **返回值類型**

| 輸入值 | 返回值 | 類型 | 說明 |
|--------|--------|------|------|
| `"3005"` | `3005` | `number` | 有效端口字串轉數字 |
| `3005` | `3005` | `number` | 有效端口數字保持不變 |
| `"8080"` | `8080` | `number` | 有效端口字串轉數字 |
| `"abc"` | `"abc"` | `string` | 無效字串保持原樣 |
| `-1` | `false` | `boolean` | 負數端口返回 false |
| `""` | `""` | `string` | 空字串保持原樣 |

### 🔧 **實際使用範例**

在你的 `www.js` 中：
```javascript
var port = normalizePort(process.env.PORT || '3005')
app.set('port', port)
```

#### 情況分析：
1. **環境變數存在**：
   - `process.env.PORT = "8080"` → `normalizePort("8080")` → 返回 `8080`
   
2. **環境變數不存在**：
   - `process.env.PORT = undefined` → `normalizePort("3005")` → 返回 `3005`

3. **無效值**：
   - `process.env.PORT = "abc"` → `normalizePort("abc")` → 返回 `"abc"`

### ⚠️ **為什麼需要這個函數？**

1. **類型安全**：確保端口值是可用的格式
2. **錯誤處理**：優雅地處理無效的端口值
3. **兼容性**：支援字串和數字兩種輸入格式
4. **標準化**：將各種格式統一為 Node.js 可接受的格式

### 🚨 **在你的專案中的問題**

根據你之前的問題，`normalizePort` 函數本身是正常的，問題在於：

1. **環境變數未載入**：`.env.production` 文件沒有被正確讀取
2. **預設值使用**：因為 `process.env.PORT` 為 `undefined`，所以使用了預設值 `'3005'`
3. **結果**：`normalizePort('3005')` 返回 `3005`，這就是為什麼你的後端運行在 3005 端口

### 💡 **解決方案**

要讓後端運行在 8080 端口，你需要：
1. 確保 `.env.production` 文件存在且包含 `PORT=8080`
2. 確保環境變數載入機制正常工作
3. 或者直接修改預設值：`process.env.PORT || '8080'`

這個函數是 Express.js 應用程式的標準端口處理方式，確保了端口配置的穩定性和可靠性。

**502 錯誤是什麼：**
- **定義**: 伺服器作為閘道或代理時，從上游伺服器收到無效回應
- **簡單說**: 前端能連到某個服務，但那個服務無法正常回應

1. **後端服務沒有運行**

   ```bash
   # 如果前端和後端都用 8080，會衝突
   前端: localhost:8080 ❌
   後端: localhost:8080 ❌  # 衝突！
   ```

   - 資料庫連接失敗
   - 環境變數載入失敗
