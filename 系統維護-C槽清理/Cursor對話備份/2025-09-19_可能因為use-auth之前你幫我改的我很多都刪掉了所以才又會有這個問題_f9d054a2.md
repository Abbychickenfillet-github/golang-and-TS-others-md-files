## Cursor
很好的問題！這是 Prisma 的命名慣例：

## 📝 Prisma 命名慣例說明

### 1. **模型名稱 (Model Name)**
- 使用**單數**形式：`Step`, `TimeLog`, `User`
- 代表資料表中的**單一記錄**

### 2. **關聯欄位名稱 (Relation Field)**
- 使用**複數**形式：`steps`, `timeLogs`, `users`
- 代表**多個記錄的集合**

### 3. **實際例子**

```prisma
model User {
  user_id    Int        @id @default(autoincrement())
  name       String?
  timeLogs   TimeLog[]  // 複數：一個用戶有多個時間戳
  steps      Step[]     // 複數：一個用戶有多個步驟
}

model TimeLog {
  id     Int    @id @default(autoincrement())
  title  String
  steps  Step[] // 複數：一個時間戳有多個步驟
  user   User   @relation(fields: [userId], references: [user_id])
}

model Step {
  id       Int     @id @default(autoincrement())
  title    String
  timeLog  TimeLog @relation(fields: [timeLogId], references: [id])
  user     User?   @relation(fields: [userId], references: [user_id])
}
```

### 4. **為什麼這樣設計？**

- **語義清晰**：`user.timeLogs` 表示"用戶的時間戳們"
- **直觀理解**：`timeLog.steps` 表示"時間戳的步驟們"
- **程式碼可讀性**：`user.steps` 比 `user.step` 更清楚表達這是多個步驟

### 5. **資料庫層面**
- 資料表名稱通常也是複數：`steps`, `time_logs`, `users`
- 但 Prisma 模型名稱用單數，關聯欄位用複數

這是 Prisma 的標準慣例，讓程式碼更加清晰易懂！ 🎯

### 2. JWT 加密與解密核心 (Core JWT Operations)

#### 函式：`encrypt(payload, exp = '7d')` (加密 JWT Session)

這個非同步函式負責將您的使用者資料 (例如 `userId`、`username` 等) 轉換成一個安全的 JWT 字串。

* **`payload`**: 包含使用者資料的 JavaScript 物件，這是 JWT 的有效負載。
* **`exp`**: Session 的有效期限，預設為 `7d` (7 天)。
* **流程：**
    1.  `new SignJWT(payload)`：開始建立一個新的 JWT，並放入有效負載。
    2.  `.setProtectedHeader({ alg: 'HS256' })`：設定 JWT 的標頭，使用 **HS256** (HMAC-SHA256) 演算法進行簽名。
    3.  `.setIssuedAt()`：設定 JWT 的簽發時間。
    4.  `.setExpirationTime(exp)`：設定 JWT 的過期時間。
    5.  `.sign(encodedKey)`：使用秘密金鑰 (`encodedKey`) 對整個 Token 進行簽名，產生最終的 JWT 字串。

#### 函式：`decrypt(session)` (解密 JWT Session)

這個非同步函式負責驗證收到的 JWT 字串是否有效且未被竄改。

* **`session`**: 從 Cookie 中讀取到的 JWT 字串。
* **流程：**
    1.  `jwtVerify(session, encodedKey, { algorithms: ['HS256'] })`：
        * 使用秘密金鑰驗證簽名。
        * 驗證 Token 是否已經過期。
        * 驗證 Token 的演算法是否正確 (HS256)。
    2.  如果驗證成功，它會返回一個完整的 `result` 物件，其中包含了原始的有效負載 (`result.payload`)。
    3.  如果驗證失敗（例如簽名不匹配或已過期），會進入 `catch` 區塊，並在控制台列印 `'Failed to verify session'`。

---

### 3. Session 管理函式 (Session Management)

#### 函式：`createSession(payload, exp = '7d', cookieName = 'ACCESS_TOKEN')` (建立並設定 Cookie)

這是將 JWT 寫入瀏覽器 Cookie 的主函式。

* **過期時間計算：** 函式首先根據傳入的 `exp` (例如 `'7d'`) 計算出精確的**絕對過期時間** (`expiresAt`)，轉換成毫秒數後，加上當前時間。
    * `switch` 區塊負責將 'd', 'h', 'm', 's' 轉換為對應的毫秒數。
* **加密：** `const session = await encrypt({ payload, expiresAt }, exp)` 建立 JWT。
* **設定 Cookie：**
    * `cookieStore.set(cookieName, session, {...})`：將加密後的 JWT 字串設定為 Cookie 的值。
    * `httpOnly: false`：**注意：** 通常為了安全，`httpOnly` 應設為 `true`，防止前端 JavaScript 讀取。此處設定為 `false` 表示允許前端 JavaScript 訪問此 Cookie。
    * `secure`: 判斷是否為生產環境，若是則設定 `secure` 屬性，確保 Cookie 只在 **HTTPS** 連線中傳送。
    * `expires`: 使用計算出來的精確過期時間。
    * `sameSite: 'lax'`：設定 SameSite 策略，防止 CSRF 攻擊。

#### 函式：`updateSession(cookieName = 'ACCESS_TOKEN')` (更新 Session 有效期)

此函式的目的是在使用者與網站互動時，延長其登入狀態，而**不更動** JWT 本身的內容。

* **流程：**
    1.  從 Cookie 中讀取現有的 `session` JWT。
    2.  對 Session 進行 `decrypt` 驗證 (確保 Session 仍然有效)。
    3.  如果有效，它計算一個新的過期時間 (`expires`，預設為 7 天後)。
    4.  它**重新設定**相同的 `session` 值，但使用這個**新的** `expires` 時間來更新 Cookie 的有效期限。

#### 函式：`deleteSession(cookieName = 'ACCESS_TOKEN')` (刪除 Session)

這是登出操作的標準做法。

* `cookieStore.delete(cookieName)`：直接從瀏覽器中移除指定的 Cookie，使 Session 失效。

## 🔍 JWT 字段解釋

### `iat` (Issued At) 是什麼？

`iat` 是 JWT 的**標準字段**，不是 Google 登入特有的：

- **`iat`**: Issued At - JWT 的**簽發時間**
- **`exp`**: Expiration - JWT 的**過期時間**
- **`userId`**: 我們自定義的**用戶 ID**
- **`expiresAt`**: 我們自定義的**過期時間**（用於 Cookie）

### 📊 從你的 JWT 可以看到：

```json
{
  "payload": {
    "userId": 6,
    "expiresAt": "2025-09-27T15:08:05.309Z",
    "iat": 1758726485,  // 簽發時間 (Unix 時間戳)
    "exp": 1758985685   // 過期時間 (Unix 時間戳)
  }
}
```

### 🤔 為什麼有 `iat`？

這是因為在 `lib/jwt-session.js` 的 `encrypt` 函數中：

```javascript
export async function encrypt(payload, exp = '7d') {
  return new SignJWT(payload)
    .setProtectedHeader({ alg: 'HS256' })
    .setIssuedAt()        // ← 這裡設置了 iat
    .setExpirationTime(exp) // ← 這裡設置了 exp
    .sign(encodedKey)
}
```

- **`.setIssuedAt()`** 自動添加 `iat` 字段
- **`.setExpirationTime()`** 自動添加 `exp` 字段

這是 JWT 的**標準做法**，與 Google 登入無關！

## Next.js 架構對檔案名稱的要求

Next.js 框架對於一些特殊功能（例如 Middleware、Layouts、Pages 等）有嚴格的檔案命名慣例。

### 1. 官方 Middleware 檔案名稱

Next.js 唯一會自動偵測和執行的中介軟體檔案名稱是：

* **`middleware.js`**
* **`middleware.ts`** (如果是 TypeScript)

這個檔案必須放在專案的根目錄（例如 `my-app/middleware.js`）或 `src` 目錄下。

### 2. `middleware-sa-test.js` 的用途

由於您的檔案名稱不匹配官方的 `middleware` 慣例，它在 Next.js 的架構中會被視為一個**普通的 JavaScript 模組**，不會被自動執行。

這個檔案通常是以下用途之一：

1.  **測試檔案 (Test File)**：用於儲存與中介軟體相關的測試程式碼。
2.  **備份或草稿 (Backup/Draft)**：開發者在實作正式的 `middleware.js` 之前，暫時儲存邏輯的地方。
3.  **被引入的工具檔案 (Imported Utility)**：它可能包含一些邏輯，然後被您**手動**從**正式的** `middleware.js` 檔案中 `import` 進來使用。

**總結：**

### 📋 `matcher` 的正則表達式組成

```javascript
export const config = {
```

**語法分解**：
- `/(...)` - 正則表達式開始和結束
- `(?!...)` - **負向先行斷言**（Negative Lookahead）
- `api|_next/static|_next/image|.*\\.png$` - 要排除的路由模式
- `.*` - 匹配任何字符
- `$` - 字串結尾

### 🎯 各部分的含義

| `_next/static` | Next.js 靜態資源 | 避免對靜態檔案執行認證檢查 |
| `_next/image` | Next.js 圖片優化 | 避免對圖片處理執行認證檢查 |
| `.*\\.png$` | 所有 .png 檔案 | 避免對圖片檔案執行認證檢查 |

## X-LINE-Nonce 說明

### 什麼是 X-LINE-Nonce？
`X-LINE-Nonce` 是 LINE Pay API 要求的請求頭，用於防重放攻擊。

### 作用
1. 防重放攻擊：確保請求唯一性
2. 時間戳驗證：配合 `X-LINE-Timestamp` 使用
3. 簽名驗證：參與簽名計算

### 在程式碼中的使用
```javascript
// 在 services/line-pay.service.js 中
const nonce = crypto.randomBytes(16).toString('hex')  // 生成隨機 nonce

const headers = {
  'X-LINE-Nonce': nonce,           // 隨機字串
  'X-LINE-Timestamp': timestamp,   // 時間戳
  'X-LINE-Signature': signature,   // 簽名（包含 nonce）
}
```

### 簽名計算過程
```javascript
// 簽名包含 nonce
const signature = crypto
  .createHmac('sha256', channelSecretKey)
  .update(channelSecretKey + endpoint + body + nonce + timestamp)
  .digest('base64')
```

## EPOCH 說明

### 什麼是 EPOCH？
```sql
EXTRACT(EPOCH FROM (t."end_time" - t."start_time"))/3600 AS "duration_hours"
```

- EPOCH：Unix 時間戳（1970-01-01 00:00:00 UTC 起的秒數）
- 用途：計算時間差
- 計算：`EXTRACT(EPOCH FROM 時間差)` 得到秒數，除以 3600 轉為小時

### 範例
