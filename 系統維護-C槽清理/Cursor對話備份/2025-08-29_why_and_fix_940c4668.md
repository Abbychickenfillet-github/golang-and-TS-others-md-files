- **LF (Line Feed)**：Unix/Linux/macOS 使用的換行符號 `\n`
- **CRLF (Carriage Return + Line Feed)**：Windows 使用的換行符號 `\r\n`

**為什麼會發生**：
- 你的專案可能在不同作業系統間開發
- 或者 Git 配置的行尾符號處理設定不一致

```bash
git config core.autocrlf true
```

### 方法 2：建立 `.gitattributes` 檔案
在專案根目錄建立 `.gitattributes` 檔案：
```
# 設定所有文字檔案使用 LF
* text=auto eol=lf

# 特定檔案類型
*.js text eol=lf
*.jsx text eol=lf
*.ts text eol=lf
*.tsx text eol=lf
*.css text eol=lf
*.scss text eol=lf
*.html text eol=lf
*.json text eol=lf
*.md text eol=lf
```

### 方法 3：重新設定行尾符號
```bash
# 重新檢出所有檔案，統一行尾符號
git add .
git commit -m "Fix line endings"
git rm --cached -r .
git reset --hard
```

1. **這不是程式碼錯誤**，不會影響程式運行
2. **只是 Git 的警告**，不會破壞你的程式碼
3. **建議使用方法 1**，讓 Git 自動處理
4. **如果團隊開發**，建議使用方法 2 統一規範

## �� 總結：

你看到的 Git 警告是行尾符號問題，不是程式碼問題。這很常見，特別是在 Windows 和 Unix 系統間切換時。設定好 Git 配置後，這個警告就會消失，不會再困擾你了！
