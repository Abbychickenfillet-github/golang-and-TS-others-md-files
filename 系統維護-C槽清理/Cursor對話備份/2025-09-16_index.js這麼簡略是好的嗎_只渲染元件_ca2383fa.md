## 🔍 **Switch 語句詳細解釋**

### **1. Switch 語句結構**
```javascript
const getTitleText = (index, title) => {
  switch (index) {           // 根據 index 的值進行判斷
    case 0:                  // 如果 index === 0
      return 'ROG Strix G16'
    case 1:                  // 如果 index === 1
      return 'MSI Cyborg 15'
    case 2:                  // 如果 index === 2
      return 'Acer Predator'
    default:                 // 如果都不符合上述條件
      return title           // 返回原本的 title
  }
}
```

### **2. Index 參數的來源**
```javascript
// 在第 67 行，map 函數會提供 index
{bannerData.map((item, index) => (
  <button
    onClick={() => handleImageChange(index)}  // index 被傳入
    // ...
  >
    <span>{getTitleText(index, item.title)}</span>  // 這裡調用函數
  </button>
))}
```

### **3. Index 的型別和值**
- **型別**: `number` (數字)
- **值**: `0`, `1`, `2` (因為 `bannerData` 有 3 個元素)
- **來源**: `Array.map()` 的第二個參數

## 📊 **執行流程**

```javascript
// bannerData 有 3 個元素
bannerData = [
  { id: 1, title: 'ITEM' },  // index = 0
  { id: 2, title: 'ITEM' },  // index = 1  
  { id: 3, title: 'ITEM' }   // index = 2
]

// 當 map 執行時：
// 第一次: index = 0 → getTitleText(0, 'ITEM') → 返回 'ROG Strix G16'
// 第二次: index = 1 → getTitleText(1, 'ITEM') → 返回 'MSI Cyborg 15'
// 第三次: index = 2 → getTitleText(2, 'ITEM') → 返回 'Acer Predator'
```

## �� **為什麼這樣設計？**

1. **自定義標題** - 原本的 `title` 都是 'ITEM'，但顯示時要顯示具體的產品名稱
2. **硬編碼映射** - 根據陣列索引來決定顯示什麼標題
3. **向後兼容** - 如果 `index` 超出範圍，就使用原本的 `title`

## 🔧 **JavaScript 型別判斷**

JavaScript 是**動態型別**語言，所以：
- **不需要** 事先聲明型別
- **執行時** 才知道 `index` 的型別
- **Switch 使用嚴格相等** (`===`) 進行比較

這就是為什麼 `switch` 能夠正確處理 `index` 參數的原因！

1. **組件銷毀但定時器仍在運行**

3. **瀏覽器無法回收這些記憶體**
4. **長時間運行會讓瀏覽器變慢**

### **什麼是 % 運算子？**

```javascript
// 語法：a % b = a 除以 b 的餘數
7 % 3 = 1    // 7 ÷ 3 = 2 餘 1

(1 + 1) % 3 = 2 % 3 = 2  // 1 → 2  
(2 + 1) % 3 = 3 % 3 = 0  // 2 → 0 (回到開始)
```
