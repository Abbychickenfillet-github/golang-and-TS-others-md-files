## 关于 CSS 权重

**Chakra UI 的样式权重更高**，原因：

1. Chakra UI v3 使用 Emotion，样式通过内联 `style` 属性注入：
   ```jsx
   <Box w="100%" />  // 会渲染成 <div style="width: 100%">
   ```
   内联样式权重为 1000，高于普通 CSS 规则。

2. 原生 CSS 的权重：
   - 标签选择器（如 `body`）：1
   - 类选择器（如 `.my-class`）：10
   - ID 选择器（如 `#root`）：100
   - 内联样式（Chakra UI）：1000
   - `!important`：10000

3. 实际影响：
   - 你的 `body { display: flex; place-items: center; }` 只影响 body 本身
   - Chakra UI 的组件样式是内联的，会覆盖原生 CSS
   - 如果原生 CSS 要覆盖 Chakra UI，需要使用 `!important` 或更具体的选择器

## 关于 `place-items`

在 flex 容器中，`place-items` 是 `align-items` 和 `justify-content` 的简写：
- 垂直居中：`align-items: center`
- 水平居中：`justify-content: center`

但你的 `#root` 有 `width: 100%`，所以内容会撑满，看起来不受影响。

总结：Chakra UI 的权重更高，因为使用内联样式。原生 CSS 主要用于全局样式（如 body、#root），不会与 Chakra UI 冲突。

## Cursor
`colorScheme` 是 Chakra UI 组件（像 `Button`, `Badge`, `Switch` 等）提供的一个 prop，用来指定“主题色系”。它对应 Chakra 主题中 `theme.colors` 的一级 key。

- 内建色系：`gray`, `red`, `orange`, `yellow`, `green`, `teal`, `blue`, `cyan`, `purple`, `pink`, `linkedin`, `facebook`, `messenger`, `whatsapp`, `twitter`, `telegram`, `blackAlpha`, `whiteAlpha`…
- 当你写 `colorScheme="blue"`，Chakra 会根据这个色系和组件的 `variant`，自动设置背景、边框、文字、hover 状态等颜色。
- 不能随便填不在主题里的值；你贴的 `"black"` 不是默认色系，会被当作 `blackAlpha` 处理或直接回退，所以按钮颜色可能怪。
- 如果你想用自定义颜色，需要在 Chakra 主题里扩展 `theme.colors`，然后 `colorScheme` 才能使用那个 key。

简单看法：`colorScheme` 就是告诉 Chakra UI “这颗按钮是 blue 系列/gray 系列…”，它会自动应用那套配色。

`Box` 是 Chakra UI 裡面最基礎、最常用的元件：  
- 它其實就是一個經過 Chakra 包裝的 `<div>`。

<header style="

現在 header 的 `color="white"` 是寫死的，所以不管色系怎麼換，文字都會是白色。如果要根據 color mode 調整，應該把 `color` 改成 `useColorModeValue('gray.800', 'white')` 之類的動態值，或乾脆拿掉 `color` 讓父層繼承。

---

### `colorScheme` vs `color`

- `color`：直接指定文字顏色，就像 CSS 的 `color` 屬性。
- `colorScheme`：告訴 Chakra 使用哪一組“品牌色”來計算背景、邊框、hover 狀態。舉例 `colorScheme="coffee"`，Chakra 會套用 `coffee` 這組 token。`variant="solid"` 時會用 `coffee.500` 當背景、白色文字；`variant="outline"` 則會用 `coffee.500` 當邊框，文字預設同色。

所以如果你同時設定了 `colorScheme="coffee"` 和 `color="white"`，會以 `color` 為主，把文字改成白色，即使 `colorScheme` 想用咖啡色也會被覆蓋。

---

### 建議做法

1. 先確認你是否真的要 light/dark mode。若要，改成：

```tsx
import { useColorMode, useColorModeValue, Button } from '@chakra-ui/react'

const { toggleColorMode } = useColorMode()
const headerBg = useColorModeValue('cream.400', 'coffee.800')
const headerColor = useColorModeValue('gray.800', 'white')
```

2. 把 `color="white"` 改成 `color={headerColor}` 或拿掉，讓它隨 theme 變化。

3. 如果未來要用 `ColorModeButton`，Chakra 自帶 `IconButton` + `useColorMode` 可以自己做一顆切換按鈕；不需要額外套件。

這樣文字顏色就不會“壞掉”。

## Class 的使用

### 1. JavaScript vs TypeScript
- JavaScript（ES6+）也支持 `class`，但 TypeScript 提供了类型支持
- 在 Next.js 中如果不用 TypeScript，通常用函数式写法或普通 JavaScript class

### 2. 为什么这里用 Class？
在 `i18n.service.ts` 中使用 class 的原因：
- 封装状态：`currentLanguage`、`translations` 作为私有属性
- 单例模式：`export const i18nService = new I18nService()` 确保全局只有一个实例
- 方法组织：相关方法集中在一个类中

### 3. 函数式写法（Next.js 常见）

## 什么是“相同的顺序”？

“相同的顺序”指：每次组件渲染时，所有 hooks 必须按完全相同的顺序调用。

### 例子

```typescript
// ✅ 正确：顺序固定
function Component() {
  const [name, setName] = useState('');     // 第 1 个 hook
  const [age, setAge] = useState(0);         // 第 2 个 hook
  const color = useColorModeValue(...);      // 第 3 个 hook
  
  // 每次渲染都是：Hook #1, Hook #2, Hook #3
  // 顺序永远不变！
}
```

```typescript
// ❌ 错误：顺序可能变化
function Component() {
  const [name, setName] = useState('');     // 第 1 个 hook
  
  if (condition) {
    const [age, setAge] = useState(0);      // 第 2 个 hook（有时有）
    return <div>A</div>;
  }
  
  const color = useColorModeValue(...);     // 第 2 个 hook（有时没有）
  // 问题：Hook 数量会变化！
}
```

## 为什么顺序很重要？

React 用数组索引来追踪每个 hook 的状态：

```typescript
// React 内部（简化版）
let hookIndex = 0;
const hooks = []; // 存储所有 hooks 的状态

function useState(initialValue) {
  const currentIndex = hookIndex; // 记住位置
  hookIndex++; // 移动到下一个
  
  if (!hooks[currentIndex]) {
    hooks[currentIndex] = initialValue; // 存储状态
  }
  
  return [hooks[currentIndex], setValue];
}
```

执行流程：

```typescript
// 第 1 次渲染
const [name, setName] = useState('');     // hooks[0] = ''
const [age, setAge] = useState(0);        // hooks[1] = 0

// 第 2 次渲染
const [name, setName] = useState('');     // hooks[0] = '新名字' ✅
const [age, setAge] = useState(0);        // hooks[1] = 0 ✅
```

如果顺序变了：

```typescript
// 第 1 次渲染（condition = true）
const [name, setName] = useState('');     // hooks[0] = name
const [age, setAge] = useState(0);        // hooks[1] = age

// 第 2 次渲染（condition = false）
const [name, setName] = useState('');     // hooks[0] = name ✅
const color = useColorModeValue(...);     // hooks[1] = color ❌
// React 以为这是 age，但实际是 color！
```
