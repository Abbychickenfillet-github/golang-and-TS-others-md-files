PostgreSQL 與 Postgres 命名差異與資料庫發展史

重點整理

1. PostgreSQL 跟 Postgres 是同一個東西嗎？

✅ 是同一個東西

PostgreSQL 是正式官方名稱，Postgres 是社群／開發者最常用的簡稱（官方認可的別名）。

特性 PostgreSQL Postgres
性質 正式官方名稱 常用簡稱／別名
建議場景 正式文件、履歷、論文、合約 口頭、社群討論、技術文章
常見於 套件名 postgresql-server 指令 postgres -V

命名由來：Ingres → Post-gres（Post-Ingres，Ingres 的後繼者）→ Postgres95（1995 加入 SQL）→ PostgreSQL（1996 為強調支援 SQL 標準而正式更名）。PostgreSQL 是 PostgreSQL 全球開發小組的註冊商標。

💡 搜尋小撇步

兩個關鍵字通常可互換，但搜 PostgreSQL 通常能拿到更精確的官方技術細節。

2. Ingres 是什麼

ℹ️ Ingres = PostgreSQL 的「祖先」

Ingres（Interactive Graphics Retrieval System）是 1970 年代早期在 加州大學柏克萊分校（UC Berkeley） 開發的研究型關聯式資料庫，由 Michael Stonebraker 教授 帶領。

  • 它是最早實現關聯式模型的系統之一。
  • 是 PostgreSQL 的直系前身（Postgres = Post-Ingres）。
  • 影響了 Microsoft SQL Server、Sybase 等商業資料庫。
特性 Ingres PostgreSQL
誕生 1973 1986（Postgres 期刊發表）
查詢語言 QUEL（後來才支援 SQL） SQL（標準）
現況 商業／開源併行（Actian） 全球最受歡迎開源資料庫之一
📝 小知識

Ingres 確立的許多技術（B-tree 索引、查詢優化）至今仍深深影響你寫的每一行 SQL。

3. 資料庫發展史簡表（從「森林」到「關聯」）

  1. 前關聯式（1960s）：階層式模型（樹狀）、網狀模型（多對多但極複雜）。缺點是程式碼與資料結構高度耦合，改欄位就要重寫程式。
  2. 轉折點（1970）：IBM 的 E. F. Codd 發表論文，用集合論把資料放進二維表格，用鍵值建立關聯，資料庫不再需要處理物理存取路徑，只專注邏輯結構。
  3. 三大門派（1970s–80s）：IBM System R（發明了 SQL）、Oracle（Larry Ellison 讀了 Codd 論文搶先做商用版）、Ingres（UC Berkeley，開源／PostgreSQL 的發源地）。
  4. SQL 統治（1990s）：MySQL 出現（1995，輕量、Web 2.0 寵兒）；Postgres 加入 SQL → 改名 PostgreSQL。
  5. NoSQL / NewSQL（2000s–2010s）：MongoDB、Redis 放棄部分強一致性換水平擴展；NewSQL/HTAP 想結合 SQL 嚴謹與 NoSQL 擴充。
❝ 歷史的趣味

現在許多最新技術（如向量資料庫、向量檢索）核心仍是在擴充 PostgreSQL 的功能——證明 40 年前 Ingres 奠定的「可擴充性」架構多麼前瞻。

  2. （填空）Postgres 名稱的意思是 Post-Ingres（Ingres 的後繼者）。
  3. （申論題）簡述 E. F. Codd 1970 年的論文為什麼是資料庫發展的轉折點。 答案：他提出用數學集合論處理資料，把資料存進二維表格、以鍵值建立關聯。從此資料庫不必再處理物理存取路徑，開發者只需專注邏輯結構，解決了前關聯式時代「程式碼與資料結構高度耦合、改欄位就要重寫程式」的痛點，催生了 System R、Oracle、Ingres 三大計畫與 SQL。
