CSRF 與 Antiforgery Cookie（.AspNetCore.Antiforgery）

重點整理

  • .AspNetCore.Antiforgery 這支 cookie 的用途是防止跨站請求偽造攻擊（CSRF, Cross-Site Request Forgery），用來驗證表單送出時的安全性。
  • 它跟「帳戶被鎖定」沒有關係，刪掉它也不會解鎖帳戶。帳戶鎖定通常是後端的登入失敗次數 / 安全策略造成的。
  • 正解：系統顯示「用戶帳戶被鎖定」時，應直接聯絡該網站客服處理，而不是自己刪 cookie。

除錯小技巧（這次情境學到的）

  • 送出請求後若 DevTool 的 Network 沒有出現 200 成功回應，代表該動作沒有真的串到 API（前端表單可能根本沒打到後端）。這是判斷「重置密碼到底有沒有生效」的好線索。
  • 查 cookie 用途：F12 → Application → Cookies，看 cookie 的 name 與 domain。看到 Antiforgery、XSRF-TOKEN、csrf 這類字眼，多半就是 CSRF 防護 token，不是身分驗證或鎖定相關。
