GraphQL 概念與 REST 對比

重點整理

[!info] 什麼是 GraphQL？
Facebook 2012 年開發、2015 年開源的一種 API 查詢語言（Query Language），同時也是執行查詢的伺服器端 Runtime。是傳統 REST API 的替代方案，讓前端「要什麼資料，就給什麼資料」，不多也不少。

比喻

REST 像點固定套餐（Endpoint），想吃某道小菜得點整套；GraphQL 像自助餐，拿點單（Query）勾你要的菜，引擎精準打包成 JSON 回給你。

解決的兩大痛點

  • 過度撈取（Over-fetching）：只想顯示「姓名」卻被迫下載含生日、地址、訂單等 50 個欄位，浪費流量。
  • 撈取不足（Under-fetching）／多次請求：要顯示文章列表＋作者姓名＋最新留言，REST 可能得先 /posts、再 /users/1、再 /posts/1/comments，發多次 HTTP 請求。

REST vs GraphQL 對比

特性 REST API GraphQL
進入點 Endpoints 多個（/users, /posts, /comments） 單一（通常只有 /graphql）
資料決定權 後端定義回傳結構 前端決定要哪些欄位
回傳資料量 固定，易 Over-fetching 精準，要什麼給什麼
強型別系統 需額外工具（如 Swagger） 原生內建 Schema（Schema-driven）
