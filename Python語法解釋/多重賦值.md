# Python 多重賦值(Multiple Assignment / 解包)

## 一句話說明

一行**同時指定多個變數**:左邊放多個變數名,右邊放對應數量的值,Python 會**一一配對**。

```python
a, b = 1, 2      # a = 1, b = 2
```

等於分開寫:
```python
a = 1
b = 2
```

---

## 背後原理:tuple 解包(unpacking)

右邊的 `1, 2` 其實是一個 **tuple**(序對),左邊把它**拆開**分給每個變數:

```python
a, b = 1, 2          # 右邊 (1, 2) 被拆開 → a=1, b=2
texts, tools = [], []  # 右邊 ([], []) 被拆開 → texts=[], tools=[]
```

> ⚠️ 左右**數量要對等**,否則報錯:
> ```python
> a, b = 1, 2, 3   # ValueError: too many values to unpack
> a, b, c = 1, 2   # ValueError: not enough values to unpack
> ```

---

## 常見用法

### 1. 同時開好幾個變數
```python
texts, tools = [], []     # 兩個各自獨立的空 list
x = y = z = 0             # (這是另一種:三個都指向同一個 0)
```

### 2. 交換兩個變數的值(經典!不用暫存變數)
```python
a, b = b, a               # 直接對調
```

### 3. 接收函式回傳的多個值
```python
def session_bounds(path):
    ...
    return first, last     # 回傳兩個值(其實是回傳一個 tuple)

first_ts, last_ts = session_bounds(path)   # 一次接住兩個
```

### 4. 搭配 for 迴圈解包(超常用)
```python
# enumerate 每次給 (索引, 元素),用兩個變數接住
for i, sec in enumerate(sections):
    ...

# 每個元素本身是 (name, args) 這種序對,也直接拆開
for name, args in section["tools"]:
    ...
```

### 5. 星號 `*` 收集剩下的
```python
first, *rest = [1, 2, 3, 4]   # first=1, rest=[2, 3, 4]
```

---

## ⚠️ 陷阱:`a, b = [], []` vs `a = b = []`

```python
# 寫法一:兩個「各自獨立」的 list ✅
a, b = [], []
a.append(1)
print(a, b)        # [1] []   ← b 不受影響

# 寫法二:a 和 b 指向「同一個」list ⚠️
a = b = []
a.append(1)
print(a, b)        # [1] [1]  ← 改 a 連 b 一起變!
```

要兩個獨立的容器,就用 `a, b = [], []`。

---

## 本專案範例(claude_log.py)

```python
texts, tools = [], []                      # 開兩個獨立的空 list
first_ts, last_ts = session_bounds(path)   # 接住函式回傳的兩個值
for i, sec in enumerate(sections):         # for 迴圈解包 (索引, 元素)
for name, args in section["tools"]:        # 拆開每個 (name, args)
```

## 快速總結

| 問題 | 答案 |
|------|------|
| `a, b = 1, 2` 是什麼? | 一行同時設兩個變數(a=1, b=2) |
| 原理? | 右邊組成 tuple,左邊解包拆給每個變數 |
| 左右數量不一樣? | 直接 `ValueError` |
| 交換變數? | `a, b = b, a` |
| `a, b = [], []` vs `a = b = []`? | 前者各自獨立;後者共用同一個物件(會互相影響) |
