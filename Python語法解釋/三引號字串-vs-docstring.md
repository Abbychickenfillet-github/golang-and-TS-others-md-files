# 三引號字串 `"""..."""` vs docstring

> 來源：abby-notes-rag 專案 `scripts/ask.py` 的 `SYSTEM_PROMPT = """..."""`
> 疑問：`SYSTEM_PROMPT = """..."""` 是 docstring 嗎？還是只是 Python 字串寫法？

## 結論先講

**那不是 docstring，是「三引號字串字面值」，純粹是 Python 的字串寫法。**

關鍵：**三引號是「字串的寫法」，docstring 是「字串放對位置的特殊用途」，兩件事。**

---

## 1. 三引號 `"""..."""` 是什麼？

只是 Python 建立字串的一種語法，特點：

- 可以**跨多行**（普通 `"..."` / `'...'` 不能直接換行）
- 裡面可以直接放單引號、雙引號，不用跳脫

```python
SYSTEM_PROMPT = """你是 Abby 的助理。
可以換行。
還能放 "雙引號" 不用跳脫。"""

type(SYSTEM_PROMPT)   # <class 'str'>  ← 就是普通字串
```

> 單引號版本 `'''...'''` 效果完全一樣，三個單引號也可以跨多行。

---

## 2. docstring 是「位置」決定的，不是「引號」決定的

一個字串要變成 docstring，**唯一條件**是：
它是 **模組 / 函式 / 類別 的「第一個敘述 (statement)」**。
這時 Python 會自動把它存進 `__doc__`，給 `help()`、IDE 提示、文件工具讀。

```python
def foo():
    "這是 docstring"          # ← 函式第一個敘述 → 是 docstring（單引號、單行也算！）
    x = "這不是 docstring"     # ← 有指派 → 普通字串

class Bar:
    """這是 class docstring"""  # ← class 第一個敘述 → 是 docstring

# 檔案最上面、單獨一行：
"""這是模組 docstring"""        # ← module 第一個敘述 → 是 docstring
```

重點：用**單引號、單行**的字串放在第一個敘述位置，**也是 docstring**；
用**三引號**但**指派給變數**，就**不是** docstring。

---

## 3. 套回原本的程式碼

```python
SYSTEM_PROMPT = """你是 Abby 的個人筆記問答助理。..."""
```

因為它**被指派給變數 `SYSTEM_PROMPT`**，不是「裸的第一個敘述」，
所以**不是 docstring**，就是一個普通的多行字串 → 之後被當 prompt 文字丟給 LLM。

---

## 對照表

| 寫法 | 引號種類 | 是 docstring 嗎？ | 為什麼 |
|------|----------|-------------------|--------|
| `"""abc"""` 放檔案/函式最上面，單獨一行 | 三引號 | ✅ 是 | 在「第一個敘述」位置 |
| `x = """abc"""` | 三引號 | ❌ 否 | 指派給變數了 |
| `"abc"` 放在函式最上面 | 普通引號 | ✅ 是 | 位置對就算 |
| `SYSTEM_PROMPT = """..."""` | 三引號 | ❌ 否 | 指派給變數，只是多行字串 |

---

## 一句話總結

**三引號 = 字串寫法；docstring = 字串放在「第一個敘述」位置的特殊用途。**
兩者常一起出現（docstring 多半用三引號方便跨行寫），但本質不同：
判斷是不是 docstring 看「位置」，不是看「幾個引號」。

## 延伸
- 三引號字串若**單獨**出現在程式中間（沒指派、也不在第一敘述位置），
  會被當成一個「沒用到的字串運算式」，有人拿來當「區塊註解」用，
  但它其實還是會被建立成 str（跟 `#` 註解不同，`#` 完全不產生物件）。
