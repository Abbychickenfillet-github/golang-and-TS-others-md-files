# dict 刪除方法：`del` vs `.pop()`（為什麼 `del` 不能配 `.get()`）

## 重點先講

```python
del inventory["sku-001"]       # ✅ 中括號 → 刪除
del inventory.get("sku-001")   # ❌ SyntaxError: cannot delete function call
```

`del` 只能配**中括號 `[]`**，不能配**方法呼叫 `.get()`**。

---

## 心智模型：dict 在哪一層？

很多人會把 dict 的刪除想成「前端刪除」或「資料庫刪除」，其實 **dict 是後端程式記憶體裡的一張表**，三層要分清楚：

```text
[ 瀏覽器 UI / React state ]   ← 前端記憶體，reload 就沒
        ↕ API
[ 後端程式 / Python dict ]    ← 後端記憶體，程式結束就沒
        ↕ SQL
[ 資料庫 / MySQL ]            ← 磁碟上，永久保存
```

### dict 操作 vs SQL DELETE 對照

```python
# Python dict（記憶體）
del inventory["sku-001"]
inventory.pop("sku-001")
inventory.pop("sku-001", None)
```

```sql
-- SQL（資料庫）
DELETE FROM inventory WHERE sku = 'sku-001';
DELETE FROM inventory WHERE sku = 'sku-001' RETURNING *;   -- 對應 .pop()
```

對應後端 ORM（SQLModel，FutureSign 寫法）：

```python
# 真的刪資料庫（永久）
booth = session.get(Booth, booth_id)
session.delete(booth)
session.commit()

# 順手清快取（只在這次程式執行內生效）
booth_cache.pop(booth_id, None)
```

### 三層的「破壞力」對照

| | 前端 state | dict | 資料庫 |
|--|----------|------|--------|
| 存在哪 | 瀏覽器 RAM | 後端 RAM | 磁碟 |
| 程式 / 頁面結束 | **消失** | **消失** | 還在 |
| 跨使用者 | 一個分頁一份 | 一個 process 一份 | 大家共用 |
| 刪錯了能不能救 | reload 就回來 | 重啟程式就回來 | 真的回不來（要 backup） |

### 比喻

| 動作 | 比喻 |
|------|------|
| 前端 `setState` 移除 item | 腦中便條紙劃掉（沒人看得到） |
| dict 的 `del` / `.pop()` | 桌上便利貼撕一張（明天可能就被清掉） |
| `session.delete()` + `commit()` | 從檔案櫃抽出正式文件丟掉（其他人也看不到） |

dict 操作的影響介於前端 state 跟資料庫之間 —— 比前端嚴重（影響整個後端 process），比資料庫輕（重啟就回到原狀）。

### 實務：常常兩個一起做

```python
# 刪資料庫之後，記得清快取，否則下次查還會拿到舊資料
session.delete(booth)
session.commit()
booth_cache.pop(booth.id, None)
```

## 為什麼？`del` 操作的是「位置」不是「值」

```python
inventory["sku-001"]   # 中括號 = 指向 dict 裡的「那個位置」
inventory.get("sku-001")  # .get() = 函式呼叫，回傳「那個位置裡的影本（值）」
```

`del` 是「把某個位置清掉」，所以：

- `del inventory["sku-001"]` —— 清掉 dict 裡 `sku-001` 那個格子 ✅
- `del inventory.get("sku-001")` —— 試圖清掉「`.get()` 回傳的值」，但值是影本、不是位置，Python 直接 `SyntaxError` ❌

類比：你可以叫人「把抽屜裡的東西丟掉」（位置），但不能叫人「把這份影印的內容丟掉」（值的影本沒意義）。

---

## 對照：取值 vs 刪除的「安全版」設計

Python 的 dict 提供一組對稱的 API：

| 動作 | 嚴格版（不存在會炸） | 安全版（不存在不炸） |
|------|--------------------|--------------------|
| 取值 | `d["key"]` | `d.get("key", 預設)` |
| 刪除 | `del d["key"]` | `d.pop("key", 預設)` |
| 檢查 | （沒有嚴格版） | `"key" in d` |

`.get()` 是「安全取值」，`.pop()` 是「安全刪除」，是一對的設計。

## 三種刪除寫法

### 1. `del`（最常見）

```python
inventory = {"sku-001": "攤位 A"}

del inventory["sku-001"]    # ✅ 刪除
print(inventory)            # {}

del inventory["sku-999"]    # ❌ KeyError: 'sku-999'
```

**特點**：簡單直接，但 key 不存在會炸。

### 2. `.pop(key)` — 刪除 + 回傳被刪的值

```python
inventory = {"sku-001": "攤位 A"}

removed = inventory.pop("sku-001")
print(removed)      # 攤位 A
print(inventory)    # {}

inventory.pop("sku-999")   # ❌ KeyError: 'sku-999'
```

**特點**：刪除的同時把值還給你。如果你還要用那個值，這寫法比 `del` 方便。

### 3. `.pop(key, 預設值)` — 安全刪除

```python
inventory = {"sku-001": "攤位 A"}

removed = inventory.pop("sku-999", None)
print(removed)      # None（不存在，回傳預設）
print(inventory)    # {"sku-001": "攤位 A"}（沒改變）
```

**特點**：key 不存在時回傳預設值，**不會炸**。最安全的寫法。

## 實戰：什麼時候用哪個？

### 情境 1：確定 key 存在 → 用 `del`

```python
# 已經檢查過 / 邏輯保證存在
if "sku-001" in inventory:
    del inventory["sku-001"]
```

### 情境 2：要拿被刪的值 → 用 `.pop()`

```python
# 把商品從庫存搬到「已售出」
sold_item = inventory.pop("sku-001")
sold_list.append(sold_item)
```

### 情境 3：不確定 key 是否存在 → 用 `.pop(key, None)`

```python
# 嘗試清掉某個 key，不管它原本在不在
inventory.pop("sku-999", None)   # 不存在也不炸
```

### 反例：用 try/except 包 del（不推薦）

```python
# ❌ 多餘
try:
    del inventory["sku-999"]
except KeyError:
    pass

# ✅ 一行解決
inventory.pop("sku-999", None)
```

## `.pop()` 跟 `list.pop()` 不一樣

容易混淆：

```python
# dict 的 pop：用 key
d = {"a": 1, "b": 2}
d.pop("a")          # 用 key
d.pop("a", None)    # 可帶預設值

# list 的 pop：用 index
lst = [10, 20, 30]
lst.pop()           # 預設刪最後一個 → 30
lst.pop(0)          # 刪 index 0 → 10
lst.pop(99)         # IndexError（不能帶預設值）
```

兩個方法都叫 `pop`，但：
- dict 用 **key** + 可帶**預設值**
- list 用 **index** + **不能**帶預設值

---

## 還有一個：`.popitem()`

刪除並回傳「最後插入的」一對 key-value（Python 3.7+ 保證插入順序）：

```python
inventory = {"sku-001": "A", "sku-002": "B"}

key, val = inventory.popitem()
print(key, val)     # sku-002 B（最後一個被刪）
print(inventory)    # {"sku-001": "A"}
```

實務用途：把 dict 當 stack（後進先出）用，比較少見。

---

## 總結

```python
# 刪除一筆
del d["key"]               # 嚴格，不存在會炸
d.pop("key")               # 嚴格，不存在會炸，但會回傳值
d.pop("key", None)         # 寬鬆，不存在回傳預設

# 清空整個 dict
d.clear()                  # → {}

# 刪除整個 dict 變數
del d                      # 連變數都不見了
```

| 想做的事 | 用法 |
|---------|------|
| 確定刪掉某個 key | `del d["key"]` |
| 刪掉並拿回值 | `d.pop("key")` |
| 不確定 key 在不在，安全刪除 | `d.pop("key", None)` |
| 清空但保留 dict | `d.clear()` |
| 連 dict 都不要了 | `del d` |
