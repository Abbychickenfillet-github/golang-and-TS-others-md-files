# Python list 方法總整理（含可變性 / 是否複製 / 回傳值）

## 核心觀念：list 是 mutable（可變）

```python
lst = [1, 2, 3]
lst.append(4)        # ← 直接動到原 lst，不會回傳新 list
print(lst)           # [1, 2, 3, 4]
```

**list 不是 immutable**，所有 `.append`、`.pop`、`.sort`、`.remove` 都會**原地修改**。
想要 immutable 版本，要用 [`tuple`](#10-list-vs-tuple-immutable-對照)。

---

## 1. 兩大分類

| 分類 | 行為 | 回傳什麼 | 例子 |
|------|------|---------|------|
| **Mutating（原地修改）** | 直接動到原 list | 多半回傳 `None` | `.append`、`.sort`、`.reverse` |
| **Non-mutating（不動原 list）** | 不動原 list | 回傳新值或新 list | `.copy`、`sorted()`、`reversed()` |
| **Pop 系列（特例）** | 動到原 list | 回傳被刪掉的值 | `.pop` |

**Python 慣例：mutating 方法多半回傳 `None`，提醒你「我已經動到原本的東西了」。**

---

## 2. Mutating 方法（原地修改，動到原 list）

### `.append(x)` — 加一個元素到尾端

```python
lst = [1, 2]
result = lst.append(3)
print(lst)       # [1, 2, 3]    ← 原 lst 改了
print(result)    # None         ← 沒回傳新 list
```

### `.extend(iterable)` — 加多個元素到尾端

```python
lst = [1, 2]
lst.extend([3, 4])
print(lst)       # [1, 2, 3, 4]

# 跟 + 的差別
lst = [1, 2]
new = lst + [3, 4]   # 不動原 lst，回傳新 list
print(lst)           # [1, 2]
print(new)           # [1, 2, 3, 4]
```

### `.insert(i, x)` — 插入到指定 index

```python
lst = [1, 2, 4]
lst.insert(2, 3)     # 在 index 2 的位置插入 3
print(lst)           # [1, 2, 3, 4]
```

### `.remove(x)` — 刪除第一個出現的 x

```python
lst = [1, 2, 3, 2]
lst.remove(2)        # 只刪第一個 2
print(lst)           # [1, 3, 2]

lst.remove(99)       # ❌ ValueError（找不到）
```

### `.pop([i])` — 刪除並回傳指定 index（預設最後一個）

```python
lst = [10, 20, 30]
last = lst.pop()     # 預設刪最後一個
print(last)          # 30           ← 回傳被刪的值
print(lst)           # [10, 20]     ← 原 lst 改了

first = lst.pop(0)   # 刪 index 0
print(first)         # 10
print(lst)           # [20]

lst.pop(99)          # ❌ IndexError（list 的 pop 不能帶預設值）
```

`list.pop` 跟 [`dict.pop`](../dict/刪除方法.md) **不同**：
- `list.pop(i)` 用 **index**，沒有預設值參數
- `dict.pop(k, default)` 用 **key**，可帶預設值

### `.clear()` — 清空

```python
lst = [1, 2, 3]
lst.clear()
print(lst)       # []
```

### `.sort()` — 原地排序

```python
lst = [3, 1, 2]
result = lst.sort()
print(lst)       # [1, 2, 3]    ← 原 lst 改了
print(result)    # None

# 倒序
lst.sort(reverse=True)         # [3, 2, 1]

# 自訂排序
lst = ["abc", "a", "ab"]
lst.sort(key=len)              # ["a", "ab", "abc"]
```

### `.reverse()` — 原地反轉

```python
lst = [1, 2, 3]
lst.reverse()
print(lst)       # [3, 2, 1]
```

## 3. Non-mutating 方法（不動原 list）

### `.copy()` — 淺拷貝（shallow copy）

```python
lst = [1, 2, 3]
new = lst.copy()
new.append(4)
print(lst)       # [1, 2, 3]    ← 原 lst 沒動
print(new)       # [1, 2, 3, 4]
```

注意是**淺拷貝**：巢狀的 list 還是共用記憶體。詳見 [§7 別名陷阱與深淺拷貝](#7-別名陷阱與深淺拷貝)。

### `.index(x)` — 找某個值的 index

```python
lst = ["a", "b", "c", "b"]
lst.index("b")       # 1（第一個出現的位置）
lst.index("z")       # ❌ ValueError
```

### `.count(x)` — 計算某個值出現幾次

```python
lst = [1, 2, 2, 3, 2]
lst.count(2)         # 3
```

## 4. 內建函式（不是 list 方法，但常配 list 用）

### `sorted(lst)` — 排序**回傳新 list**

```python
lst = [3, 1, 2]
new = sorted(lst)
print(lst)       # [3, 1, 2]    ← 原 lst 沒動
print(new)       # [1, 2, 3]
```

**`lst.sort()` vs `sorted(lst)`**：

| 寫法 | 動到原 lst？ | 回傳 |
|------|------------|------|
| `lst.sort()` | ✅ 動 | `None` |
| `sorted(lst)` | ❌ 不動 | 新 list |

要保留原資料就用 `sorted()`。

### `reversed(lst)` — 反轉，回傳 iterator（不是 list）

```python
lst = [1, 2, 3]
rev = reversed(lst)
print(rev)              # <list_reverseiterator object>
print(list(rev))        # [3, 2, 1]    ← 要 list() 才看得到內容
print(lst)              # [1, 2, 3]    ← 原 lst 沒動
```

跟 `.reverse()` 對照：

| 寫法 | 動到原 lst？ | 回傳 |
|------|------------|------|
| `lst.reverse()` | ✅ 動 | `None` |
| `reversed(lst)` | ❌ 不動 | iterator（要 `list()` 轉） |

### 其他常用

```python
len([1, 2, 3])          # 3
sum([1, 2, 3])          # 6
max([1, 2, 3])          # 3
min([1, 2, 3])          # 1
any([0, 0, 1])          # True（有任何 truthy）
all([1, 1, 1])          # True（全部 truthy）
list("abc")             # ['a', 'b', 'c']（轉成 list）
```

## 5. 切片（slicing）—— 預設不動原 list

```python
lst = [1, 2, 3, 4, 5]

# 取切片：回傳新 list
lst[1:3]                # [2, 3]
lst[:]                  # [1, 2, 3, 4, 5]   ← 整個複製（淺拷貝）
lst[::-1]               # [5, 4, 3, 2, 1]   ← 反轉新 list

print(lst)              # [1, 2, 3, 4, 5]   ← 原 lst 沒動
```

### 切片賦值（少見、但會動原 list）

```python
lst = [1, 2, 3, 4, 5]
lst[1:3] = [20, 30, 40]      # ✅ 動原 lst
print(lst)                   # [1, 20, 30, 40, 4, 5]
```

## 6. Mutating 方法為什麼回傳 `None`？

這是 Python 故意的設計，避免你寫出**看起來像有回傳值、實際沒有**的程式碼：

```python
# ❌ 常見錯誤
new_list = [3, 1, 2].sort()    # 你以為 new_list 是排好的 [1, 2, 3]
print(new_list)                # None！

# ✅ 正確
lst = [3, 1, 2]
lst.sort()                     # 動原 lst
print(lst)                     # [1, 2, 3]

# 或
lst = [3, 1, 2]
new_list = sorted(lst)         # 用 sorted() 才有回傳
```

**口訣：方法呼叫**改了原本的東西 → 不要期待它還給你新值（會回傳 `None`）。

JS 的設計反過來：

```javascript
// JS 的 .sort() 既動原陣列又回傳同個陣列
const arr = [3, 1, 2];
const sorted = arr.sort();
console.log(arr);      // [1, 2, 3]
console.log(sorted);   // [1, 2, 3]（同一個物件）
```

Python 故意只給一種行為，避免你以為 sorted 是新的、結果改一個動到另一個。

## 7. 別名陷阱與深淺拷貝

### 別名（不是複製）

```python
a = [1, 2, 3]
b = a              # ⚠️ 這不是複製！b 跟 a 指向同一個 list
b.append(4)
print(a)           # [1, 2, 3, 4]    ← a 也變了
print(b)           # [1, 2, 3, 4]
```

`=` 只是讓兩個名字指向同一個物件。要真的複製要用：

```python
b = a.copy()           # 淺拷貝
b = a[:]               # 切片整個 → 淺拷貝
b = list(a)            # 用 list() 轉 → 淺拷貝
```

### 淺拷貝（shallow copy）

```python
a = [[1, 2], [3, 4]]
b = a.copy()           # 淺拷貝
b.append([5, 6])       # b 多一筆，a 沒變
print(a)               # [[1, 2], [3, 4]]
print(b)               # [[1, 2], [3, 4], [5, 6]]

# 但巢狀的內層還是共用！
b[0].append(99)
print(a)               # [[1, 2, 99], [3, 4]]   ← a 的內層也變了
print(b)               # [[1, 2, 99], [3, 4], [5, 6]]
```

### 深拷貝（deep copy）

```python
import copy

a = [[1, 2], [3, 4]]
b = copy.deepcopy(a)   # 連巢狀都複製
b[0].append(99)
print(a)               # [[1, 2], [3, 4]]      ← a 不變
print(b)               # [[1, 2, 99], [3, 4]]
```

實務上 90% 用淺拷貝就夠了，巢狀結構又想完全獨立才用 `deepcopy`。

## 8. 記憶體空間：list 在記憶體裡長怎樣

### 變數名 vs 記憶體物件

```python
lst = [1, 2, 3]
#  ↑       ↑
# 名字   實際的 list 物件（住在記憶體某個位址）
```

`lst` 只是個**名牌**指向記憶體裡的 list。可以用 `id()` 看實際位址：

```python
lst = [1, 2, 3]
print(id(lst))    # 例如 140234567890432（每次跑可能不同）
```

### `=` 賦值：兩個名牌指同一塊記憶體

```python
a = [1, 2, 3]
b = a              # ⚠️ b 不是新 list，是同一個 list 的「第二張名牌」

print(id(a) == id(b))   # True   ← 同一塊記憶體

b.append(4)             # 動到那塊記憶體
print(a)                # [1, 2, 3, 4]   ← a 也「變了」（其實是同一塊）
```

圖示：

```text
記憶體位址 0x100：
┌──────────────┐
│ [1, 2, 3, 4] │  ← 真正的 list 物件
└──────────────┘
       ↑  ↑
       a  b      ← 兩張名牌都指向這裡
```

### `.copy()` / `[:]` / `list()` ：真的開新記憶體

```python
a = [1, 2, 3]
b = a.copy()

print(id(a) == id(b))   # False   ← 不同位址
b.append(4)
print(a)                # [1, 2, 3]      ← a 沒變
print(b)                # [1, 2, 3, 4]
```

圖示：

```text
記憶體位址 0x100：        記憶體位址 0x200：
┌──────────────┐          ┌──────────────┐
│  [1, 2, 3]   │          │ [1, 2, 3, 4] │
└──────────────┘          └──────────────┘
       ↑                          ↑
       a                          b
```

### Mutating 方法 = 改原本那塊記憶體

```python
lst = [3, 1, 2]
print(id(lst))    # 0x100

lst.sort()        # mutating
print(id(lst))    # 0x100   ← 位址沒變，只是內容變了
print(lst)        # [1, 2, 3]
```

### Non-mutating 方法 / 建新 list = 新記憶體

```python
lst = [3, 1, 2]
print(id(lst))           # 0x100

new = sorted(lst)        # 回傳新 list
print(id(new))           # 0x200   ← 新位址
print(id(lst))           # 0x100   ← 原 lst 還在原位

new2 = lst + [4]         # + 也是建新 list
print(id(new2))          # 0x300   ← 又一個新位址
```

### 淺拷貝的記憶體真相

淺拷貝會新開**外層**記憶體，但內層巢狀結構**還是共用**：

```python
a = [[1, 2], [3, 4]]
b = a.copy()

print(id(a) == id(b))         # False  ← 外層 list 不同位址
print(id(a[0]) == id(b[0]))   # True   ← 內層 [1, 2] 同一塊！

b[0].append(99)               # 改內層
print(a)                      # [[1, 2, 99], [3, 4]]   ← a 也變
```

圖示：

```text
            外層
a → 0x100 ┌─────────┐
          │ [0x500, │
          │  0x600] │
          └─────────┘
              ↓ 共用

b → 0x200 ┌─────────┐
          │ [0x500, │   ← 內層位址一樣
          │  0x600] │
          └─────────┘
                ↓
            0x500: [1, 2, 99]   ← 兩邊都指到這
            0x600: [3, 4]
```

要連內層都獨立要用 `copy.deepcopy(a)`。

### Mutable 物件的記憶體成本

list 為了支援 `.append()` 等動態擴容，記憶體配置會**預留多餘空間**：

```python
import sys

lst = []
print(sys.getsizeof(lst))    # 例如 56 bytes（即使空的）

lst.append(1)
print(sys.getsizeof(lst))    # 88 bytes（一口氣多配空間，下次 append 不用重新配）

lst.extend([2, 3, 4, 5])
print(sys.getsizeof(lst))    # 88（還在預留空間內，沒擴容）

# 超過容量時 Python 會「整批搬到更大的位置」
for i in range(100):
    lst.append(i)
print(sys.getsizeof(lst))    # 例如 920
```

對照 tuple（immutable）：

```python
t = (1, 2, 3)
print(sys.getsizeof(t))      # 64 bytes（剛好夠用，沒預留）
```

**list 比 tuple 佔記憶體多**，因為它需要預留擴容空間。

### 函式參數傳遞：傳的是「名牌」不是「複製」

```python
def add_one(lst):
    lst.append(1)            # 動到外面那塊記憶體

original = [1, 2, 3]
add_one(original)
print(original)              # [1, 2, 3, 1]   ← 真的被改了
```

Python **物件是用 reference 傳遞的**。傳 list 進函式不是給「影本」，是給「同一張名牌」。

想保護原資料：

```python
def add_one_safe(lst):
    new = lst.copy()         # 先拷貝
    new.append(1)
    return new

original = [1, 2, 3]
result = add_one_safe(original)
print(original)              # [1, 2, 3]      ← 沒變
print(result)                # [1, 2, 3, 1]
```

### TL;DR 記憶體版

| 動作 | 是否新開記憶體 | 例 |
|------|--------------|---|
| `b = a` | ❌ 共用同一塊 | 別名 |
| `b = a.copy()` / `a[:]` / `list(a)` | ✅ 外層新塊（淺拷貝） | 大部分情況夠用 |
| `b = copy.deepcopy(a)` | ✅ 內外全部新塊 | 巢狀要完全獨立 |
| `lst.append(x)` 等 mutating | ❌ 改原塊 | 位址不變 |
| `sorted(lst)` / `lst + lst2` | ✅ 新塊 | 不動原 lst |
| 傳進函式 | ❌ 共用 | 函式裡 `.append` 會動到外面 |

| 方法 | 動原 list？ | 回傳 | 找不到 |
|------|-----------|------|--------|
| `.append(x)` | ✅ | `None` | — |
| `.extend(iter)` | ✅ | `None` | — |
| `.insert(i, x)` | ✅ | `None` | — |
| `.remove(x)` | ✅ | `None` | ValueError |
| `.pop()` / `.pop(i)` | ✅ | 被刪的值 | IndexError |
| `.clear()` | ✅ | `None` | — |
| `.sort()` | ✅ | `None` | — |
| `.reverse()` | ✅ | `None` | — |
| `.copy()` | ❌ | 新 list（淺拷貝） | — |
| `.index(x)` | ❌ | int | ValueError |
| `.count(x)` | ❌ | int | 回傳 0 |
| `sorted(lst)` | ❌ | 新 list | — |
| `reversed(lst)` | ❌ | iterator | — |
| `lst[i:j]` | ❌ | 新 list | — |
| `lst + lst2` | ❌ | 新 list | — |
| `len(lst)` | ❌ | int | — |

## 10. list vs tuple（immutable 對照）

| 特性 | `list` | `tuple` |
|------|--------|---------|
| 可變性 | mutable（可變） | **immutable**（不可變） |
| 字面語法 | `[1, 2, 3]` | `(1, 2, 3)` |
| 加元素 | `.append()` 等 | ❌ 不能改，只能重建新 tuple |
| 改元素 | `lst[0] = ...` ✅ | `t[0] = ...` ❌ TypeError |
| 當 dict key | ❌ 不能（unhashable） | ✅ 可以（hashable） |
| 效能 | 略慢、佔記憶體多 | 略快、佔記憶體少 |
| 用途 | 動態集合 | 固定資料（座標、回傳多值） |

```python
# tuple 不能改
t = (1, 2, 3)
t[0] = 99           # ❌ TypeError: 'tuple' object does not support item assignment
t.append(4)         # ❌ AttributeError

# 想「改」tuple 只能建新的
t = t + (4,)        # ✅ (1, 2, 3, 4)
```

詳見 [Dict&Tuple&Class.md](../Dict&Tuple&Class.md)。

## 11. 跟 dict 的對照

| 動作 | list | dict |
|------|------|------|
| 加東西 | `lst.append(x)` | `d[key] = value` |
| 取東西 | `lst[i]` 或 `lst[i:j]` | `d[key]` 或 `d.get(key)` |
| 刪除 | `lst.pop(i)` / `del lst[i]` | `d.pop(key, default)` / `del d[key]` |
| 查存在 | `x in lst`（O(n)） | `key in d`（O(1)） |
| 長度 | `len(lst)` | `len(d)` |
| 排序 | `.sort()` 或 `sorted(lst)` | `sorted(d.items(), key=...)` |

list 的 `in` 是 O(n) 線性掃描，dict 的 `in` 是 O(1) 雜湊查詢。資料多的時候差很多。

## 12. 實戰建議

```python
# ✅ 加東西到尾端 → append
lst.append(x)

# ✅ 合併兩個 list → extend 或 +
a.extend(b)        # 動原 a
c = a + b          # 不動原 a

# ✅ 移除最後一筆 → pop
last = lst.pop()

# ✅ 想保留原資料的排序 → 用 sorted()，不用 .sort()
new = sorted(lst)

# ⚠️ 想複製 list → 用 .copy() 或 list(lst)，不要直接 =
new = lst.copy()
```
