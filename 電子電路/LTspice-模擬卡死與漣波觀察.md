# LTspice 模擬卡死(fs/s)與漣波觀察

## 重點整理

情境:全波整流 + 電容濾波 + Op-Amp(LM741)/電晶體線性穩壓電路,模擬卡在 4.7%,Simulation Speed 掉到 1.8 fs/s,波形只有一條平滑上升直線、看不到漣波。

### 不是 CPU 性能問題

LTspice 用**不固定步長(Variable Time Step)**:波形平緩時大步前進;遇到「無法收斂的狀態」(例如 Op-Amp 沒接電源導致內部電晶體模型異常)會把步長無限縮小 ns → ps → fs。掉到 fs 等級後再強的 CPU 也跑不完。可開 `View → Spice Error Log`(Ctrl+L)看 `Iteration limit reached` / damping factor 警告找出卡死元件。

### 為什麼看不到漣波

- 輸入 60 Hz,週期 T = 1/60 ≈ 16.67 ms;模擬只推進到 2.3 ms,還在電容線性充電的「開機階段」。
- 漣波要等電容開始放電,模擬時間至少跑到 20–30 ms 之後才會出現。

### 常見卡死原因(本例)

1. **Op-Amp 電源腳懸空/接錯**:LM741 的 Pin 7(V+)/Pin 4(V−)必須接正確直流電源;負電源腳接到交流訊號源會讓內部模型崩潰、矩陣無法收斂。
2. **回授接反(正回授)**:主動穩壓要把輸出經 R2/R3 分壓接到**反相端(−)**,與 Zener 基準(接**非反相端(+)**)比較;接反會變正回授,電壓衝到飽和後卡死。
3. **Zener(1N750, 4.7V)方向**:陰極(橫線端)要朝上;接反變順向導通(~0.7V)。
4. **變壓器耦合語法**:`k1 l1 l2 l3 1` 三線圈寫法部分版本解析不良,建議分開寫 `K1 L1 L2 0.99`、`K2 L1 L3 0.99`(100% 理想耦合在開機瞬間易產生奇異點)。

### 除錯方法(抓錯法)

先 Stop,把右半邊(Q1、Op-Amp 穩壓)剪掉,只模擬「變壓器 + 二極體整流 + C1 + 負載電阻」;能順利跑完 50 ms 就會看到漣波,再逐步接回穩壓電路定位問題。

另一個觀念:「分壓」只決定輸出電壓,**消除漣波靠的是負回授的主動穩壓機制**,分壓本身不會消除漣波。
