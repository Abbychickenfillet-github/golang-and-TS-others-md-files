# 我的電腦語音輸入 / 語音辨識工具盤點

> 調查日期：2026-06-07
> 平台：Windows 11 Home

這份筆記記錄電腦上跟「語音輸入 / 語音辨識」相關的所有東西，以及推薦的使用順序。

---

## TL;DR 結論

- **日常想用語音打字 → 直接按 `Win + H`**（Windows 內建，零安裝、全域可用、每個 App 都能用）。
- 進階免手控制 / 語音寫程式 → **Talon**（已安裝但尚未初始化）。
- Python 環境**沒有**裝任何語音辨識套件（Whisper / Vosk 等），目前不需要。

---

## ⭐ 首選：Windows 內建語音輸入（Win + H）

**為什麼優先推薦它**：完全內建、全域、不挑專案、不用處理「全域 vs 專案安裝」「下載模型」這類套件常見的麻煩。

### 用法
1. 把游標點進任何可以打字的地方（Word、瀏覽器、聊天框、程式碼編輯器都行）。
2. 按 **`Win + H`** → 跳出語音輸入工具列。
3. 開始講話，文字就會直接打進游標位置。
4. 可以講標點符號：「逗號」「句號」「問號」「換行」。

### 支援的語言
- 系統已安裝 **繁體中文語音套件**：`MicrosoftWindows.Speech.zh-TW`
- 所以中文聽寫可以直接用。若辨識語言不對，到「設定 → 時間與語言 → 語音」確認語音語言。

### 小技巧
- 第一次用可能要在「設定 → 隱私權與安全性 → 語音」開啟「線上語音辨識」效果較好。
- 標點自動插入：設定裡可開「自動標點符號」。

## 🟦 進階：Talon（已安裝，未設定）

| 項目 | 內容 |
|------|------|
| 名稱 | Talon (Talon Voice LLC) |
| 版本 | 0.1.0.0 |
| 設定資料夾 | `C:\Users\User\AppData\Roaming\talon`（目前是空的 → 還沒初始化過） |
| 用途 | 專業級語音控制 + 語音輸入，常用於 hands-free coding（用嘴巴寫程式、操作游標、按快捷鍵） |

**現況**：裝了但 `talon` 設定資料夾是空的，代表第一次啟動流程（下載語音引擎、建立 user 設定）還沒跑過。Talon 比 Win+H 強大但學習曲線陡（要學語法指令、通常會搭配 community / knausj 設定包）。

**建議**：先用 Win+H 滿足日常需求；真的需要免手寫程式再回頭設定 Talon。

## 🪟 系統內建的其他語音元件（非主動安裝，列出備查）

| 元件 | 說明 | 跟語音輸入的關係 |
|------|------|------------------|
| `MicrosoftWindows.Speech.zh-TW` | Windows 繁中語音套件 | ✅ 支撐 Win+H 的中文辨識 |
| `Microsoft.XboxSpeechToTextOverlay` | Xbox 遊戲語音轉字幕疊層 | ❌ 遊戲聊天用，無關 |
| `AcerIncorporated.AcerPurifiedVoiceConsoleR` | Acer 筆電麥克風 AI 降噪 | ⚠️ 音訊「收音優化」，不是辨識，但會讓 Win+H 收音更乾淨 |

## ❌ 沒有安裝的（確認過不存在）

- 商業語音軟體：Dragon NaturallySpeaking、Nuance、Otter 等 → 無
- Python (3.13) 語音辨識套件：Whisper、Vosk、DeepSpeech、SpeechRecognition、wav2vec → **皆無**
- 目前無任何語音相關程式在背景執行

> 備註：若之後想玩 Whisper 之類的，那是「離線語音轉文字」的程式庫，屬於開發用途，跟日常打字的 Win+H 是不同層次的東西。
