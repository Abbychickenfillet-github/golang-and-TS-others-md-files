Deno：現代 JavaScript / TypeScript 執行環境

重點整理

Deno 是現代的 JavaScript / TypeScript / WebAssembly 執行環境（Runtime），可視為 Node.js 的現代升級版，用 Rust 打造、更安全更快、對 TS 極友好。

[!info] 有趣的身世
Deno 創始人 Ryan Dahl 同時是 Node.js 原作者。他在 2018 演講「我對 Node.js 感到遺憾的十件事」指出 Node 在安全性、模組管理（巨大的 node_modules）、非同步支援上的設計缺陷，於是打造 Deno 來解決。

核心特色：

  • 原生支援 TypeScript & JSX：免設定 Babel/tsc 即可直接執行 TS；內部用 SWC（Speedy Web Compiler）大幅加快啟動與編譯。
  • 預設安全（Secure by Default）：除非明確加參數（--allow-net、--allow-read 等），否則無法存取網路、檔案系統、環境變數，執行未知程式碼更安心。
  • 去中心化模組管理：拋棄 npm / package.json，沒有 node_modules，透過標準 URL 直接 import 模組。
  • 內建工具鏈：自帶格式化/Linter（deno lint）、測試執行器、打包工具，省去配置一堆前端工具。
  • 相容 Web API：盡量貼近瀏覽器標準（fetch、window 等可直接用），讓前後端撰寫邏輯一致。
