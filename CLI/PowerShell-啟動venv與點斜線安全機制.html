PowerShell 啟動 venv:為什麼一定要加 .\(命令劫持防護)

重點整理

症狀:venv 沒啟動成功

在 PowerShell 直接輸入 Activate.ps1 會跳紅字錯誤 ObjectNotFound(找不到物件),代表 venv 沒有成功啟動。

正解:加上 .\

在 PowerShell 執行「目前目錄下」的檔案時,不能只打檔名,必須在前面加 .\:

# 錯誤:會 ObjectNotFound
Activate.ps1

# 正確
.\Activate.ps1
# 常見完整路徑
.\venv\Scripts\Activate.ps1

[!tip] PowerShell 出錯時,下方 Suggestion [3,General] 通常已經貼心提示正確寫法(就是叫你加 .\)。

為什麼人就在該目錄,PowerShell 還故意「裝看不到」?

這是「避免命令劫持(Command Hijacking)」的安全機制:PowerShell 預設不把「目前目錄」放進搜尋路徑(PATH)。

原因:如果預設可直接執行當前目錄的檔案,當你下載了一個惡意專案,裡面藏著偽裝成常用指令的惡意腳本(例如 cd.ps1、ls.ps1),你習慣性打 ls 想看檔案時,就會誤觸惡意程式。強制要求 .\ 等於要你「明確指定:我就是要跑這個目錄裡的這支檔案」,避免被劫持。

[!note] 這跟 Unix/Linux 一樣:在 bash 跑當前目錄腳本也要 ./script.sh,道理相同(當前目錄不在 PATH 是刻意的安全設計)。
