# 重定向 (>) vs 管道 (|) vs PowerShell 差異

## 核心概念

### 管道 `|` (Pipe)
**作用**：把**前一個命令的輸出**作為**下一個命令的輸入**

```bash
命令1 | 命令2
```

- 資料在**記憶體中**傳遞（不寫入硬碟）
- 可以無限串接：`cmd1 | cmd2 | cmd3 | cmd4`
- 資料**邊產生邊處理**（串流處理）

### 重定向 `>` (Redirection)
**作用**：把**命令的輸出**存入**檔案**

```bash
命令 > 檔名
```

- 資料寫入**硬碟**
- 會**覆蓋**現有檔案（如果檔案已存在）
- 產生**持久化**的結果

---

## 實際案例對比

### 場景 1：搜尋並查看結果

**使用管道 `|`**（推薦）：
```bash
grep "error" log.txt | less
```
- ✅ 直接查看搜尋結果
- ✅ 不產生額外檔案
- ✅ 記憶體效率高

**使用重定向 `>`**：
```bash
grep "error" log.txt > result.txt
cat result.txt
```
- ❌ 產生暫存檔
- ❌ 需要兩步操作
- ⚠️ 佔用硬碟空間

---

### 場景 2：儲存處理結果

**使用重定向 `>`**（推薦）：
```bash
grep "error" log.txt | sort | uniq > errors_summary.txt
```
- ✅ 儲存最終結果
- ✅ 可以稍後查看
- ✅ 可以分享給他人

**只用管道 `|`**：
```bash
grep "error" log.txt | sort | uniq | less
```
- ⚠️ 只能當場查看
- ❌ 結果不會儲存

---

## 使用時機

### 用 `|` 管道的情況：

1. **中間處理**：資料需要經過多個步驟
   ```bash
   cat file.txt | grep "user" | sort | head -10
   ```

2. **即時查看**：不需要儲存結果
   ```bash
   docker logs container | grep "error"
   ```

3. **計數統計**：只要最終數字
   ```bash
   ls -l | wc -l
   ```

### 用 `>` 重定向的情況：

1. **儲存報告**：需要持久化結果
   ```bash
   mysqldump database > backup.sql
   ```

2. **產生設定檔**：
   ```bash
   echo "port=3306" > config.ini
   ```

3. **匯出資料**：
   ```bash
   docker ps -a > container_list.txt
   ```

## Bash vs PowerShell 的差異

### 1️⃣ 重定向編碼問題

#### Bash (Linux/Mac)
```bash
# UTF-8 編碼（標準）
mysqldump database > backup.sql
```
- ✅ 預設 UTF-8 編碼
- ✅ 相容性好

#### PowerShell (Windows)
```powershell
# UTF-16 編碼（問題！）
mysqldump database > backup.sql
```
- ❌ 預設 UTF-16 編碼
- ❌ 會導致檔案編碼錯誤
- ⚠️ 檔案大小會變成兩倍

**PowerShell 解決方案**：
```powershell
# 方案 1: 使用 Out-File 指定編碼
mysqldump database | Out-File -Encoding UTF8 backup.sql

# 方案 2: 使用命令自帶的輸出參數
mysqldump database --result-file=backup.sql

# 方案 3: 使用 Docker volume 掛載（避免 PowerShell 重定向）
docker run -v C:\sql:/backup mysql mysqldump --result-file=/backup/dump.sql
```

---

### 2️⃣ 管道傳遞的資料型別

#### Bash
```bash
ls -l | grep "txt"
```
- 📝 傳遞**純文字**（字串）
- 每一行就是一個字串

#### PowerShell
```powershell
Get-ChildItem | Where-Object {$_.Extension -eq ".txt"}
```
- 🎯 傳遞 **.NET 物件**
- 每個檔案是一個物件，包含屬性和方法

**範例：**
```powershell
# Bash 風格（文字處理）
ls | Select-String ".txt"

# PowerShell 風格（物件處理）
Get-ChildItem | Where-Object {$_.Extension -eq ".txt"}
```

---

### 3️⃣ 實際案例對比

#### 案例：匯出資料庫

**Bash (正確)**：
```bash
mysqldump -h host -u user -p'pass' database > dump.sql
```
✅ 檔案編碼：UTF-8
✅ 檔案大小：正常

**PowerShell (錯誤)**：
```powershell
mysqldump -h host -u user -p'pass' database > dump.sql
```
❌ 檔案編碼：UTF-16
❌ 檔案內容：每個字元之間有空格
❌ 檔案大小：變成兩倍

**PowerShell (正確方式 1)**：
```powershell
mysqldump -h host -u user -p'pass' database | Out-File -Encoding UTF8 dump.sql
```
✅ 指定 UTF-8 編碼

**PowerShell (正確方式 2 - 推薦)**：
```powershell
docker run --rm -v ${PWD}/sql:/backup mysql:8.0 mysqldump -h host -u user -p'pass' --result-file=/backup/dump.sql database
```
✅ 避開 PowerShell 重定向
✅ 使用 mysqldump 自帶的檔案輸出

## 組合使用

### 管道 + 重定向（常見模式）

```bash
# 處理資料，然後儲存結果
grep "error" log.txt | sort | uniq > error_summary.txt

# 多步驟處理
cat data.csv | grep "2024" | cut -d',' -f1,3 | sort > result.csv

# 統計並儲存
find . -name "*.js" | wc -l > js_file_count.txt
```

**執行順序**：
```
檔案 → grep 過濾 → sort 排序 → uniq 去重 → 存入檔案
     ↑ pipe    ↑ pipe     ↑ pipe    ↑ redirect
```

## 其他重定向符號

| 符號 | 名稱 | 作用 | 範例 |
|------|------|------|------|
| `>` | 輸出重定向（覆蓋） | 覆蓋寫入檔案 | `echo "hello" > file.txt` |
| `>>` | 輸出重定向（附加） | 附加到檔案尾 | `echo "world" >> file.txt` |
| `<` | 輸入重定向 | 從檔案讀取輸入 | `mysql < script.sql` |
| `2>` | 錯誤重定向 | 只重定向錯誤 | `command 2> error.log` |
| `2>&1` | 錯誤合併到標準輸出 | 錯誤和輸出合併 | `command > all.log 2>&1` |
| `&>` | 全部重定向（Bash） | 輸出+錯誤一起重定向 | `command &> all.log` |

## 實際案例：mysqldump 匯出

### 問題場景

```powershell
# ❌ 錯誤：使用 PowerShell 重定向
PS> docker run --rm mysql:8.0 mysqldump -h host -u user -p'pass' database > dump.sql

# 結果：
# - 檔案編碼：UTF-16
# - 檔案內容：C R E A T E   T A B L E（每個字元間有空格）
# - 檔案大小：從 50KB 變成 200KB
# - 中文註解：變成亂碼 ????
```

### 解決方案

**方案 1：使用 Docker volume**（推薦）
```powershell
docker run --rm -v C:\coding\template\sql:/backup mysql:8.0 mysqldump \
  -h host -u user -p'pass' \
  --result-file=/backup/dump.sql \
  database
```
- ✅ 避開 PowerShell 重定向
- ✅ 檔案直接在 Docker 容器內產生（UTF-8）
- ✅ 無編碼問題

**方案 2：使用 Out-File**
```powershell
docker run --rm mysql:8.0 mysqldump -h host -u user -p'pass' database |
  Out-File -Encoding UTF8 dump.sql
```
- ⚠️ 需要記得加 `-Encoding UTF8`
- ⚠️ 容易忘記導致錯誤

**方案 3：使用 Git Bash**
```bash
# 在 Git Bash 中執行（推薦）
docker run --rm mysql:8.0 mysqldump -h host -u user -p'pass' database > dump.sql
```
- ✅ Bash 預設 UTF-8
- ✅ 無需額外參數

## 記憶口訣

### 管道 `|`
> **「左邊輸出，右邊接收，資料在記憶體中流動」**

### 重定向 `>`
> **「命令輸出，存入檔案，資料寫進硬碟裡」**

### PowerShell 重定向
> **「PowerShell 重定向有陷阱，UTF-16 編碼會出錯」**
> **「要嘛用 Out-File 指定 UTF8，要嘛用 Docker volume 掛載」**

---

## 總結表格

| 符號 | 用途 | 資料流向 | 持久化 | Bash | PowerShell |
|------|------|----------|--------|------|-----------|
| `\|` | 管道 | 命令 → 命令 | ❌ 否 | ✅ UTF-8 文字 | ⚠️ .NET 物件 |
| `>` | 重定向 | 命令 → 檔案 | ✅ 是 | ✅ UTF-8 | ❌ UTF-16 |
| `>>` | 附加 | 命令 → 檔案 | ✅ 是 | ✅ UTF-8 | ❌ UTF-16 |

## 實戰技巧

### ✅ 推薦做法

```bash
# 中間處理用管道，最終結果用重定向
cat log.txt | grep "error" | sort | uniq > error_list.txt

# 多個處理步驟串聯
ls *.sql | xargs wc -l | sort -n > file_sizes.txt
```

### ❌ 避免做法

```bash
# 不要過度使用暫存檔
grep "error" log.txt > temp1.txt
sort temp1.txt > temp2.txt
uniq temp2.txt > final.txt
rm temp1.txt temp2.txt

# 應該用管道一次完成
grep "error" log.txt | sort | uniq > final.txt
```

## 作者筆記

**日期**: 2026-01-24

**今天學到**：
1. `|` 是在記憶體中傳資料，`>` 是寫檔案
2. PowerShell 的 `>` 會用 UTF-16 編碼，導致檔案變大和亂碼
3. 解決方案：用 Docker volume 或 `Out-File -Encoding UTF8`
4. 管道可以無限串聯：`cmd1 | cmd2 | cmd3 | cmd4`

**常見錯誤**：
- 在 PowerShell 用 `>` 匯出資料庫 → 檔案編碼錯誤
- 忘記管道可以串聯 → 產生很多暫存檔
