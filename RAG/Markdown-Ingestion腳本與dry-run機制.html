RAG Markdown Ingestion 腳本(ingest.py)與 --dry-run 機制

重點整理

這支腳本在做什麼

ingest.py 是把 Markdown 檔案匯入 pgvector 的進入點(entry point),供 RAG 存放 embedding。

執行模式(命令列旗標):

旗標 行為
(預設) 增量匯入(incremental):只處理有變動的檔案
--full 重新匯入全部檔案(re-ingest everything)
--dry-run 預演:只回報「會更新哪些檔案」,但不實際改動資料庫

這些旗標哪來的?argparse

argparse 是 Python 標準函式庫(內建,不用安裝),負責處理命令列參數。--dry-run、--full 這些選項就是在這支 ingest.py 的 main() 函式裡,用 argparse 自己定義出來的,不是來自其他外部套件。

增量匯入與 --dry-run 的核心:雜湊比對

腳本判斷檔案有沒有變動的方法是比對「目前檔案的 hash」與「資料庫中已存的 hash」:

  • 兩者相同 → 檔案沒變 → 跳過(skip)。
  • 兩者不同 → 檔案有更新 → 刪除舊 chunks、插入新 chunks。

當 args.dry_run 為真時,腳本只「回報」哪些檔案會被更新,但刻意略過資料庫修改步驟(不刪除、不插入 chunks),所以不會對 pgvector 做任何實際變更。

[!tip] dry-run 的價值:在真的動資料庫前,先安全地預覽「這次會改到哪些檔案」,避免誤刪/誤改 embedding。增量 + hash 比對則讓重複執行很便宜(沒變的檔案不重做)。
