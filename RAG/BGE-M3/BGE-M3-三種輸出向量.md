# BGE-M3 與它的三種輸出向量（Dense / Sparse / Multi-Vector）

> **這份筆記回答**：
> 1. BGE-M3 是什麼？為什麼叫「M3」？
> 2. 它說的「把文本轉成三種數學向量」到底是哪三種？
> 3. 每一種怎麼算「像不像」？跟我已經會的**餘弦相似度 / 歐式距離**怎麼接上？
> 4. 我的 `abby-notes-rag` 專案實際用到哪一種？
>
> **建立日期**：2026-05-21
> **前置知識**：[[餘弦相似度與pgvector]]（這份假設你已經懂「一段文字→一個向量→用餘弦比方向」）

---

## 0. 先立一個錨點：你已經會的那一招

你熟的流程是這樣（傳統 embedding）：

```
一段文字  ──模型──►  一個向量（例如 1024 個小數）  ──餘弦相似度──►  比方向像不像
```

**一段文字 = 一個向量**，然後用餘弦比。這是你已經會的。

BGE-M3 特別的地方只有一句話：

> **同一段文字，它一次吐出「三種不同形態」的東西**，每一種用不同方式比相似度。

所以這份筆記不是要你學三個新模型，而是看懂「同一段文字的三種長相」。你會的餘弦相似度，在三種裡面都還會用到——只是組裝方式不一樣。

---

## 1. BGE-M3 是什麼？M3 = 三個 Multi

**BGE-M3** 是北京智源（BAAI）2024 年開源的 embedding 模型，底層是 XLM-RoBERTa。名字裡的 **M3** 代表它三個「Multi（多）」：

| M | 全名 | 白話 |
|---|------|------|
| **Multi-Functionality**（多功能）| 一個模型同時做三種檢索 | ← **本篇主題：Dense / Sparse / Multi-Vector** |
| **Multi-Linguality**（多語言）| 支援 100+ 種語言 | 中英混查不用換模型 |
| **Multi-Granularity**（多粒度）| 短句到 8192 tokens 長文都吃 | 長 chunk 也不爆 |

你問的「三種數學向量」就是第一個 M（Multi-Functionality）。其他兩個 M 是「為什麼它適合你的中文筆記庫」的理由。

---

## 2. 三種向量總覽（先看全貌再逐個拆）

| | **Dense 稠密** | **Sparse 稀疏** | **Multi-Vector 多向量** |
|---|---|---|---|
| 一段文字輸出 | **1 個** 向量 | **1 組** 詞→權重 | **N 個** 向量（每個 token 一個）|
| 形狀 | 1024 維、格格有值 | ≈25 萬維、只有幾格有值 | N × 1024（N=token 數）|
| 抓住什麼 | 整段「意思」 | 精確「字詞」 | 逐詞細節對應 |
| 怎麼比相似度 | 餘弦相似度（你會的）| 共同詞的權重相乘加總 | 逐詞餘弦 + max + 加總 |
| 別名 | semantic / embedding | lexical weights | ColBERT / late interaction |
| 成本（存+算）| 低 | 中 | 高 |

下面三章逐個拆。

---

## 3. Dense 稠密向量（就是你最熟的那種）

### 長相

一段文字 → **一個 1024 維向量**，每一格都有值（小數）：

```
"Docker 啟動失敗" → [0.012, -0.034, 0.088, 0.005, ..., -0.021]
                     └────────── 1024 個格子，幾乎沒有 0 ──────────┘
```

**為什麼叫「稠密 dense」**：1024 格幾乎**塞滿**非零的數字，密密麻麻，所以叫稠密。

### 怎麼比相似度 → 餘弦（你已經會）

兩段文字各得一個向量，用**餘弦相似度比方向**，就是 [[餘弦相似度與pgvector]] 那一套，`<=>` 算的就是這個。

### 順帶連到你熟的「歐式距離」

bge-m3 的 dense 向量會做正規化（長度=1）。對單位向量，**歐式距離和餘弦其實是同一件事的兩種寫法**：

```
‖a − b‖²  =  2 − 2·cos(a, b)
              ↑
        餘弦越大（越像）⟺ 歐式距離越小
```

所以在正規化向量上，用 `<=>`（餘弦）排序、還是 `<->`（歐式 L2）排序，**名次完全一樣**。你不用糾結選哪個。

### Dense 的脾氣

- ✅ 抓「**整段意思**」：「狗無法啟動」vs「Docker container 起不來」會被判為相近，**換句話說、同義詞也抓得到**。
- ❌ 對「**精確關鍵字**」較弱：型號 `bge-m3`、罕見專有名詞、某個錯誤碼，意思被「平均」進整段語意裡，可能被糊掉。

→ 這個缺點,正好由下一種（Sparse）來補。

---

## 4. Sparse 稀疏向量 / 詞彙權重（Lexical Weights）

### 長相

不是壓成一個方向向量，而是對輸入裡**每個出現的詞，算一個「重要度權重」**：

```
"pgvector 怎麼建 HNSW 索引"
        │
        ▼
{ "pgvector": 0.92,  "HNSW": 0.88,  "索引": 0.61,  "建": 0.20,  "怎麼": 0.05 }
   └──────── 重要的詞權重高，虛詞權重低 ────────┘
```

把它想成一個**超高維向量**：維度 = 模型的詞彙表大小（約 **25 萬**個詞），但**只有句子裡出現的那幾個詞是非零，其餘 24 萬多格全是 0**。

**為什麼叫「稀疏 sparse」**：25 萬格裡 99.99% 是 0，零零落落，所以叫稀疏。跟 Dense 正好相反：

```
Dense 稠密：  [0.01, -0.03, 0.08, 0.05, 0.02, ...]   1024 格，格格有值
Sparse 稀疏： [0, 0, 0, ..., 0.92, 0, ..., 0.88, 0, ...]  25萬格，只有幾格有值
```

### 怎麼比相似度 → 不是餘弦！是「共同詞的權重相乘加總」

兩段文字，**只看它們共同出現的詞**，把各自權重相乘再全部加起來：

```
查詢: { pgvector: 0.92,  HNSW: 0.88,  索引: 0.61 }
文件: { pgvector: 0.80,  索引: 0.50,  向量: 0.70 }
            │共同              │共同
            ▼                  ▼
分數 = (0.92 × 0.80)  +  (0.61 × 0.50)  =  0.736 + 0.305 = 1.04
        └ pgvector ┘     └─ 索引 ─┘
       （"HNSW"、"向量" 只有一邊有，不算）
```

> 數學上這其實就是**兩個稀疏向量的內積**——只是因為大多數維度是 0，只有共同的詞算得出乘積。所以你可以理解成：**Sparse 還是在算內積，只是在一個超高維但極空的空間裡。**

### Sparse 的脾氣

- 像一個**會「學習」的關鍵字搜尋**（升級版 TF-IDF / BM25）。傳統關鍵字搜尋只會數次數，它會判斷「哪個詞才是重點」。
- ✅ 抓「**精確字詞命中**」：型號、專有名詞、錯誤碼，只要字面有出現就抓得準——**正好補 Dense 的弱點**。
- ❌ 不懂同義詞：「狗」和「犬」字面不同，Sparse 看不出關係（那是 Dense 的強項）。

## 5. Multi-Vector 多向量（ColBERT 風格 / Late Interaction）

### 長相

前兩種都把整段壓成「一個東西」。Multi-Vector 偏不壓——**每個 token 各留一個自己的向量**：

```
"Docker 啟動 失敗"
   │      │     │
   ▼      ▼     ▼
 v_docker v_啟動 v_失敗     ← 3 個 token → 3 個 1024 維向量
```

一段文字 = **一疊向量**（N 個 token 就 N 個向量），不是單一向量。

### 怎麼比相似度 → 逐詞餘弦 + 取 max + 加總（late interaction）

對**查詢的每一個 token**，去文件裡**找最像它的那個 token**（這一步就是你熟的餘弦/內積），記下最高分；最後把查詢所有 token 的最高分加起來：

```
查詢 token:   q1=Docker      q2=啟動       q3=失敗
                │              │             │
   各自去文件裡找最像的 token（逐對算餘弦，取最大那個）
                │              │             │
              max=0.95       max=0.88      max=0.91
                └──────────────┼─────────────┘
                          相加 → 總分 2.74
```

公式長這樣（看得懂就好）：

```
分數 = Σ      max   ( q_i · d_j )
      查詢每個   文件每個
      token i   token j
```

「先各自獨立編碼，最後比對時才逐詞互動」——這個「最後才互動」就是 **late interaction（後期互動）** 的意思。

### Multi-Vector 的脾氣

- ✅ **最精準**：細到「逐詞對逐詞」，能抓到「查詢的某個詞，精準對上文件某個詞」的細節。
- ❌ **最貴**：一段文字要存一疊向量（不是一個），儲存和計算成本都最高。實務上常**只拿它來對前面初篩出的少數候選做重排序（rerank）**，不會拿來掃全庫。

## 6. 三種怎麼一起用？Hybrid（混合檢索）

BGE-M3 的賣點就是「一個模型同時給你三種」，所以可以**加權合併**，互補短處：

```
最終分數 = w1 · Dense        （懂語意、抓同義詞）
         + w2 · Sparse       （抓精確關鍵字）
         + w3 · Multi-Vector （逐詞細修，最準）
```

| 單獨用某一種的弱點 | 被誰補 |
|---|---|
| Dense 抓不到精確型號/錯誤碼 | Sparse |
| Sparse 不懂同義詞、換句話說 | Dense |
| 兩者都是「整段 vs 整段」，不夠細 | Multi-Vector 逐詞重排 |

→ 常見做法：**Dense + Sparse 快速初篩**一批候選，再用 **Multi-Vector 重排**前幾名。

---

## 7. 一張表收束：三種其實都在用你會的「向量相似度」

別被三個名詞嚇到。骨子裡都是「算向量像不像」，差別只在**用幾個向量、怎麼聚合**：

| | 用幾個向量比 | 核心運算 | 聚合方式 |
|---|---|---|---|
| Dense | 1 對 1 | 餘弦 / 內積 | 不用聚合，一次就出分 |
| Sparse | 超高維但極空的 1 對 1 | 內積（共同詞權重相乘）| 共同詞加總 |
| Multi-Vector | N 對 M（逐詞）| 餘弦 / 內積 | 每個查詢詞取 max，再全部加總 |

**你已經會的餘弦/內積，是這三種共用的積木。** 三種的差別是「拿幾塊積木、怎麼疊」。

## 8. ⚠️ 你的 abby-notes-rag 現在用到哪一種？

**只用了 Dense 一種。**

證據：
- `scripts/init_db.sql` 的 `chunks.embedding` 是 `vector(1024)` —— 一個 chunk 存**一個**向量，這就是 Dense 的形狀。
- `src/embedder.py` 用 `sentence-transformers` 的 `model.encode(...)`，這個 API **預設只回 dense 向量**。
- 查詢用 pgvector 的 `<=>`（餘弦距離）—— 對應 Dense 的比法。

也就是說：**你目前是把 BGE-M3 當成一個「普通的 1024 維 dense 模型」在用，三種能力只開了第一種。** 這對個人筆記 RAG 來說完全夠用，不是缺點。

想之後解鎖 Sparse / Multi-Vector 的話：`sentence-transformers` 的 `.encode()` 拿不到，要改用官方的 **FlagEmbedding** 套件（`BGEM3FlagModel`），它能一次回傳 `dense_vecs` / `lexical_weights` / `colbert_vecs` 三種。那是 Phase 2 hybrid search 才需要的東西，現在不用急。

## 9. 速記卡

```
BGE-M3 = BAAI 的 embedding 模型，M3 = 三個 Multi
  ├ Multi-Functionality → Dense / Sparse / Multi-Vector（本篇）
  ├ Multi-Linguality    → 100+ 語言（中英混查）
  └ Multi-Granularity   → 8192 tokens 長文

三種向量：
  Dense  稠密  → 1 個 1024 維向量，格格有值 → 餘弦比 → 抓「意思」
  Sparse 稀疏  → 詞→權重，25萬維但極空 → 共同詞權重相乘加總 → 抓「字詞」
  Multi  多向量 → 每 token 一個向量 → 逐詞餘弦+max+加總 → 最準最貴

骨子裡都是「餘弦/內積」，差在用幾個向量、怎麼聚合
正規化後：餘弦排序 = 歐式排序（名次一樣）

abby-notes-rag 目前：只用 Dense（vector(1024) + <=>）
想解鎖另外兩種：改用 FlagEmbedding 的 BGEM3FlagModel
```
