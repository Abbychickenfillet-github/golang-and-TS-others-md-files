🐍 ModuleNotFoundError: No module named 'tiktoken'

Abby-notes RAG ｜ venv 沒啟動的經典症狀 ｜ 2026-06-21 ｜ Abby + Claude

錯誤訊息 
PS C:\coding\futuresign\abby-notes-rag> python scripts/ingest.py --dry-run
...
  File "src\chunker.py", line 7, in <module>
    import tiktoken
ModuleNotFoundError: No module named 'tiktoken'

tiktoken 沒有不見! 它好好地裝在專案 venv 裡: venv\Lib\site-packages\tiktoken(version 0.8.0,寫在 requirements.txt)。

真正原因:你那個 PowerShell 沒有啟動 venv。看提示字元 — PS C:\...abby-notes-rag> 前面沒有 (venv), 代表 python 用到的是系統的 Python,而不是專案 venv 的 Python。系統 Python 當然沒裝 tiktoken。

① 為什麼會這樣（觀念圖）

你打 python → Windows 沿 PATH 找到第一個 python.exe
▼
沒啟動 venv 時 → 命中系統/全域 Python(它的 site-packages 沒有 tiktoken)
▼
啟動 venv 後 → PATH 最前面換成 venv\Scripts → 命中 venv 的 python(有 tiktoken) ✓

venv = 專案專屬的隔離 Python 環境。套件裝進 venv,只有「啟動該 venv」或「直接呼叫 venv 的 python.exe」時才看得到。

② 解法（擇一）

方法 A：啟動 venv 再跑（建議,跟 README 一致）

.\venv\Scripts\Activate.ps1
# 成功後提示字元會變成:  (venv) PS C:\...abby-notes-rag>
python scripts\ingest.py --dry-run

方法 B：不啟動,直接用 venv 的 python（一次性）

.\venv\Scripts\python.exe scripts\ingest.py --dry-run

確認你現在用的是哪個 python：

python -c "import sys; print(sys.executable)"
# 有啟動 venv 應印出 ...\abby-notes-rag\venv\Scripts\python.exe

⚠️ 若 Activate.ps1 也報紅字

如果 .\venv\Scripts\Activate.ps1 跳出 「因為這個系統上已停用指令碼執行」 — 那跟你的 claude.ps1 跑不出來 是同一個根因:PowerShell 執行原則擋 .ps1。同一帖藥:

Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

按 Y 確認,重開終端機再 Activate。👉 詳見關聯筆記 [[claude-指令跑不出來-排查筆記]]。

③ 萬一 venv 真的壞了 / 套件缺一堆

# 重新安裝相依(在已啟動的 venv 內)
pip install -r requirements.txt

# 或整個 venv 重建
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt

④ 填空題（點灰框看答案）

1. 提示字元前面出現 (venv) 才代表虛擬環境已啟動。

2. PowerShell 啟動 venv 的指令:.\venv\Scripts\Activate.ps1。

3. 查目前 python 執行檔在哪:python -c "import sys; print(sys.executable)"。

4. 不啟動也能用 venv 跑腳本,直接呼叫 .\venv\Scripts\python.exe。

⑤ 是非題

Q1. 出現 ModuleNotFoundError: tiktoken 一定是因為 tiktoken 沒裝,要重新 pip install。

Q2. Activate.ps1 報「停用指令碼執行」與 claude.ps1 跑不出來是同一個 ExecutionPolicy 根因。

⑥ 申論題

Q：用「PATH 搜尋順序」解釋:為什麼啟動 venv 前後,同一句 python 會跑到不同的直譯器、進而決定看不看得到 tiktoken?

▸ 顯示參考答案

輸入 python 時,系統會依 PATH 由前往後找第一個 python.exe 並執行。沒啟動 venv 時,PATH 最前面是系統/全域 Python,於是用到它的 site-packages(沒有 tiktoken)。

Activate.ps1 做的事,本質上是把 venv\Scripts 插到 PATH 最前面(並設 VIRTUAL_ENV 等變數),於是同一句 python 改命中 venv 的直譯器,看到的是 venv 的 site-packages(有 tiktoken)。

所以「套件在哪個環境」+「現在 python 是哪個環境」要對得上;對不上就 ModuleNotFound。直接呼叫 venv\Scripts\python.exe 等於略過 PATH、強制指定直譯器,同樣能解。
