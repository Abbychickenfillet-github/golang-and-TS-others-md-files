冒泡排序法、range() 參數與時間複雜度

重點整理

冒泡排序法（Bubble Sort）的核心是「相鄰兩兩比較、較大的往後沉」。理解它的 range() 邊界與真正的執行次數，就能把演算法跟 Python 語法一次打通。

為什麼內層 range() 的 stop 要寫 n-1？

關鍵在於冒泡排序是兩兩比較，程式碼會同時存取 arr[j] 和 arr[j+1]。

  • 若寫 range(0, n)：最後 j = n-1 時會去比 arr[n-1] 與 arr[n] → arr[n] 超出索引、IndexError。
  • 若寫 range(0, n-1)：因為 Python 的 range(start, stop) 是「包含起點、不包含終點」，索引最大只到 n-2，剛好拿 arr[n-2] 與 arr[n-1]（最後一項）比，完美覆蓋又不越界。

[!example] 以 arr = [50, 30, 40, 10, 20]（n=5）為例
第一輪只需比 4 次：(0,1)(1,2)(2,3)(3,4)。range(0, n-1) = range(0,4) 產生 0,1,2,3，正好對應這 4 次比較。

對比：單純遍歷時為什麼「不用」 -1？

x = 'runoob'
for i in range(len(x)):   # range(6) -> 0,1,2,3,4,5
    print(x[i])           # 只存取 x[i]，沒有 x[i+1]

差別在於這裡是「單個元素獨立處理」，只寫了 x[i]、沒有 x[i+1]，所以索引必須完整走到最後一格。若誤加 -1 變成 range(5)，最後一個字元 'b'（索引 5）就會被漏掉。

[!note] 口訣
兩兩比較、有寫到 +1 ➡️ range 終點要 -1（防越界）。
單純巡邏、只寫 [i] ➡️ range 直接用 len()（確保每個都點到名）。

range() 的本質：計數器

range() 的核心任務是「數數 / 產生索引」，不是用來「跑函式」。三大用途：

  1. 控制重複次數：for i in range(3): say_hello()
  2. 產生索引存取資料（最常見）：for i in range(len(names)): print(names[i])
  3. 產生規律數字：range(2, 11, 2) → 2,4,6,8,10

[!tip] 更 Pythonic 的寫法
若只是想把每個元素丟進函式處理，直接 for num in numbers: square(num)（迭代 Iterable）比 for i in range(len(...)) 更乾淨。

單一參數的 range() 代表什麼？

range() 完整簽名是 range(start, stop, step)，其中 start 預設 0、step 預設 1。當只給一個參數時，Python 把它當作 stop（而非 start），所以 range(10) = 0,1,...,9 共 10 個。這是 range 這個函式自己的參數設計，並非所有 Python 函式都這樣。

外層 j 為什麼跑 range(n)？真的是 N×N 次嗎？

每跑完一整輪內層，就有一個當前最大值被定位到最右端。理論上外層只要 n-1 輪就夠（最後一個自動歸位），但教學常直接寫 range(n)：直覺好懂，且最後一輪 range(0, 0) 是空迴圈、不會出錯。

實際比較次數不是 N×N。以 arr=[80,50,44,50,70,11]（n=6）為例：

j=0 -> range(0,5) -> 5 次
j=1 -> range(0,4) -> 4 次
j=2 -> range(0,3) -> 3 次
j=3 -> range(0,2) -> 2 次
j=4 -> range(0,1) -> 1 次
j=5 -> range(0,0) -> 0 次
合計 5+4+3+2+1+0 = 15 次（而非 6×6 = 36 次）

[!warning] 大 O 為什麼還是 O(N²)？
精確次數是 N×(N-1)/2 = ½N² − ½N。當 N 很大時，後面的 ½N 與常數係數 ½ 都會被忽略，只看成長「等級」，所以統一記作 O(N²)。實際大約只做「一半」的次數，但等級不變。

[!note] 小提醒：print(arr) 縮排
若把 print(arr) 縮排在 if 裡，代表「只有發生交換才印」，印出的次數是「交換次數」而非「比較次數」，會少於 15 次。

優化寫法

arr = [80, 50, 44, 50, 70, 11]
n = len(arr)
for j in range(n - 1):            # 只跑 n-1 輪，省掉沒意義的最後一輪
    for i in range(0, n - j - 1):
        if arr[i] > arr[i + 1]:
            arr[i], arr[i + 1] = arr[i + 1], arr[i]

延伸：Brute Force（暴力法 / 蠻力法）

Brute Force 的核心邏輯是：不做任何聰明的過濾或優化，單純窮舉所有可能、全部測一遍直到找到答案。

  • 在演算法中：冒泡排序就是一種暴力排序；找 3 位數密碼就從 000 試到 999。優點是「簡單、保證找到」，缺點是「資料量大時次數爆炸、很慢」（如冒泡的 O(N²)）。
  • 在資安中：暴力破解攻擊（Brute Force Attack）靠電腦每秒上百萬次猜密碼；變形是「字典攻擊」拿常見密碼清單狂試。這就是為什麼網站要求密碼複雜、且「錯誤幾次就鎖定帳號」。

口語：「我先用 Brute force 寫個能動的版本，再來優化」＝用時間與效能換取簡單邏輯。
