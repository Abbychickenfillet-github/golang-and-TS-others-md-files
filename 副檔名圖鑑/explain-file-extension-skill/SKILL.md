# 解釋副檔名 SOP

當使用者出現陌生副檔名時,用以下結構回答(簡單化、不要學術腔):

## 回答四要素

1. **是什麼**:全名 + 一句話用途。
   (例:`.vscdb` = VS Code 系列的 SQLite 狀態資料庫,存工作區狀態/聊天紀錄)
2. **誰在用**:哪個程式/情境會產生它。
3. **文字還是二進位**:
   - 純文字(.json/.yml/.ini/.env/.md/.log)→ 可用編輯器打開/修改
   - 二進位(.dll/.exe/.vscdb/.db/.vhdx)→ 別用記事本硬改,會壞
4. **能不能刪**:
   - 系統/設定/資料庫檔 → 別亂刪(先備份)
   - 快取/log/暫存/編譯產物(.pyc/.log/cache)→ 通常可刪

## 判斷小抄

- 名字含 `cache / temp / log` → 多半可刪
- `.dat`、`.sys`、`NTUSER.DAT`、`.vscdb`、`.db` → 別亂刪(可能是設定/資料/系統)
- `.wal` 一定跟著同名 `.db` 主檔,不要單獨刪
- `.dll` 缺了會讓程式報「找不到 xxx.dll」→ 別刪

## 收尾動作(重要)

如果這個副檔名還沒在圖鑑裡,**把它補進**:
`C:\coding\futuresign\Abby-notes\副檔名圖鑑\副檔名意義總表.md`
(加到對應分類的表格,或最底部「待補」表加一列:副檔名 | 意義 | 日期)

## 風格
- 用繁體中文、口語、給比喻(例:.dll 像「多個 App 共用的工具箱」)。
- 順便說「能不能刪 / 能不能編輯」,因為使用者常在清理硬碟時問。
