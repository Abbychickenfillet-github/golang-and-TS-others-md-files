# 文档结构说明

## 文档组织原则

### 后端相关文档 → `backend/backend-docs/`
所有后端相关的技术文档都应该存放在 `backend/backend-docs/` 目录中，包括：
- FastAPI 路由和 API 说明
- 数据库相关文档
- Docker Compose 配置说明
- 后端开发指南
- API 测试文档

### 前端相关文档 → `frontend_docs/`
前端相关的技术文档存放在 `frontend_docs/` 目录中。

### 项目通用文档 → `docs/`
项目级别的通用文档存放在 `docs/` 目录中。

### 项目流程文档 → `flow&docs/`
项目开发流程、阶段总结、规划文档存放在 `flow&docs/` 目录中。

### 配置和状态文档 → `docs&memory/`
开发过程中的配置状态、环境变量说明等存放在 `docs&memory/` 目录中。

## 文档命名规范

- 使用描述性的文件名
- 使用 kebab-case 命名（例如：`routing-and-docker-explanation.md`）
- 日期格式的文档使用 `YYYY-MM-DD_description.md` 格式
