# 文档目录说明

## 文档组织结构

### `/docs/` - 项目通用文档
存放项目级别的通用文档，适用于整个项目（前端、后端、部署等）。

### `/backend/backend-docs/` - 后端专用文档
存放后端相关的技术文档，包括：
- API 路由说明
- FastAPI 使用指南
- 数据库相关文档
- 后端开发指南
- Docker Compose 配置说明

### `/frontend_docs/` - 前端专用文档
存放前端相关的技术文档。

### `/flow&docs/` - 项目流程和规划文档
存放项目开发流程、阶段总结、规划文档等。

### `/docs&memory/` - 开发记忆和配置文档
存放开发过程中的配置状态、环境变量说明等。

## 文档命名规范

- 使用描述性的文件名
- 后端文档统一放在 `backend/backend-docs/`
- 使用 kebab-case 命名（例如：`routing-and-docker-explanation.md`）
