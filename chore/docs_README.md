# 文檔目錄說明

## 文檔組織結構

### `/docs/` - 項目通用文檔
存放項目級別的通用文檔，適用於整個項目（前端、後端、部署等）。

### `/backend/backend-docs/` - 後端專用文檔
存放後端相關的技術文檔，包括：
- API 路由說明
- FastAPI 使用指南
- 資料庫相關文檔
- 後端開發指南
- Docker Compose 配置說明

### `/frontend_docs/` - 前端專用文檔
存放前端相關的技術文檔。

### `/flow&docs/` - 項目流程和規劃文檔
存放項目開發流程、階段總結、規劃文檔等。

### `/docs&memory/` - 開發記憶和配置文檔
存放開發過程中的配置狀態、環境變數說明等。

## 文檔命名規範

- 使用描述性的文件名
- 後端文檔統一放在 `backend/backend-docs/`
- 使用 kebab-case 命名（例如：`routing-and-docker-explanation.md`）
