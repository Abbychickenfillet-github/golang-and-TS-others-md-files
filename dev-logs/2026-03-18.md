## 攤位價格計算錯誤（booth_price_snapshot = 0）

### 現象
- 精品香水-01、精品香水-02 兩個攤位，類型「擴大空間 Expended」default_price = 30,000
- 但訂單 BTH-20260318-2TW0A2 的 booth_price_snapshot 都是 0
- 訂單 total_amount 只有 3,050（= 電力 3,000 + 設備 50），缺少攤位費 60,000

### 原因
- `override_booth_price = 0.00`（攤位沒設個別價格）
- `event_booth_type_pricing` 三筆時間定價的 `target_areas` 都不包含「精品香水」區域
- 後端 registration_service 計算價格時，沒有 fallback 到 `event_booth_type.default_price`

### 資料庫現況
```sql
-- 兩個攤位
精品香水-01 (8413cf1c): override_booth_price=0, booking_status=available
精品香水-02 (58755bbe): override_booth_price=0, booking_status=available

-- 攤位類型定價（都不匹配精品香水區域）
早鳥價:     15,001  (02/10~03/09)  target_areas=NULL（已過期）
新早鳥價:   13,444  (03/12~03/14)  target_areas=["台灣品牌"]
測試早鳥價:  6,600  (03/15~03/17)  target_areas=["餐飲區","台灣品牌"]
```

### 待修復
- [ ] registration_service.go 價格計算邏輯：當沒有匹配的時間定價時，fallback 到 default_price
