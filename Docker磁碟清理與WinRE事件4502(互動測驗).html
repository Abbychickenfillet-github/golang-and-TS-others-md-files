1. 核心觀念:vhdx「只長不縮」

Docker 的資料存在虛擬磁碟檔 docker_data.vhdx 裡。它是動態擴充——用多少長多少,但刪東西後不會自動縮小。

比喻:vhdx 像吹大的,把裡面的水倒掉(prune)後,它還是那麼大,要 compact 才會消氣。

填空

清內部內容的指令是 ;把檔案實體縮小的動作叫 。

氣球;prune 清水、compact 消氣。兩步都做完,Windows 才拿得回空間。

2. 清理兩步驟

步驟指令作用
① 清 Docker 內部docker builder prune -a刪掉沒在用(甚至有用)的建置快取
② 關 WSLwsl --shutdown停掉 WSL 虛擬機、放開 vhdx 檔案鎖
③ 壓縮 vhdxdiskpart → compact vdisk把空洞擠掉,空間真正還給 C 槽
順序重點:先 Quit Docker Desktop → 再 wsl --shutdown → 再進 diskpart compact。Docker 開著會把 WSL 叫回來,vhdx 又被鎖住。

3. Docker 四大佔空間物件

名稱是什麼比喻
image唯讀範本App 安裝檔
container從 image 跑起來的實例正在跑的 App
volume掛在 container 的持久資料資料庫檔案
build cache建 image 的中間層編譯暫存

是非題

綠色「in use」的 image 應該優先刪掉來省空間。

刪掉 volume 會連同裡面的資料庫資料一起消失。

第一題「錯」:in use 代表有容器在用,那是你正在運作的專案(backend-go、mysql…),不該亂刪。真正肥的是 build cache。第二題「對」:volume 是持久資料,刪了不可逆。

4. 為什麼 vhdx 會被「鎖住」

docker_data.vhdx 就是 WSL2 那顆 Linux 虛擬機的硬碟。VM 運作時把它獨佔掛載(exclusive lock),同一時間只能一個程序用一顆磁碟。

盲點:工作管理員關掉的是 Docker 程序,但 WSL 的虛擬機(Vmmem)是另一個獨立東西,不會跟著停。只有 wsl --shutdown 才會優雅關掉它、放開鎖。用工作管理員硬殺 Vmmem 有讓檔案系統損毀的風險。

申論題

用自己的話說明:為何 wsl --shutdown 比在工作管理員結束 Vmmem 更好?

wsl --shutdown 是優雅關機,會通知 WSL 正常卸載檔案系統、把緩衝資料寫回磁碟再關;工作管理員結束 Vmmem 是硬拔電源,可能讓 vhdx 內的 ext4 停在未寫完狀態(雖有日誌通常能自救,但有風險)。兩者都能放開檔案鎖,但前者安全得多。

5. diskpart vs 磁碟管理

diskpart 是磁碟分割管理工具,能分割、格式化、compact vdisk…功能強大但危險(clean 會清空整顆磁碟、無法復原)。

一般分割 / 格式化 / 處理外接硬碟,改用圖形介面 diskmgmt.msc(磁碟管理),誤刪風險低很多。diskpart 動手前先 list disk 看清楚編號。

填空

離開 diskpart 的指令是 ;查看有哪些磁碟用 。

exit;list disk。注意 wsl、docker 指令不能在 diskpart 裡跑,提示字元 DISKPART> 代表你人還在 diskpart 內。

6. Windows 事件 4502 / WinREAgent 事件日誌

WinREAgent = Windows Recovery Environment Agent,負責維護/更新電腦裡的修復磁碟分割(重設此電腦、開機修復用的救援環境)。跟 AI Agent 完全無關。

事件 4502「Windows 修復環境服務失敗」:某次 Windows Update 想更新救援分割區但失敗,最常見原因是磁碟太滿 / 救援分割區空間不足。

日期時間來源事件 ID層級
2026/7/9 21:24:03WinREAgent4502重大
2026/7/9 16:04:07WinREAgent4502重大
2026/7/3 10:06:24WinREAgent4502重大
2026/6/28 20:09:37WinREAgent4502重大
7/3 那次失敗時 C 槽只剩約 1.5MB,完全對得上「磁碟太滿」。通常不影響日常使用,只是救援環境的更新沒套用。空間清出來後未來更新有機會自動補上;若持續失敗,可依微軟 KB 調整救援分割區大小。

是非題

WinREAgent 是我電腦裡的 AI 代理程式出錯了。

錯。WinREAgent 是 Windows 修復環境的維護元件,與 AI 無關;4502 多半肇因於磁碟/救援分割區空間不足。

7. Kernel-EventTracing 警告(ETW) 事件日誌

來源 Kernel-EventTracing、事件 ID 1、作業碼 Write Buffer 的一堆「警告」,訊息大意是:某個即時追蹤工作階段(例如 intel.collector.lbr)的記錄檔達到大小上限,在騰出空間前暫停寫入。

更精確的說法:這是 ETW(Windows 事件追蹤 / Event Tracing for Windows) 某個追蹤工作階段自己的記錄檔寫滿了,不是整顆硬碟容量不足。多半是背景追蹤器的雜訊,通常無害。跟 WinRE 4502「磁碟太滿」是兩回事。

是非題

Kernel-EventTracing 的警告代表我的硬碟整體空間不足。

錯。指的是「那個追蹤工作階段的記錄檔」達到它自己的大小上限,不是磁碟滿。屬雜訊等級警告。

8. 權限:其實只有 2 個真概念

「管理員身分」和「系統管理員身分」在中文是同一個詞(Administrator),不是兩層。真正的區別是:

概念意思
① 帳戶是不是管理員帳戶類型(靜態身分)
② 以系統管理員身分執行UAC 提權——把程式換成你帳戶的「完整權杖」。不是切換去內建 Administrator
③ 內建 Administrator 帳戶另一個獨立、預設停用的超級帳戶
「以系統管理員身分執行」提升到的是「你自己帳戶的完整管理員權限」,不是變成內建 Administrator。另外 wsl --list --running / wsl --shutdown 不需要管理員權限。

9. 環境變數與兩個 vhdx

Docker 在 WSL 裝了兩個發行版,各有各的虛擬磁碟:

檔案屬於存什麼大小
main\ext4.vhdxdocker-desktop(引擎)Docker 的 Linux 系統小
disk\docker_data.vhdxdocker-desktop-dataimages / containers / volumes / build cache大(要 compact 這個)

dir /s /b 的參數:/s = 遞迴搜尋所有子資料夾;/b = 精簡格式,只印完整路徑。

環境變數指向用途
%USERPROFILE%C:\Users\User家目錄本體
%LOCALAPPDATA%C:\Users\User\AppData\Local本機 App 資料(Docker vhdx 放這)
%APPDATA%C:\Users\User\AppData\Roaming可漫遊的 App 設定
💡 用變數的好處:不管使用者叫什麼名字,%LOCALAPPDATA% 都會自動指到對的地方。寫腳本、貼指令時用變數,就不用管對方帳號是 User、Abby 還是別的——路徑永遠正確、可攜。

填空

家目錄的環境變數是 ;要 compact 的大檔是 。

%USERPROFILE%;docker_data.vhdx(docker-desktop-data 的虛擬磁碟)。

10. diskpart 是什麼 & .msc 附檔名 小教學

diskpart 不是 cmd、也不是 powershell,而是一支獨立的命令列程式(diskpart.exe)。你在 CMD 或 PowerShell 任一個打 diskpart 都能叫起來,叫起來後就進入它自己的介面(提示字元 DISKPART>)——這裡只吃 diskpart 的指令。需要系統管理員權限。

啟動與離開:

diskpart          <- 在 CMD/PowerShell 啟動(需管理員)
list disk         <- 先看清楚有哪些磁碟(重要!)
exit              <- 離開,回到原本的 shell
危險提醒:clean / delete / format 會清空整顆磁碟且無法復原。動手前一定先 list disk 確認編號。一般分割/格式化改用圖形介面 diskmgmt.msc 更安全。

.msc 附檔名是什麼?

.msc = Microsoft Management Console(MMC)的主控台檔。雙擊它會用 MMC 開啟對應的「管理主控台」。常用的有:

.msc 檔開啟什麼
diskmgmt.msc磁碟管理(圖形化分割/格式化)
eventvwr.msc事件檢視器
services.msc服務
devmgmt.msc裝置管理員
compmgmt.msc電腦管理(把上面幾個包在一起)
taskschd.msc工作排程器

是非題

diskpart 是 PowerShell 內建的指令,只能在 PowerShell 裡用。

錯。diskpart 是獨立程式,CMD 或 PowerShell 都能啟動,進去後是它自己的 DISKPART> 介面。

填空

圖形化磁碟管理的檔案是 ;事件檢視器是 。

diskmgmt.msc;eventvwr.msc。這兩個 .msc 都由 MMC 開啟。

申論題

什麼時候該用 diskpart、什麼時候該用 diskmgmt.msc?

日常的分割、格式化、指派磁碟機代號、處理外接硬碟 → 用圖形介面 diskmgmt.msc,看得到名稱容量、誤刪風險低。只有圖形介面做不到的特殊操作(例如 compact vdisk 壓縮虛擬磁碟、清除保護分割)才用 diskpart,且動手前先 list disk。
