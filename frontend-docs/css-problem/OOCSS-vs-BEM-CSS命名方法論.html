OOCSS vs BEM（CSS 命名方法論）

重點整理

兩者都是讓 CSS 模組化、可重複利用、好維護的方法論，目的都只有兩個：讓 CSS 檔案變小（好重複利用）、讓程式碼好維護（不會改 A 壞 B）。它們不是敵對，而是可以搭配的「內功心法」與「招式套路」。

OOCSS（Object-Oriented CSS）

核心思想：結構（Structure）與外觀（Skin）分離，把元件當成獨立「物件」，像樂高積木一樣自由組合。

/* 結構 Structure：大小、邊距、字體 */
.btn { padding: 10px 20px; border: none; border-radius: 4px; cursor: pointer; }
/* 外觀 Skin：只負責顏色、背景 */
.primary-skin   { background-color: blue; color: white; }
.secondary-skin { background-color: gray; color: white; }

<button class="btn primary-skin">主按鈕</button>
<button class="btn secondary-skin">次要按鈕</button>
  • 優點：重複利用性極高（卡片想用同藍色，直接套 .primary-skin）。
  • 缺點：HTML class 變很長，難一眼看出 class 寫給誰用；不同人實作差異大。

BEM（Block、Element、Modifier）

核心思想：嚴格的命名規範，讓結構一目了然。

  • B（Block）：獨立區塊，例如 btn
  • E（Element）：區塊內子元素，用 __ 連接（例如 btn__icon）
  • M（Modifier）：狀態或變體，用 -- 連接（例如 btn--primary）
.btn { padding: 10px 20px; border: none; border-radius: 4px; cursor: pointer; }
.btn--primary { background-color: blue; color: white; }

<button class="btn btn--primary">主按鈕</button>
  • 優點：命名嚴格有邏輯，看到 btn--primary 就知它依附在 btn 底下，不會誤用。
  • 缺點：名字常變很長（例如 header__navigation-list--dark）。

快速總結表

特性 OOCSS BEM
核心精神 樣式抽成「樂高積木」自由組合 用嚴格命名表達元件隸屬關係
Class 命名 通常很短（btn, primary-skin） 較長但清晰（btn, btn--primary）
適用場景 減少 CSS 大小、重視重複利用 團隊協作、好維護、不易改壞別人

混血（Hybrid）實務作法

很多前端會結合兩者：用 OOCSS 的「心法」（結構/外觀分離）+ BEM 的「規範」（命名）。把 OOCSS 的外觀（Skin）用 BEM 的 Modifier（--）命名：

/* 結構（OOCSS 結構 + BEM Block 命名） */
.box { padding: 20px; border-radius: 8px; display: flex; }
/* 外觀（OOCSS 外觀 + BEM Modifier 命名）：獨立存在、可自由組合 */
.theme--light   { background: #fff; }
.theme--warning { background: #ffd; }

<div class="box theme--warning">…</div>

重要延伸

  • React 開發中，BEM 精神（或 CSS Modules）更常被使用，因為能確保樣式不互相污染。
  • Utility-First（如 Tailwind CSS）本質上是把 OOCSS「結構與外觀分離、極大化重複利用」發揮到極致的演變。
  • 釐清：React 組件 ≠ OOCSS。React 組件是「拆 UI 成可重用部件」；OOCSS 是「寫 CSS 的方法論」。可在 React 組件內套用 OOCSS / BEM 來組織樣式。
