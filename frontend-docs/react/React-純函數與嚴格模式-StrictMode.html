重點整理

純函數 (Pure Function) 是什麼

純函數指的是：給定相同的輸入，就一定會產生相同的輸出，而且不能有任何副作用 (side effect)。

為什麼 React 要求 render 是純的

React 在渲染元件時，把元件當成一個「輸入 props/state → 輸出 UI」的純函數。如果你在渲染（或計算狀態更新）的過程中去修改外部變數、或直接發 API 請求，這些都算「副作用」，很容易導致元件狀態不穩定、畫面對不上資料。

正解：副作用要放在對的地方 —— 例如 useEffect、事件處理函式 (event handler) 裡，而不是寫在 render 主體中。

嚴格模式 (StrictMode) 的角色

StrictMode 是 React 用來幫你抓出這些潛在問題的開發工具。它會故意「重複執行」一段程式碼（例如把 render／某些函式呼叫兩次），藉此放大「不純」的行為 —— 如果你的 render 有副作用，跑兩次結果就會不一致，問題馬上現形。

補充：StrictMode 的雙重呼叫只發生在開發模式 (development)，正式上線 (production) 不會重複執行，所以不用擔心效能。
