✍️ D. 申論題打字作答 → 對參考答案

寫出關鍵字就會被判定「抓到重點」。

D1. 「狀態容器」和「狀態管理」差在哪？為什麼說 Context 不等於狀態管理？

狀態容器只是「放狀態的地方」；狀態管理涵蓋狀態的流動、更新、存取邏輯。Context 是容器，沒內建全域更新/非同步處理，且過度使用會因 value 變動造成大量重渲染、引發效能問題，所以不等於狀態管理；複雜場景用 useReducer 或 Redux。

D2.〔你踩過的 bug〕return 後換行寫 JSX 卻渲染不出來，為什麼？怎麼修？

JS 的自動分號插入（ASI）會在換行的 return 後補 ;，變成 return;，後面的 JSX 不會被回傳。修法：JSX 跟 return 寫同一行，或用小括號 return ( … ) 包起來成同一語句。

D3. 用 Context 做路由保護，未登入要踢去登入頁。實作上有哪兩個關鍵點？

①等 router.isReady 為真才判斷（否則 pathname 可能還沒就緒）。②user 初始設 null 代表未登入；若 !user 且 protectedRoutes.includes(router.pathname) 就 router.push(loginRoute) 導向登入頁。

D4.〔跨領域對照〕前端的 AuthProvider 跟後端登入（JWT/OAuth）各自負責什麼？

前端 AuthProvider 用 Context 在瀏覽器端「保存／共享」登入狀態（多半是後端發回的 token / 使用者資訊），並做路由保護。真正「驗證身分」在後端：OAuth 取得授權、核發 JWT。前端只是拿著後端發的憑證。對照 [[GOOGLE_OAUTH_SETUP]]、[[JWT_TOKEN_EXPLANATION]]。

⭕❌ E. 是非題判斷對錯

每題先判斷⭕/❌，再點看答案。

E1. 用了 Context 之後，中間層元件必須把值再用 props 轉傳給更深的子元件。
❌ 錯。Context 的重點就是跨層直送，中間層不需要轉傳，正好消掉 props drilling。
E2. 把整站所有狀態都塞進同一個 Context 是推薦做法。
❌ 錯。value 一變所有消費者都重渲染，會有效能問題。應拆分小 Context 或用 useReducer/Redux。
E3. {Children}（大寫）可以直接當作要渲染的子內容。
❌ 錯。大寫 Children 是 React 的工具 API（map/forEach/count/toArray/only），直接渲染會噴「Objects are not valid as a React child」。要用小寫 children。
E4. 一個 Next.js 專案的 _app.js 只能有一個。
⭕ 對。它是唯一入口，放在 pages/ 下，包全站 Provider 與全域 CSS，頁面切換不卸載。
E5. 把登入 token 存在 localStorage 跟存在 HttpOnly cookie，安全性是一樣的。
❌ 錯。localStorage 可被 JS（含 XSS 注入的腳本）讀取；HttpOnly cookie 的 JS 讀不到，較安全。對照 Cookie-與-Session 的安全旗標。
