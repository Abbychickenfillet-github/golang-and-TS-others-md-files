VS Code / Cursor 設定介面（UI vs JSON）與側邊欄位置

重點整理

[!warning] 為什麼按 Ctrl + , 會直接跳到文字編輯？
沒出現平常的圖形化設定搜尋介面、卻直接開啟類似 settings.json 的文字檔，代表預設設定檢視器被鎖定在「文字（JSON）模式」。

找回圖形化設定介面（兩步驟）

方法一：命令面板（最快）

  1. 按 Ctrl + Shift + P 開啟命令面板。
  2. 輸入 Preferences: Open User Settings（中文：開啟使用者設定）。
  3. 點選沒有寫 (JSON) 的那個選項，即回到圖形介面。

方法二：永久改回 Ctrl + , 開圖形介面

在 settings.json 檢查或加入：

"workbench.settings.editor": "ui"

若這行目前是 "json"，每次按快捷鍵就會直接跳文字編輯；改回 "ui" 即恢復正常（在圖形介面搜尋 Workbench > Settings: Editor 也能改）。

對調／移動側邊欄位置

回到圖形介面後搜尋 Side Bar Location 即可把側邊欄在左右之間切換。
