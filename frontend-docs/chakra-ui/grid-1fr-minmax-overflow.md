# CSS Grid 欄寬對齊問題：1fr、minmax、文字溢出

## 情境

票券頁表格的 header 和 data 用了相同的 `templateColumns`，但銷售期間那欄的日期文字太長，把後面的操作欄推歪了。

---

## `1fr` 是什麼？

`fr` = **fraction**（比例單位），代表「剩餘空間的一份」。

```
templateColumns="200px 1fr 100px"
```

意思是：
- 第一欄固定 200px
- 第三欄固定 100px
- **第二欄拿走所有剩下的空間**

如果容器寬 1000px → 第二欄 = 1000 - 200 - 100 = **700px**

多個 `fr` 會按比例分配：

```
templateColumns="2fr 1fr"
```

容器 900px → 第一欄 600px、第二欄 300px（2:1）

---

## `1fr` 的隱藏行為

`1fr` 實際上等於 **`minmax(auto, 1fr)`**。

`auto` 的意思是：「這欄最小不能比內容還窄」。

所以如果內容很長（例如不換行的日期文字），這欄會被撐大，超過「剩餘空間」的預期寬度，把旁邊的欄位擠歪。

---

## `minmax(min, max)` 是什麼？

設定一個欄位的**最小寬度**和**最大寬度**範圍。

```
minmax(0, 1fr)
```

- **第一個參數 `0`** → 最小寬度 = 0px（允許縮到很小）
- **第二個參數 `1fr`** → 最大寬度 = 剩餘空間的一份

### 對比

| 寫法 | 最小寬度 | 最大寬度 | 內容太長時 |
|------|----------|----------|------------|
| `1fr` | auto（跟著內容走） | 剩餘空間 | 欄位被撐大，推歪旁邊的欄 |
| `minmax(0, 1fr)` | 0px | 剩餘空間 | 欄位不會被撐大，內容溢出 |

### 結論

想要欄寬不被內容撐開 → 用 **`minmax(0, 1fr)`** 取代 `1fr`

---

## 文字溢出處理三件套

用了 `minmax(0, 1fr)` 後，欄位不會被撐大了，但文字會跑出格子外面。
需要這三個屬性配合：

### `whiteSpace="nowrap"`

文字**不換行**，全部擠在同一行。

```
沒有 nowrap → 2025/01/01
               23:59
有 nowrap   → 2025/01/01 23:59（不管多長都一行）
```

### `overflow="hidden"`

超出容器的部分**直接隱藏**（剪掉）。

```
容器 [___________]
文字  很長很長很長很長的文字
顯示 [很長很長很長] ← 後面被剪掉了
```

### `textOverflow="ellipsis"`

被剪掉的地方加上 **`...`** 省略號，讓使用者知道還有更多內容。

```
沒有 ellipsis → [很長很長很長]
有 ellipsis   → [很長很長很...]
```

### 三個要一起用

```jsx
<Text
  whiteSpace="nowrap"       // 不換行
  overflow="hidden"         // 超出隱藏
  textOverflow="ellipsis"   // 顯示 ...
>
  2025/01/01 23:59 ~ 2025/12/31 23:59
</Text>
```

少任何一個都不會正常運作：
- 沒有 `nowrap` → 文字會換行，不會溢出，`ellipsis` 沒用
- 沒有 `overflow="hidden"` → 文字溢出但不隱藏，`ellipsis` 沒用
- 沒有 `textOverflow="ellipsis"` → 文字被剪掉但沒有 `...` 提示

---

## 實際修正

```jsx
// 修正前
templateColumns="150px 100px 60px 60px 60px 80px 1fr 50px"
//                                                 ^^^ 會被撐大

// 修正後
templateColumns="150px 100px 60px 60px 60px 80px minmax(0, 1fr) 50px"
//                                                 ^^^ 不會被撐大
```
