# 速查:「洞(hole)」與「hydration」

> 兩個最常忘、又最基礎的詞。忘了就看這一張。
> 回主筆記 → [tree.md](tree.md)

---

## 🕳️ 洞(hole)

**一句話:洞 = 蓋樹的人說「這一格我不蓋,留空,等別人來填」的空位。**

為什麼有洞?server 蓋樹時碰到「需要會動」的元件(按鈕、表單),它蓋不出來
(因為「會動」要瀏覽器才行),所以留個空位 + 貼便條:「這裡放 LikeButton,postId=42」。

```
server 蓋好的樹:
  <article>
     <h1>標題</h1>            ← 死的內容,server 直接蓋好
     <p>內文...</p>           ← 死的內容,server 直接蓋好
     [ 🕳️ 洞:放 LikeButton,postId=42 ]   ← server 不蓋,留空 + 貼便條
  </article>
```

> 「中間有洞」= 整棵樹大部分 server 蓋好了,只有需要互動的地方挖空,等瀏覽器來補。

## 💧 hydration(水合)

**先記字面意思:hydrate = 加水(像泡麵、乾海綿)。乾的東西加水會「活過來」。**

| 階段 | 比喻 | 實際狀況 |
|---|---|---|
| server 送來的 HTML | 🍜 乾的(乾泡麵) | 畫面馬上看得到、長相完美,但點按鈕**沒反應**(就是一張照片) |
| 載入 JavaScript | 💧 加水 | 把 JS 接到那些已顯示的 HTML 上 |
| hydration 完成 | 🍲 活的 | 按鈕能點、輸入框能打字、畫面會反應 |

**三步驟:**
1. server 送 HTML → 瀏覽器**馬上顯示**(快,但死的)
2. JS 載入 → React 走過現有 HTML,把**事件、state 接上去**
3. 接好(hydrated)→ **能互動了**

> hydration = 把已經顯示的「死」HTML,接上 JavaScript,讓它「活」過來(能點、能反應)。

---

## 兩個合在一起

在「server 擁有樹」的世界,瀏覽器收到東西後做兩件事:

1. **填洞** → 把每個 🕳️ 洞放進真正的互動元件(`LikeButton`)
2. **hydration** → 對填進去的元件「加水」,讓它活過來、能互動

而 server 蓋的死內容(`h1`、`p`)**不需要 hydration** —— 它們本來就只是給你看的,不用互動。
