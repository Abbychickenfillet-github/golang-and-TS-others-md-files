JS 作用域 Scope（Global / Function / Block）必考重點

作用域決定變數「在哪裡可以被訪問，在哪裡會被關在門外」，是面試與實戰超高頻考點。

重點整理

三種作用域對比

1. Global Scope（全域作用域） — 宣告在所有函式或區塊 {} 之外，任何地方都能存取。
必考陷阱：瀏覽器環境下，var 宣告的全域變數會變成 window 的屬性（var a=1 等於 window.a）；但 let／const 的全域變數不會掛到 window。過多全域變數會造成「全域命名空間汙染」。

2. Local / Function Scope（區域 / 函式作用域） — 宣告在函式內部，只能在該函式內存取。var、let、const 在函式內都被限制在此。

3. Block Scope（區塊作用域） — 宣告在任何一對 {}（如 if、for、while）內。只有 let 與 const 支援 Block Scope；var 完全不支援（var 會無視大括號跑到外面）。

作用域類型 適用關鍵字 外部可存取 常見場景
Global var, let, const 是 整個 JS 檔最外層
Function (Local) var, let, const 否 函式內部
Block let, const（var 不支援） 否 if、for 迴圈

🔥 面試必考三大情境題

考點一：var 在 for 迴圈中的 Block Scope 災難（常結合 setTimeout）

for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 1000);
}
// 答案：1 秒後連續印出 3, 3, 3

為什麼？var 沒有 Block Scope，i 是共用變數；setTimeout 執行時迴圈早跑完，i 已是 3。
解法：把 var 改成 let，每次迭代產生獨立的區塊作用域，鎖住當下的 i：

for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 1000);
}
// 答案：印出 0, 1, 2

考點二：範疇鍊(Scope Chain) 與 靜態作用域(Lexical Scope)
JS 採用語法作用域(Lexical Scope)：作用域在程式碼「寫好時」就決定，不是「執行時」。

let name = 'Global';
function foo() { console.log(name); }
function bar() { let name = 'Local'; foo(); }
bar();
// 答案：印出 'Global'，不是 'Local'

foo() 定義時上層作用域就是 Global；內部找不到 name 就往外層找，這個鏈結叫 Scope Chain。

考點三：未宣告變數的「自動全域化」陷阱

function test() { a = 10; }  // 前面沒寫 var/let/const
test();
console.log(a); // 答案：10（不會報錯！）

非嚴格模式下，對未宣告變數賦值，JS 會自動在 Global Scope 建立它。這是不良習慣，開啟 "use strict"; 可避免。
