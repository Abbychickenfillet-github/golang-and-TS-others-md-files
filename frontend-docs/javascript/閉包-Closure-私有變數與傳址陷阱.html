重點整理

一句話定義：閉包是一個內層函式，能「記憶」並存取外層函式的範疇（Scope），即使外層函式已經執行完畢、從 Call Stack 離開了。 在 JS 中閉包不是要特別開啟的功能，而是伴隨函式建立而自然誕生的底層機制。

兩大基石：

  • 詞法範疇 Lexical Scope：變數的存取權限在「程式碼被寫下來的那一刻（宣告位置）」就決定了，而不是函式被執行的那一刻。
  • 垃圾回收 GC：一般函式執行完，局部變數會被釋放；但若該變數還被內層函式引用，瀏覽器就不敢釋放，會繼續留在記憶體（Heap）中。

應用：用閉包做私有變數

function createCounter() {
  let count = 0; // 鎖在 createCounter 範疇內，外部摸不到
  return {
    increment: function() { count++; console.log(count); },
    decrement: function() { count--; console.log(count); }
  };
}
const counter = createCounter();
counter.increment(); // 1
counter.increment(); // 2
// console.log(count); // ReferenceError: count is not defined

createCounter() 執行完回傳物件後，因為回傳的方法仍引用 count，所以 count 活在閉包裡，成為這些方法的專屬私有變數。

經典面試題：var 迴圈 + setTimeout

for (var i = 1; i <= 3; i++) {
  setTimeout(() => { console.log(i); }, 1000);
}

直覺以為 1,2,3；實際是 4, 4, 4。
因為 var 是函式作用域，迴圈裡沒有函式包裹時它幾乎等同全域變數；1 秒後回呼執行時迴圈早跑完，全域 i 已累加到 4，三個閉包指向同一個 i。
修正：把 var 改成 let（區塊作用域），每次迭代建立獨立範疇，三個閉包各自鎖定 1, 2, 3。

魔王題：閉包「漏水」（傳值 vs 傳址）

function createWallet(initialAmount) {
  let money = initialAmount;
  let history = [];
  return {
    checkBalance: () => money,
    getHistory: () => history,         // 直接回傳陣列參照！
    spend: (amount) => { if (amount <= money) { money -= amount; history.push(`Spent ${amount}`); } }
  };
}
const myWallet = createWallet(100);
const record = myWallet.getHistory();  // record 與閉包內 history 指向同一陣列
myWallet.spend(30);                     // money 70, history=['Spent 30']
record.push('Found 100 on the street');// 直接竄改了閉包內部私有狀態！
myWallet.spend(20);                     // money 50
myWallet.checkBalance(); // 50
myWallet.getHistory();   // ['Spent 30','Found 100 on the street','Spent 20']

核心反思：用閉包做私有變數時，若直接回傳物件或陣列，私有性就破功了——外部拿到參照就能繞過你的方法任意改內部資料。
正解：回傳時切斷參照，回傳複本，例如 getHistory: () => [...history]（展開運算子產生新陣列）。

觀念釐清（是非題）

  • return money 會不會讓 money 變成全域變數？ 不會。 money 既不在全域、也沒「回到」函式內部，而是被封裝在專屬 myWallet 的閉包環境裡。return 只是把值複印一份送給外面的變數，口袋裡的本體不會穿透到外面。
  • money 是 checkBalance 的區域變數嗎？ 不是。 money 是 createWallet 函式的區域變數，checkBalance 是透過閉包才能存取到外層 createWallet 裡的 money。（注意：判斷一個變數屬於誰，看的是它宣告在哪個函式，而不是被哪個函式用到。）
  • myWallet 是「函式物件變數」嗎？ 不是。 createWallet 本身才是函式物件（可被呼叫）；myWallet 是 createWallet(100) 回傳的普通物件（Plain Object），myWallet() 會噴 TypeError: myWallet is not a function。物件裡的 spend、checkBalance 因綁在物件上，正名是方法（Method）。

C++ 也有閉包嗎？（對比）

有，C++11 起用 Lambda 表達式引入：[捕捉列表](參數) { 函式本體 }。最大差別：JS 閉包自動發生且靠 GC 保命；C++ 沒有 GC，必須明確寫出要捕捉哪些變數。

特性 JavaScript 閉包 C++ Lambda
觸發方式 自動（內層引用外層即觸發） 必須明寫在捕捉列表 [...]
記憶體管理 靠 GC 自動保留 開發者手動管理，注意 Stack 存活期
底層實現 範疇鏈 + 環境物件 編譯器產生的匿名仿函式（Functor）
  • 傳值捕捉 [count]：建立當下複製數值進閉包，外層銷毀也安全。
  • 引用捕捉 [&money]：像 JS 一樣能改外部變數，但回傳引用捕捉的 Stack 變數會造成懸空引用（Dangling Reference），觸發未定義行為甚至 Crash。
