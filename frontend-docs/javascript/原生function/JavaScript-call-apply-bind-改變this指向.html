JavaScript call / apply / bind（改變 this 指向）

this 綁定規則：預設綁定 vs 隱性綁定（回呼陷阱）

先懂 this 的基本規則：this 的值看「函式怎麼被呼叫」，不是看它寫在哪裡。
setTimeout(function () {
  console.log("我是被偷偷拿出來執行的函式");
  // 這裡的 this 是誰？
}, 1000);

「匿名函式」＝ function () { ... } 整段沒有名字的函式；「它是獨立被呼叫的」的「它」＝這個匿名函式（精確說是它執行時的 call-site 與 this 綁定）。

呼叫方式寫法內部 this
隱性綁定obj.myMethod()指向 obj
預設綁定func()（獨立呼叫）非嚴格模式指向window；嚴格模式 undefined

回呼陷阱：時間到時瀏覽器把匿名函式獨立拉出執行（等同 anonymousFunc()），前面沒有物件點它，this 脫鉤指向 window，而非外層物件。解法：用箭頭函式（this 繼承外層）或 bind 顯性綁定。

重點整理

call、apply、bind 都是用來改變函式執行時 this 指向的方法，讓你靈活控制函式的上下文（context）。

1. call() — 立即執行，參數逐一傳入

立即執行函式並手動指定 this，參數以逗號分隔依序傳入。適合需要立即調用且參數數量明確時。

const person = { name: 'Alice' };
function greet(greeting, punctuation) {
  console.log(`${greeting}, ${this.name}${punctuation}`);
}
greet.call(person, 'Hello', '!'); // Hello, Alice!

2. apply() — 立即執行，參數用陣列

功能與 call 完全相同（立即執行 + 綁定 this），唯一差別：參數以「陣列」傳入。當參數本來就在陣列中、或參數數量不固定時更方便。

const person = { name: 'Bob' };
function introduce(job, hobby) {
  console.log(`${this.name} is a ${job} and likes ${hobby}.`);
}
const args = ['Engineer', 'coding'];
introduce.apply(person, args); // Bob is a Engineer and likes coding.

3. bind() — 不執行，回傳綁定後的新函式

與前兩者不同：bind 不會立即執行，而是回傳一個把 this 永久綁定到指定物件的新函式。適合事件監聽或需要延遲執行的場景。

const user = { name: 'Charlie' };
function showName() { console.log(this.name); }
const boundFunc = showName.bind(user);
boundFunc(); // Charlie

總結比較表

方法 是否立即執行 參數傳遞方式 主要用途
call 是 逐一傳入（參數清單） 借用方法、立即改變上下文
apply 是 陣列傳入 處理陣列參數、數學運算（如 Math.max）
bind 否 逐一傳入 建立新的回呼函式、預設參數

快問快答（自我測驗）

  • call 可以延遲執行嗎？ → 不行，它會直接調用函式。
  • 什麼時候最適合用 apply？ → 當參數已在陣列中，或要用 Math.max 等需要多個參數的內建函式時。
  • bind 綁定後的函式 this 還能再變嗎？ → 不能，bind 建立的是強連結（永久綁定）。
