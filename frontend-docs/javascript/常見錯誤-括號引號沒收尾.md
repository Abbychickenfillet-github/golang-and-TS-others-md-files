# 常見錯誤：括號 / 引號沒收尾 → unexpected end of input

> 來源練習：`JavaScript-practicing/while-loop.html`（map 練習時誤打）
> 相關：[[new-Date-日期物件重點]]（報錯位置≠報錯原因）、[[陣列遍歷-forEach與callback]]

## 症狀

```
Uncaught SyntaxError: unexpected end of input  （錯誤行號常指向檔案最後一行）
```

## 真正原因：少了一個收尾符號

有個 `` ` ``、`)`、`}`、`]` 沒有關起來，解析器讀到**檔案結尾**還在等它收尾。

實際踩過的例子（多打一個反引號）：
```js
let labels = people.map(p => `${p.name} is ${p.age} years old``)
//                                                          ↑↑ 兩個反引號！
```
- 第 1 個反引號正常收尾字串。
- 第 2 個反引號**又開了一個新樣板字串**，直到檔尾都沒關 → unexpected end of input。
- 修正：刪掉多餘的那個反引號。

## 關鍵心法（和日期筆記同一條原則）

> **報錯位置 ≠ 報錯原因。** 少收尾符號時，錯誤幾乎都報在「最後一行」（如 `</script>`），
> 真兇在上面某一行沒關起來的符號 → **往上找沒成對的 `` ` `` / `)` / `}` / `]`**。

## 怎麼快速抓

1. 看編輯器的**括號配對highlight**（游標移到 `)` 會highlight對應的 `(`）。
2. 把可疑那行的 `` ` ``、`(`、`{` 一個一個數，確認**成雙成對**。
3. 善用編輯器自動縮排：縮排突然怪掉的地方，通常就是沒收尾的位置。
