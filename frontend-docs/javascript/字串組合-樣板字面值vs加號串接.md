# 字串組合：樣板字面值 vs 加號串接

> 來源練習：`JavaScript-practicing/while-loop.html`（for...in 印出物件）
> 相關：[[loops-and-increment-operators]]、[[new-Date-日期物件重點]]

## 一句話

`+` 串接和 `` `${}` `` 樣板字面值是「**組字串的兩種寫法**」，技術上**等價、可互換**；但「結果相不相等」要看**拼進去的內容是否一樣**。

---

## 兩種寫法互相對照（內容相同才叫等價）

| 加號串接 | 等價的樣板字面值 |
|---|---|
| `objName + "." + i + " = " + obj[i] + "\n"` | `` `${objName}.${i} = ${obj[i]}\n` `` |
| `i + ":" + Joe[i]` | `` `${i}:${Joe[i]}` `` |

上下兩種輸出完全一樣 → 這才是「等價」。

## 常見誤會：不同例子拿來比

```js
result += objName + "." + i + " = " + obj[i] + "\n"  // → "Joe.name = 阿弘弘\n"
`${i}:${Joe[i]}`                                      // → "name:阿弘弘"
```

這兩個**不等價**，不是因為寫法不同，而是**內容不同**：

| | 物件名前綴 | 分隔符 | 結尾換行 |
|---|---|---|---|
| 第一行 | 有 `Joe.` | ` = ` | 有 `\n` |
| 第二行 | 無 | `:` | 無 |

> 重點：寫法（語法）等價 ≠ 結果（輸出字串）相等。輸出相不相等只看「文字＋變數拼出來的內容」。

---

## 差別與建議

| | `+` 串接 | `` `${}` `` 樣板字面值 |
|---|---|---|
| 可讀性 | 引號和 `+` 多，易漏 | 變數直接塞進文字，清楚 |
| 換行 | 要寫 `"\n"` | 可寫 `\n`，也可在反引號內直接換行 |
| 慣例 | 較舊 | **主流、推薦** |

```js
// 反引號可直接多行
`第一行
第二行`
```

---

## 記憶

- 兩種寫法是「同一件事的兩種寫法」，功能等價。
- 想說「結果相等」→ 內容（前綴、分隔符、換行）要一模一樣。
- 現代寫法優先用 `` `${}` ``。

---

## 英文術語對照（容易翻錯）

| 中文 | 正確英文 | 易錯點 |
|------|---------|--------|
| 組字串 | **building a string / string building** | 別用 `concatenation`，那只是其中一種 |
| 等價 | **equivalent** | 不是 `equal`；`equal`=數值相等，`equivalent`=作用相同 |
| 串接（用 `+`） | concatenation | 是「組字串」的子集 |
| 樣板字面值 | template literal | 用反引號 `` ` `` 和 `${}` |

「組字串的技術等價」→ **The string-building techniques are equivalent.**

```text
組字串 building a string ← 大概念
 ├── 串接 concatenation       → "a" + b + "c"
 └── 樣板字面值 template literal → `a${b}c`
```
