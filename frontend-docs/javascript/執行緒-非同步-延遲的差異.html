執行緒 vs 非同步 vs 延遲

重點整理

三者都跟「時間、等待」有關，但本質完全不同。用飲料店比喻最好懂：

1. 執行緒 Thread —「做事的人」

作業系統能進行運算排程的最小單位，就是「實際動手做事的員工」。

  • 單執行緒：只有一個員工，做完一杯才能接下一單；遇到要現榨很久的果汁，整條人龍就卡住。這是 JavaScript 的本質。
  • 多執行緒：請了四五個員工分工，某人卡住其他人仍能服務別的客人。

2. 非同步 Asynchronous —「做事的方法（切換任務）」

一種程式設計架構模式，任務執行順序不必死板地等前一件完全做完。

  • 同步（Synchronous）：員工點完餐死站在廚房等茶煮好，不收錢不接單，整店「卡死（Blocking）」。
  • 非同步（Asynchronous）：把茶葉丟進熱水（發出請求）後不等它好，立刻轉身服務下一位；計時器逼逼叫（完成通知）才回來把茶倒出。

[!note] 核心：JavaScript 雖然只有「一個員工（單執行緒）」，但它是時間管理大師，靠「非同步」在等待空檔塞別的事做，所以網頁才不會動不動卡死。

3. 延遲 Latency / Delay —「等待的時間」

不是人也不是方法，而是一個物理指標：從發出指令到看到結果中間消耗的時間差。

  • 飲料店：點完餐到拿到飲料隔了 10 分鐘，這 10 分鐘就是延遲。
  • 程式中：
  • 網路延遲（Network Latency）：fetch 請求到 API 伺服器、資料傳回花了 200ms。
  • 人為延遲（Delay）：你刻意寫 setTimeout(() => {}, 3000) 製造的等待。

一句話總結：執行緒是「誰做」、非同步是「怎麼做」、延遲是「等多久」。
