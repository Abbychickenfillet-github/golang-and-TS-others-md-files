# 迴圈與加值運算子

> 來源練習：`JavaScript-practicing/while-loop.html`

## 加值：`++` 與 `+=`

### 一句話

`++` 只能讓變數 **+1**；要每次 **+2、+3…** 請用 `i += 2`，**不能**寫 `i++2`（語法錯誤）。

### 你遇到的寫法

```js
// ❌ 語法錯誤
for (let i = 5; i < 80; i = i++2) { }

// ✅ 每次加 2
for (let i = 5; i < 80; i += 2) {
  console.log(i) // 5, 7, 9, 11 …
}
```

### 對照表

| 寫法 | 意思 | 常見用途 |
|------|------|----------|
| `i++` | `i` 加 1 | `while` 裡更新計數 |
| `++i` | `i` 加 1（先加再用，較少在迴圈更新用） | 較少見 |
| `i += 2` | `i` 加 2 | `for` 第三段、跳步迴圈 |
| `i = i + 2` | 同上，較長 | 語意最直白 |

**記憶：** `++` 是固定「加 1」的符號；要加其他數字用 `+=`。

## 四種常見寫法（含 switch 說明）

前三種是**迴圈**（會重複執行）；`switch` 是**依值分支**，不是迴圈，但寫程式時常一起學，所以放在這裡對照。

### 1. `while` — 先判斷，再執行

條件一開始就是 `false` 的話，**一次都不跑**。

```js
let n = 0
while (n < 3) {
  console.log(n) // 0, 1, 2
  n++
}
```

### 2. `do...while` — 先執行，再判斷

**至少會跑一次**，再檢查條件。

```js
let n = 0
do {
  console.log(n) // 0, 1, 2
  n++
} while (n < 3)
```

| | `while` | `do...while` |
|---|---------|----------------|
| 判斷時機 | 進迴圈前 | 每輪結束後 |
| 最少執行次數 | 0 次 | 1 次 |

#### 為什麼結束值會「多衝一格」？（常見困惑）

```js
let a = 3, b = 2
do {
  a++; b++
  console.log(a)
} while (a < 10)
// 結束時 a = 10，不是 9！
```

因為 `do...while` 是「**先做事、最後才檢查**」，而 `a++` 在檢查之前就先加了：

| `a++` 後 a = | 檢查 `a < 10` | 結果 |
|---|---|---|
| 9 | 9 < 10 ✅ true | **還要再跑一次** |
| 10 | 10 < 10 ❌ false | 停止 |

- a=9 時條件還成立 → 它沒理由停 → 再跑一輪又把 a 加到 10。
- **記憶：迴圈的結束值＝「讓條件第一次變 false」的那個數。** 條件 `a < 10` → 結束在 10。
- 想停在 9 就把條件改成 `a < 9`。

> 補充：`let a = 3, b = 2`。若寫成 `let a = 3; b = 2`，第二個 `b` 沒有 `let` 會變隱性全域變數（嚴格模式報錯）。

#### `do...while` 多用在哪？

精神是「**至少先做一次，再決定要不要繼續**」，常用在「**條件得等 body 跑過一次後才知道**」：

1. **輸入驗證 / 重複詢問**（最典型）：先問一次，不合法就再問。
   ```js
   let age
   do { age = Number(prompt("年齡(1~120):")) }
   while (isNaN(age) || age < 1 || age > 120)
   ```
2. **選單迴圈**：選單先顯示一次，選「離開」才停。
3. **重試機制 retry**：操作至少試一次，失敗才重試（常配最大次數）。
4. **讀資料到結束標記**：先讀一筆，再判斷是不是結束符號。

判斷口訣：

| 問自己 | 選 |
|---|---|
| 不管怎樣**至少要先做一次**？ | `do...while` |
| 一開始條件就可能 false、**可能一次都不做**？ | `while` |
| **條件依賴 body 裡產生的東西**（輸入/回傳/亂數）？ | `do...while` |

> 老實說：實務前端 `do...while` 用得不多，`for`/`while`/`for...of` 更常見，也常用 `while(true){ … if(條件) break }` 取代。但「輸入驗證」「重試」這兩個情境它最貼切。

### 3. `for` — 計數型迴圈最常用

初始化、條件、更新寫在同一行，適合「跑固定次數」或「有索引」。

```js
// 每次 i 加 1
for (let i = 0; i < 3; i++) {
  console.log(i) // 0, 1, 2
}

// 每次 i 加 2（對應你的練習）
for (let i = 5; i < 10; i += 2) {
  console.log(i) // 5, 7, 9
}
```

`for` 的三段：

```text
for (初始化; 條件; 更新) { 區塊 }
     ↑         ↑      ↑
   只跑一次   每輪前檢查  每輪結束後執行
```

### 4. `switch` — 依值選分支（不是迴圈）

依變數的值執行**其中一個**分支，不會自動重複。

```js
const day = 3
switch (day) {
  case 1:
    console.log('Mon')
    break
  case 2:
    console.log('Tue')
    break
  case 3:
    console.log('Wed')
    break
  default:
    console.log('Other')
}
```

- 每個 `case` 結尾通常要 `break`，否則會**穿透**到下一個 `case`。
- 若只是比大小、區間，用 `if / else if` 較常見；`switch` 適合「等於某幾個固定值」。

---

## 什麼時候用哪一種？

| 情境 | 建議 |
|------|------|
| 次數明確、有計數變數 | `for` |
| 條件簡單、不一定知道跑幾次 | `while` |
| 至少要執行一次（例如選單、讀取直到有效輸入） | `do...while` |
| 依固定幾個值做不同動作 | `switch`（或 `if`） |

## 和你的練習檔對照

```js
// while：條件成立就繼續
let variable_abby = 0
while (variable_abby < 10) {
  variable_abby++
  console.log(variable_abby)
}

// for：第三段用 i += 2，不要用 i++2
for (let i = 5; i < 80; i += 2) {
  console.log(i)
}
```
