重點整理

核心觀念：Markdown 不能「直接」被渲染成 DOM

瀏覽器原生只看得懂 HTML、CSS、JavaScript 三種東西。把一個 .md 檔案直接丟進瀏覽器，它只會當成純文字（Plain Text）顯示，不會自動把 # 變成 <h1>、把 **字** 變成 <strong>。

所以一定要經過一個「轉換」步驟，先把 Markdown 變成 HTML 字串，瀏覽器才能解析成 DOM。

兩種轉換做法

做法 何時轉換 代表工具 特點
前端即時轉換 (Runtime) 瀏覽器執行時 marked.js 動態、彈性，但要在用戶端跑 JS
靜態預編譯 (Build-time) 專案打包時 Next.js / Gatsby / Astro 使用者下載到的已是現成 DOM，最快

做法一：前端即時轉換（Runtime）

利用 JavaScript 在瀏覽器執行時動態把 Markdown 轉成 HTML，再塞進 DOM：

// 使用 marked.js 範例
import { marked } from 'marked';

const markdownString = "# 這是標題";
// 1. 轉成 HTML 字串
const htmlString = marked.parse(markdownString);
// 2. 塞進 DOM，瀏覽器才會渲染
document.getElementById('content').innerHTML = htmlString;

關鍵兩步：marked.parse() 轉成 HTML 字串 →（再）用 innerHTML 塞進 DOM。

[!warning] 安全提醒
用 innerHTML 注入由 Markdown 轉出的 HTML，若來源不可信會有 XSS 風險，正式專案通常會搭配 DOMPurify 之類的工具先做淨化（sanitize）。

做法二：靜態預編譯（Build-time）

現代部落格框架（Next.js、Gatsby、Astro）最常用。在專案打包（Build）的時候，系統就先把所有 .md 檔案轉成 .html。使用者瀏覽時下載到的已經是現成的 DOM 結構，不需要在瀏覽器再跑轉換，載入更快、對 SEO 更友善。
