# Test Plan: 票券通行碼 (access_code)

## 測試流程

### 1. Dashboard 設定通行碼
- [ ] 新增票券（Modal）→ 進階設定 → 輸入通行碼 `VIP2026` → 建立成功
- [ ] 新增票券（/tickets/create 頁面）→ 同上
- [ ] 編輯票券 → 修改通行碼 → 儲存成功
- [ ] 編輯票券 → 清空通行碼 → 儲存成功（變回無限制）
- [ ] 票券詳情頁 → 有通行碼時顯示 Badge，無通行碼時不顯示

### 2. Official Website 消費者端購票驗證
- [ ] 有 access_code 的票種 → 顯示鎖頭圖示 + 輸入框
- [ ] 輸入正確通行碼 → 解鎖，可選數量 + 加入購物車
- [ ] 輸入錯誤通行碼 → 顯示錯誤提示，不能購買
- [ ] 大小寫不敏感 → `vip2026` 和 `VIP2026` 都能通過
- [ ] 沒有 access_code 的票種 → 正常顯示，不需要輸入碼
- [ ] 清空已輸入的通行碼 → 重新鎖定

### 3. 後端 API 驗證
- [ ] `POST /orders/:id/tickets` 帶正確 access_code → 成功加入
- [ ] `POST /orders/:id/tickets` 帶錯誤 access_code → 回傳錯誤
- [ ] `POST /orders/:id/tickets` 不帶 access_code（但票券有設定）→ 回傳錯誤
- [ ] `GET /tickets/event/:id/available` → 回傳 `has_access_code: true`，不暴露實際碼
- [ ] `GET /tickets/:id`（Dashboard API）→ 回傳完整 `access_code` 值

### 4. 邊界測試
- [ ] 通行碼含空白 → 前後 trim 後比對
- [ ] 通行碼為空字串 → 視為無通行碼
- [ ] 同一活動多種票種，部分有通行碼部分沒有 → 互不影響

## 測試環境
- Dashboard: http://localhost:5003
- Official Website: http://localhost:3000
- Backend: http://localhost:8080
- DB: staging (hnd1.clusters.zeabur.com:32195)

## 測試活動
使用既有測試活動，新增一張有通行碼的票券進行測試
