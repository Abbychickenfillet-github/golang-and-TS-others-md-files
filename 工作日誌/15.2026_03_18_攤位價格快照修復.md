# 2026-03-18 攤位價格快照修復

## 一、Stage 攤位價格沒記錄到

### Abby 說的話
> 「stage 這邊發現攤位價格沒有記錄到」
> 「他這邊就是要記錄 display_price 了」
> 「怎麼還一定要有 override_booth_price? 沒有設定也沒關係啊」
> 「現在是只有攤位價格不見了」

### 問題分析
前端選攤位時，後端 API 已經計算好 `display_price`（含三層定價邏輯：override > 時間區間定價 > default），前端確認頁也用 `display_price` 顯示金額。

但付款頁呼叫 `registration API` 建立訂單時，`BoothRegistrationItem` **只傳了 `booth_id`**，沒有帶 `display_price`。後端收到後用 `calculateBoothPrice()` 重新算，而這個函式只查 `override_booth_price` → `EventBoothTypePricing`，沒有 fallback 到 `default_price`，導致 `booth_price_snapshot` 記錄為 0。

## 二、修復內容

### Claude 做了什麼

#### 後端（futuresign_backend）
1. **`internal/dto/registration.go`** — `BoothRegistrationItem` 新增 `DisplayPrice *decimal.Decimal`
2. **`internal/service/registration_service.go`** — `BoothRegistrationItem` 新增 `DisplayPrice`，計算攤位費用時優先用前端傳入的 `display_price`，沒傳才 fallback 到原本的 `calculateBoothPrice()`
3. **`internal/handler/registration_handler.go`** — DTO 轉 service struct 時帶上 `DisplayPrice`

#### 前端（official_website）
1. **`src/lib/hooks/useOrderDraft.ts`** — `BoothDraft` 新增 `displayPrice?: number`
2. **`src/pages/EventRegisterBoothPage.tsx`** — 存 draft 時從 booth 資料取 `display_price` 存入
3. **`src/lib/api/registration.ts`** — `BoothRegistrationItem` 新增 `display_price?: number`
4. **`src/pages/EventRegisterPaymentPage.tsx`** — 兩處組裝 registration request 時帶上 `display_price`
