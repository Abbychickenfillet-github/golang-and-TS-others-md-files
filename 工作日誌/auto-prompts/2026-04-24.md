## 1. 關於 `$APPDATA` 為什麼叫這個名字

`APPDATA` 不是 bash 自己的變數，而是 **Windows 系統內建的環境變數**。

| 變數 | 指向 | 用途 |
|---|---|---|
| `%APPDATA%` | `C:\Users\<你>\AppData\Roaming` | 應用程式設定（會跟著網域帳號漫遊） |
| `%LOCALAPPDATA%` | `C:\Users\<你>\AppData\Local` | 本機才用的快取/大檔（不漫遊） |
| `%PROGRAMFILES%` | `C:\Program Files` | 程式安裝目錄 |
| `%USERPROFILE%` | `C:\Users\<你>` | 個人資料夾根目錄 |

Microsoft 從 Windows NT 時代就定下這命名規則 — `APPDATA` 是「**App**lication **Data**」的縮寫。

Git Bash / MSYS2 啟動時會把所有 Windows 環境變數**原封不動匯入** shell，所以你可以直接用 `$APPDATA`（bash 變數語法）去存取，效果跟 `%APPDATA%`（cmd 語法）一樣。例如我剛剛跑：
```bash
ls "$APPDATA/Claude"
# 等於 cmd 的：dir "%APPDATA%\Claude"
```
