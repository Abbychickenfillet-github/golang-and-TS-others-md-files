## 14:52 — 那個marketplace是指誰的marketplace? 譬如有chrome marketplace

**Claude:**

好問題，這裡的 **marketplace 是 Claude Code 自己的 plugin 系統**，不是 Chrome 那種第三方商店。

## Claude Code 的 marketplace 架構

Anthropic 在 Claude Code 裡內建了一套 **plugin marketplace 機制**，但跟 Chrome Web Store 不一樣的是 —— 它**沒有單一的中央商店**，而是**任何人都可以開自己的 marketplace**。

## 有哪幾種 marketplace？

### 1. 官方 marketplace（Anthropic 營運）
- 名字：`claude-plugins-official`
- 由 Anthropic 自己維護、審核
- 類似 Apple App Store（官方把關）
- 安裝指令：
  ```
  /plugin install superpowers@claude-plugins-official
  ```

### 2. 第三方 marketplace（任何人都能開）
- 例如 `obra/superpowers-marketplace`（Jesse Vincent 個人開的）
- 本質上就是一個 **GitHub repo**，裡面放一個 `marketplace.json` 清單
- 你要先「註冊」這個 marketplace，然後才能從裡面裝東西：
  ```
  /plugin marketplace add obra/superpowers-marketplace
  /plugin install superpowers@superpowers-marketplace
  ```

## 類比

| Chrome | Claude Code |
|---|---|
| Chrome Web Store（唯一官方） | `claude-plugins-official`（官方，但只是其中一個） |
| ❌ 不能自己開商店 | ✅ 任何人都能開 marketplace（只要有 GitHub repo） |
| 直接點「安裝」 | 用 `/plugin install 插件名@marketplace名` |

## 為什麼會這樣設計？

因為 Claude Code 走**去中心化**路線——官方 marketplace 跑得比較慢（要審核），但開發者（像 obra）可以自己架一個 marketplace，第一時間推新功能。你信任誰就加誰的 marketplace。

所以 `@claude-plugins-official` 跟 `@superpowers-marketplace` 後面那個 `@xxx` 就是在告訴 Claude Code「要從哪一個 marketplace 裝這個 plugin」。

## 17:40 — 啟動程式                   │ C:\coding\voice-input-method\.venv\Scripts\voice-input.exe   為何放在.venv?

**Claude:**

因為 `.venv` 是這個專案的**專屬 Python 環境**，裡面裝了 40+ 個套件（PySide6、pynput、funasr、torch 等）。`voice-input.exe` 需要這些套件才能跑，所以必須跟它們綁在一起。

## 白話解釋

`voice-input.exe` 其實**不是一個完整的程式**，它是個「啟動器」——大約 5KB 的小殼，裡面寫著：

> 「請用 `.venv\Scripts\python.exe` 去執行 `voice_input_method.__main__:main`」

執行時發生的事：
```
voice-input.exe
  └─> 呼叫 .venv\Scripts\python.exe  （隔壁的 Python）
        └─> 載入 .venv\Lib\site-packages\  （PySide6、pynput、funasr 等）
              └─> 執行真正的 GUI 程式
```

## 為什麼不放外面？

如果把 `voice-input.exe` 搬到桌面，它會找不到「隔壁」的 Python 和套件，就壞了。**它是「地圖」，不是「藏寶箱」**——離開 .venv 就失去意義。

## 這樣做的好處

- 不汙染系統 Python（你電腦另外那支 `C:\Users\User\AppData\Local\Programs\Python\Python313\` 完全沒被動到）
- 哪天不要了，整個 `C:\coding\voice-input-method\` 一刪就乾淨
- 多個 Python 專案互不打架（A 用 torch 2.0、B 用 torch 2.11 可以並存）

**桌面捷徑的正確做法**是做一個 .lnk 指向這個路徑，而不是搬檔案。要我幫你做嗎？
