## 三、踩坑 — Tabs 結構壞掉導致無資料

### Abby 說的話
> 「你剛剛打過 API 嗎？為什麼攤位與地圖管理裡面沒資料」

### 問題原因
把 `<TabList>` 包在 `<Flex>` 裡面（想讓 MapSelector 跟 Tab 並排），破壞了 Chakra UI v2 `<Tabs>` 的結構。`Tabs` 需要 `TabList` 是**直接子元素**才能正確註冊 tab。

### 修正方式
把 MapSelector 移回標題列（`<Heading>` 旁邊），`TabList` 恢復為 `Tabs` 的直接子元素。

### 教訓
Chakra UI v2 的 `Tabs` 是 compound component，子元素順序和層級有嚴格要求：
```
<Tabs>           ← 容器
  <TabList>      ← 必須是 Tabs 的直接子元素
    <Tab />
  </TabList>
  <TabPanels>    ← 必須是 Tabs 的直接子元素
    <TabPanel />
  </TabPanels>
</Tabs>
```

### `detail=true` 是什麼？

後端 `GET /api/v1/order-items/` 有兩種模式：

| 參數 | 回傳 DTO | 差別 |
|------|---------|------|
| 不加 `detail` | `OrderItemPublic` | 只有 `buyer_member_id`（UUID），沒有名字 |
| `detail=true` | `OrderItemDetailPublic` | JOIN member 表，帶回 `buyer_member_name`、`buyer_member_email`、`holder_member_name` 等 |

簡單說：基本版只存 ID（省資源），detail 版會多查一次 member 表把名字帶出來。前端購買者列表需要顯示名字，所以一定要加 `detail=true`。

### `{count, data}` vs 陣列

```js
// 陣列 — 直接是 list，可以 .find() .filter() .map()
[{ id: "1", name: "票A" }, { id: "2", name: "票B" }]

// 物件 — 包了一層（常用於分頁，多帶總數），要先 .data 取出陣列
{ count: 50, data: [{ id: "1", name: "票A" }, { id: "2", name: "票B" }] }
```

對物件直接 `.find()` 會噴 `is not a function`，因為 `.find()` 是陣列方法，物件沒有。

### 退款流程設計

**核心原則**：主辦方 3 秒暫停販售，退款之後慢慢來。

```
正常販售 (active)
    │
    ├── [緊急暫停] → cancelled（改狀態，3 秒操作）
    │       ├── 前台立即停售
    │       ├── 之後在購買者列表逐筆退款
    │       └── 全部退完(sold_count=0)後可刪除
    │
    └── [售完] → sold_out（系統自動）
```

**刪除規則**：`sold_count > 0` → 禁刪，`sold_count === 0` → 可刪（不管活動有沒有結束）

**退款追蹤架構**：
- `tickets.refunded_count` — 累計退款計數器（只增不減）
- `tickets.sold_count` — 退款後會減少（歸零 = 全數退完）
- `refund_records` 表 — 完整退款紀錄（關聯 Order，含 ECPay 金流、審核軌跡）

2. 修改 `TicketRefundModal.tsx`：
   - 點「確認退款」按鈕後，先跳出 SweetAlert2 二次確認彈窗
   - 彈窗內容包含：訂單編號、購買者、訂單金額、票券數量、付款方式（線上/線下）、退款原因

## 十一、票券管理版面改卡片式

### Abby 說的話

### 問題分析
原本每張票券用 10 欄 Grid 排成一行：
```
30px 40px 150px 100px 60px 60px 60px 80px minmax(0,1fr) 50px
```
加上 gap，最少需要 ~750px 不含銷售期間欄。桌機板都擠不下。

### Claude 做了什麼
將 Accordion 行式佈局改為**卡片式佈局**：

**改前**：10 欄水平 Grid（一行塞完所有資訊）
**改後**：每張票券是一張 `<Card>`，資訊分兩行：
- **第一行**（AccordionButton）：拖曳把手 + #排序 + 票券名稱 + 狀態 Badge + 操作按鈕
