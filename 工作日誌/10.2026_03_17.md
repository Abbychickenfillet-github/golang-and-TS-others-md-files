### 1. 訂單展開顯示組成明細 + 訂單編號超連結
- `OrderAccordionItem` 展開時 lazy load 訂單摘要（`getOrderSummary`）

- 轉贈狀態標示（`buyer_member_id !== holder_member_id` → 紫色「已轉贈」）
- 訂單編號改為可點擊超連結，導航到 `/orders/$orderId` 細節頁

### 2. 修復訂單細節頁無法顯示（Outlet 問題）
- `orders.tsx` 是 `$orderId.tsx` 的 parent route，但沒有 `<Outlet />`
- 導航到 `/orders/:id` 時 URL 有變但頁面不動
- 新增 `OrdersLayout`，用 `useMatches()` 偵測子路由並渲染 `<Outlet />`
- 參考 `events.tsx` 和 `booth-products.tsx` 的做法

### 3. 付款方式顯示（payment_method）
- `order` 資料表新增 `payment_method` 欄位（`VARCHAR(30)`）
- 後端 `OrderSummaryResponse` + `OrderPublic` 回傳 `payment_method`
- 歷史訂單自動推斷：有 `ecpay_trade_no` → ecpay，已付款但無交易號 → offline_transaction
- 前端 accordion 展開面板 + 細節頁顯示付款方式 Badge

### 4. 新增 CompanyMemberSelector 元件
- 選了買方公司後，自動查 `member_company` 關聯取得該公司 approved 員工
- Cross-reference members API 顯示員工名稱/email/電話/職位
- 切換公司自動清掉已選員工
- **Bug 修正**：API 路徑 `/api/members/` → `/api/v1/members/`
- **Bug 修正**：members API 回傳是物件 `{ data: [...] }` 不是陣列，要取 `.data`

### 5. 攤位價格修正
- 後端 `GetBooths` 列表 API 新增 `display_price`（計算後的顯示價格）
- 前端改用 `display_price`（含時間區間定價），不再只用 `override_booth_price`（通常是 0）
- 選完攤位後顯示系統定價 + 可手動調整實收金額

### 6. 修復表單無法送出（isRequired 衝突）
- **問題**：`<FormControl isRequired>` 包 `EntitySearchSelector` 時，選了值但瀏覽器顯示「請填寫這個欄位」
- **原因**：`isRequired` 把 HTML `required` 傳到搜尋框 input，但選完後 `searchText` 清空，瀏覽器看到空的 required input 就擋住
- **解法**：移除 `isRequired`，改手動標示 `*` + onSubmit toast 驗證
- **文件**：`frontend-docs/chakra-isRequired-vs-custom-selector.md`

### 7. UI 文字修正
- 「聯絡人會員」→「買方公司員工」（FormLabel、placeholder、提示、錯誤訊息）
