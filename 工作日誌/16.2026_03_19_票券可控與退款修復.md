4. 前端 TicketRefundModal 有傳 `process_ecpay` query param，但後端目前沒處理這個參數

## 庫存計算說明
- `sold_count` = 目前佔用的庫存數（不是歷史總銷售）
- 退款時：`sold_count -= N`, `refunded_count += N`
- `available_count = quantity - sold_count`
- 歷史總銷售 = `sold_count + refunded_count`
- `refunded_count` 是後來才加的欄位，歷史退款資料需跑 migration 回填
