# 2026-03-17 Logger 改善計畫

## 問題描述
老闆反饋：後端的 logger 全部都是 SQL 語法，沒有寫哪張表。
出問題時看 log 只看到 GORM 的 raw error，無法快速定位是哪張資料表出錯。

## 三大問題

### 問題 1：缺少 `table` 欄位
```go
// 現狀：看不出是哪張表
logger.Error("查詢失敗", "error", err, "order_id", orderID)

// 改善：加 table 欄位
logger.Error("查詢失敗", "table", "orders", "error", err, "order_id", orderID)
```

### 問題 2：Logger 呼叫風格不一致（三種寫法混用）
```go
// 寫法 A：用 wrapper function — 大部分 service 用這個
logger.Error("xxx", "key", value)

// 寫法 B：直接用 Logger 實例 + slog 型別 — scheduler、company、event、map 用
logger.Logger.Info("xxx", slog.String("key", "value"))

// 寫法 C：instance logger — payment_service、部分 handler 用
s.logger.Info("xxx", "key", value)
```

### 問題 3：company_verification_service.go 用英文訊息
其他所有 service 都用繁體中文，唯獨這個用英文。

## Logger 風格不一致清單

| 風格 | 使用的檔案 | 問題 |
|------|-----------|------|
| `logger.Error/Info/Warn("msg", "k", v)` | 大部分 service | 標準寫法 |
| `logger.Logger.Info("msg", slog.String(...))` | `order_cleanup.go`、`company_service.go`、`event_service.go`、`map_service.go` | 應改用 wrapper |
| `s.logger.Info("msg", "k", v)` | `payment_service.go` | instance logger，可保留但要統一 |
| `h.logger.Info("msg", "k", v)` | 部分 handler | instance logger |

---

## 執行優先順序

| 優先級 | 檔案群組 | 原因 |
|--------|----------|------|
| P0 | `order_cleanup.go` | 排程自動執行，出錯最難追 |
| P0 | `payment_service.go` | 金流相關，出錯影響最大 |
| P1 | `order_service.go`、`order_item_service.go` | 訂單核心邏輯 |
| P1 | `refund_service.go` | 退款邏輯 |
| P2 | `booth_service.go`、`event_booth_type_service.go` | 攤位管理 |
| P2 | `event_coupon_service.go` | 消費券 |
| P3 | 其餘 service + handler | 依序處理 |
