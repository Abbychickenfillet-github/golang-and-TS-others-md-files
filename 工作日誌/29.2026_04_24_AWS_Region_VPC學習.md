# 區域 1：知識點整理

## 1. AWS 基礎設施的三層結構

```
Region（區域）
  └── Availability Zone / AZ（可用區）
        └── Logical Data Center（邏輯資料中心）
```

### 各層定義

| 層級 | 定義 | 舉例 |
|---|---|---|
| **Region** | AWS 在世界各地建立基礎設施的「地理區域」 | ap-northeast-1（東京）、us-east-1（北維吉尼亞）|
| **AZ** | 一個 Region 裡面**彼此實體隔離**的多個區塊 | ap-northeast-1a、1c、1d |
| **資料中心** | AZ 內部由多個邏輯資料中心組成 | 使用者看不到、AWS 內部分組 |

**重點**：**Region 和 Region 之間隔很遠（跨國）、AZ 和 AZ 之間實體隔離（不同大樓、不同電源、不同網路）**——這個「實體隔離」是後面 HA 原理的關鍵。

---

## 2. VPC — 虛擬的網路區域

### 定義

**VPC (Virtual Private Cloud)** = 你在 AWS 上**自己切出的一個虛擬網路空間**。

- **範圍**：VPC 是**綁在一個 Region 裡面**的（不能跨 Region）
- **作用**：管理放在你雲端的 EC2 等資源的網路流通
- **功能**：決定「要接在什麼網路裡」、「誰能連得到」、「對外網路是否開放」

### VPC 跟 EC2 的關係

```
VPC（虛擬網路圍籬）
  └── Subnet（子網，VPC 裡面更小的網路區塊）
        └── EC2（被放在 subnet 裡面的機器）
```

EC2 不會直接放在 VPC——**要先切 Subnet，EC2 才放在 Subnet 裡面**。

---

## 3. Subnet — VPC 裡面更小的虛擬網路區域

### 定義

**Subnet** = 在 VPC 內再切出來的**子網路**，是 EC2 真正被放進去的地方。

### Subnet 與 AZ 的關係（⚠️ 這裡有修正）

| 原本理解 | ❌ | ✅ 正解 |
|---|---|---|
| 「每一個 Subnet 會對應到各自 AZ」| 看起來像一對一 | 其實是**多對一**：**一個 AZ 可以有很多 Subnet，但一個 Subnet 只屬於一個 AZ** |

```
Region: ap-northeast-1
  │
  ├── AZ: ap-northeast-1a
  │     ├── Subnet-A1（例如放 web server）
  │     └── Subnet-A2（例如放 database）
  │
  ├── AZ: ap-northeast-1c
  │     ├── Subnet-C1
  │     └── Subnet-C2
  │
  └── AZ: ap-northeast-1d
        └── Subnet-D1
```

### 為什麼要切這麼多層？

- **Region → AZ**：地理/實體隔離（容災）
- **AZ → Subnet**：邏輯切分，同一個 AZ 裡再分「public subnet（對外）」、「private subnet（內部）」
- **Subnet → EC2**：實際機器落腳處，由 Subnet 決定它能被誰連到、能連到哪

---

## 4. EC2 — 「實體機器」的修正

### 常見的誤解

影片講解時常把 EC2 講成**實體機器**——**這是方便的說法但不完全正確**。

### 正解

**EC2 是虛擬機器（VM, Virtual Machine）**——它跑在 AWS 的**實體伺服器**上，但 AWS 會把一台實體機切成多台 VM 給不同客戶使用。

| 概念 | 實際狀況 |
|---|---|
| 你看到的 EC2 instance | VM（虛擬機）|
| VM 跑在哪裡 | AWS 機房的實體伺服器上 |
| 為什麼會以為是實體機器 | 因為用起來跟 SSH 連進一台真伺服器沒兩樣 |

### 所以要怎麼看 EC2？

- **概念上**：當成一台實體伺服器無妨，操作、配置、SSH 都一樣
- **本質上**：VM，AWS 會根據你選的規格（CPU/RAM）從實體機分一塊給你
- **選規格就是**：「我要多強的機器？」（instance type 如 t3.micro、m5.large）

### EC2 和 VPC 的分工

| 元件 | 決定什麼 |
|---|---|
| **EC2** | **機器規格**——CPU、RAM、磁碟 |
| **VPC** | **網路環境**——這台機器放哪個網段、誰能連到它 |

兩者正交（orthogonal）：同一台 EC2 可以放進不同 VPC；同一個 VPC 可以放各種規格的 EC2。

# 區域 2：我的疑問與解答

## Q1：為什麼影片說要確認 Subnet 指到不同 AZ 才能達到 HA？HA 是什麼？

### HA = High Availability（高可用性）

**目標**：服務不會因為「一個地方壞掉」而整個掛掉。

### 為什麼「Subnet 放在不同 AZ」就能達到 HA？

關鍵是前面講的那個事實——**不同 AZ 之間是實體隔離的**（不同大樓、不同電源、不同網路）。

#### 情境對比

**❌ 所有 Subnet 都在同一個 AZ**：

```
ap-northeast-1a 整個停電 / 光纖被挖斷 / 機房起火
   └── 你的所有 Subnet（裡面的 EC2）全部失聯
        → 整個服務掛掉
```

**✅ Subnet 分散在多個 AZ**：

```
ap-northeast-1a（掛了）          ap-northeast-1c（還活著）
  └── Subnet-A1（EC2 失聯）       └── Subnet-C1（EC2 正常）
                                      ↑
                         Load Balancer 把流量全部導到這邊
                                      ↓
                         使用者感覺不到問題，服務照跑
```

### 達到 HA 需要的三個條件

1. **EC2 跨 AZ 部署**（多開幾台，分散在不同 AZ 的 Subnet）
2. **前面有 Load Balancer**（Application Load Balancer 之類，自動把流量導到活著的節點）
3. **資料層也要跨 AZ**（例如 RDS 開 Multi-AZ、S3 本身就跨 AZ 複製）

**一句話結論**：**Subnet 跨 AZ 只是必要條件、不是充分條件**——還要 EC2 實際多開幾台 + Load Balancer + 資料層跨 AZ，HA 才算真的做起來。

### 為什麼這件事重要？

真實案例：AWS 的 AZ 偶爾會出事（2021 年 us-east-1 整個 Region 有多次大當機、東京也出過 AZ 級別的故障）。**大公司不會把所有雞蛋放在同一個 AZ 籃子裡**——這就是為什麼「跨 AZ 部署」是入門必學。

## Q2：Claude 網頁版做出的 HTML / 圖片 / 動畫，要進到本地的 Obsidian / Git，是不是都要手動轉一道？

### 正解

**是的，要手動。** 網頁版的本質是「聊天介面」，它**不直接寫你的本地檔案**。

### 兩種使用模式對照

| 需求類型 | 適用工具 | 為什麼 |
|---|---|---|
| **一次性的精美輸出**（簡報、海報、示意圖、動畫 demo）| **Claude 網頁版** | 能一次產生完整 HTML / SVG / 圖片、視覺最強 |
| **持續累積的筆記/程式碼**（每天寫、進 Git）| **Claude Code** | **直接操作本地檔案、直接 commit、無手動彙整步驟** |

### 如果今天需求是「把學到的東西記進 Abby-notes」

- **用 Claude Code**（就是你現在在做的）——筆記直接進檔案、直接進 Git、沒有「複製貼上到本地」的中間步驟
- **用網頁版** → 自己轉：複製 Markdown → 貼到 Obsidian → 存檔 → git add

### 判斷原則

> **「這是有規律時程的需求嗎？」**
> - **是** → 接進工具鏈（Claude Code 寫檔、MCP server 自動化、或至少用腳本）
> - **否**（偶爾做一張海報、一次性視覺設計）→ 網頁版手動轉即可，省去搞工具鏈的時間

**你的觀察完全正確**：網頁版輸出要進本地系統就是一道手續，無法省。這是兩種工具的**定位差異**，不是 bug。

## 今日 AWS 學習總結

### 學到的 mental model

```
Region（地理層）
  └── AZ（實體隔離層，HA 的基礎）
        └── Subnet（邏輯網路層，控制誰連得到）
              └── EC2（VM 層，跑你的程式）

VPC 圈起整組網路範圍，決定流量規則
Load Balancer 把流量分散到不同 AZ 的 EC2 實現 HA
```

### 一句話記憶

- **Region** = 在地球哪個角落
- **AZ** = 在那個角落裡哪一棟獨立的大樓
- **VPC** = 你租的那整片辦公園區
- **Subnet** = 園區裡的每一棟小樓
- **EC2** = 小樓裡的某一間辦公室（虛擬的，但用起來像實體）
