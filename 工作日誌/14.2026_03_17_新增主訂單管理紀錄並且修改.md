| 外鍵 | 來源表.欄位 | 目標表.欄位 | 影響 |
|------|------------|------------|------|
| `order.event_id` | order.event_id | event.id | 必須是有效的 event ID |

| `booth_order_subscription.event_id` | bos.event_id | event.id | 必須是有效的 event ID |
| `booth_order_subscription.order_id` | bos.order_id | order.id | 必須是有效的 order ID |

> **注意**：GORM model 用 `constraint:-` 避免 AutoMigrate 建立外鍵，但資料庫可能已有既有約束。需用以下查詢確認：

```sql
-- 檢查 order 表的外鍵約束
SELECT CONSTRAINT_NAME, TABLE_NAME, COLUMN_NAME, REFERENCED_TABLE_NAME, REFERENCED_COLUMN_NAME
FROM information_schema.KEY_COLUMN_USAGE
WHERE TABLE_SCHEMA = 'future_sign_prod'
  AND TABLE_NAME IN ('order', 'booth_order_subscription', 'order_item', 'order_electricity')
  AND REFERENCED_TABLE_NAME IS NOT NULL;
```
