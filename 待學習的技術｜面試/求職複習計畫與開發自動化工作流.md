# 求職複習計畫與開發自動化工作流

## 重點整理

### 一週測試衝刺計畫（5/26–5/30，每天 9:30–12:30）

- **Day 1** 演算法熱身（Bubble Sort 複雜度）+ 測試觀念：單元測試 vs 組件測試；工具定位 <mark style="background: #FFF3A3A6;">Jest（全能框架，React 標配）、Mocha（彈性高，常搭 Chai/Sinon，Node/Express 後端）、Storybook（UI 元件沙盒，非邏輯測試）</mark>。
- **Day 2** Jest 基礎：安裝、為 bubbleSort 寫 `.test.js`、Matchers（`toBe`/`toEqual`/`toContain`）；如何測排序函式（空陣列、已排序、含負數等邊界）。
- **Day 3** Jest + React Testing Library：測 render 文字、`fireEvent.click` + `jest.fn()` 模擬函式；<mark style="background: #ABF7F7A6;">RTL 理念：從使用者視角測，查 DOM 看得到的東西，不碰實作細節</mark>。
- **Day 4** Mocha & Chai：與 Jest 語法相近（describe/it），Chai 語意化斷言 `expect(foo).to.be.a('string')`，為後端 utility 寫測試。
- **Day 5** Storybook：`npx storybook@latest init`，寫 `.stories.jsx`、操作 args（primary/disabled）；證明具備元件驅動開發（CDD）與 Design System 概念。

### React 學習進度（6/2–6/7）

JS 陣列與物件存取 → React 資料流（元件階層、Props、Context 跨組件）→ 表單與 Refs → <mark style="background: #FFF3A3A6;">全域狀態管理（Redux Toolkit vs Zustand）</mark> → 前端架構對決（Next.js App Router vs Vite 構建工具）。

### 自動化流程一：截圖自動命名改用本地免費 Vision 模型

<mark style="background: #BBFABBA6;">不想付 Anthropic API 錢 → 改用本地 Ollama 開源視覺模型，完全免費、無限次、斷網可跑。</mark>

```python
import ollama
res = ollama.chat(
    model='llama3.2-vision',  # 或更輕量快速的 moondream
    messages=[{
        'role': 'user',
        'content': '這是軟體開發截圖，請給一個適合當檔名的英文簡短關鍵字（如 react-error-setup），不要額外描述。',
        'images': ['/path/to/screenshot.png']
    }]
)
new_name = res['message']['content'].strip()
```

安裝：裝 Ollama → `ollama run llama3.2-vision`（或 moondream）→ 把腳本的 anthropic 套件換成 ollama。

### Cursor（VS Code）終端機剪貼簿/Ctrl+V 修復（Windows）

衝突原因：終端機 `Ctrl+C` 預設是發送中斷訊號（SIGINT），與複製衝突。修法：

1. **改 keybindings.json**（Ctrl+Shift+P → Open Keyboard Shortcuts (JSON)）：

```json
[
  { "key": "ctrl+c", "command": "workbench.action.terminal.copySelection",
    "when": "terminalFocus && terminalHasSelection" },
  { "key": "ctrl+v", "command": "workbench.action.terminal.paste",
    "when": "terminalFocus" }
]
```

2. **改右鍵行為**（設定搜尋 `terminal right click` → Right Click Behavior 設為 `copyPaste`：有選取就複製、沒選取就貼上）。
3. **檢查 Shell**：WSL 剪貼簿可能斷開（`powershell.exe -Command "Get-Clipboard"` 測試）；PowerShell 可 `Remove-Module PSReadLine` 重設。
4. **Windows 剪貼簿歷程**：`Win+V` 測試，必要時到 設定→系統→剪貼簿 關閉再開啟。

**keybindings 語法解釋**：

- <mark style="background: #ABF7F7A6;">`command`＝按下快捷鍵時執行的功能名稱；`when`＝只有在符合此條件時才生效。</mark>
- `workbench.*`＝由編輯器「工作台（Workbench，整個 UI：側邊欄/狀態列/分頁/終端面板）」全域管理的核心功能（非某擴充套件提供）。
- <mark style="background: #FF5582A6;">複製的 when 多一個 `&& terminalHasSelection`</mark>：貼上只需遊標聚焦終端（terminalFocus）；複製需「聚焦」+「有選取文字」兩條件，否則沒選取時按 Ctrl+C 仍是中斷程式（保留 SIGINT 功能）。
