# AIoT 概念、應用與實習工程師角色

## 重點整理

<mark style="background: #ADCCFFA6;">AIoT＝AI（人工智慧）＋ IoT（物聯網）</mark>。

- <mark style="background: #FFF3A3A6;">IoT</mark> 負責收集數據（像人的「感官／神經系統」）。
- <mark style="background: #FFF3A3A6;">AI</mark> 負責分析與決策（像人的「大腦」）。
- 一句話：讓原本只會聯網傳資料的裝置，開始具備<mark style="background: #BBFABBA6;">「思考」與「自主學習」</mark>的能力。

### 傳統 IoT vs AIoT
傳統 IoT 只把資料傳上雲再由人/規則處理；AIoT 在裝置端或後端加入模型，能即時分析、預測、自主決策。

### 實習工程師在 AIoT 專案的角色
- <mark style="background: #FFB8EBA6;">資料串接（API & 腳本）</mark>：把硬體端（Edge）回傳的大量資料，穩定地透過 API 傳給後端或 AI 模型。
- <mark style="background: #FFB8EBA6;">效能優化</mark>：硬體運算資源有限，要寫精簡、高效的自動化腳本。
- <mark style="background: #FFB8EBA6;">善用 AI 工具</mark>：用 Copilot、Claude Code 等加速串接邏輯與文件撰寫。
