## 八、硬體需求釐清：完全不需要 GPU

> 常見誤解：學 LangChain / LangGraph / RAG 要本地顯卡。
> **實際上整套技術棧用 CPU 就夠**，現在這台開發機就能跑。

### 為什麼不需要 GPU

LangChain / LangGraph 是**純 Python 排流程的框架**，不跑模型：

```
使用者輸入
   ↓
[LangGraph 決定流程：A → B → C]   ← 純 Python 邏輯，CPU 就夠
   ↓
呼叫 LLM API（OpenAI / Claude / Gemini）  ← 模型在對方機房跑
   ↓
[LangGraph 接著走下一步]
```

**框架只負責「指揮」，不負責「運算」**。GPU 是給模型本身用的。

### 各元件硬體需求

| 元件 | 需要 GPU 嗎 | 說明 |
|------|------------|------|
| LangChain | ❌ | 純 Python orchestration |
| LangGraph | ❌ | 純 Python orchestration |
| LLM 推理（OpenAI / Claude / Gemini） | ❌ | 呼叫 API，模型在對方機房 |
| Embedding（text-embedding-3-small） | ❌ | API call |
| Vector DB（Qdrant / Chroma） | ❌ | HNSW 演算法用 CPU |
| Reranker（Cohere rerank） | ❌ | API call |
| FastAPI | ❌ | 一般 web 框架 |

### 三種模型部署方式比較

| 方式 | 模型在哪 | 硬體需求 | 適合學習用 |
|------|---------|---------|----------|
| **API 模式** | OpenAI / Anthropic 機房 | CPU 即可 | ✅ 推薦 |
| 本地開源模型（Ollama） | 自己電腦 | 需 GPU 或 Apple Silicon | ❌ 不必 |
| 自架 vLLM | 自己 server | A100 / H100 | ❌ 大量用才划算 |

### 什麼情況才需要 GPU（這份 JD 都不需要）

1. **隱私需求**：公司資料不能上雲 → 跑本地 Llama
2. **大量 fine-tune**：訓練自有模型
3. **省 API 成本**：用量大到要自架推理

JD 沒有提到 fine-tune，公司大概率也是純 API 模式。

### 學習成本估算

整套 8 週學習計畫總花費：

| 項目 | 估計 |
|------|------|
| Anthropic API | $5-15 USD |
| OpenAI API（embedding + 比較） | $5-10 USD |
| Cohere API（reranker，有免費額度） | $0 |
| Vector DB | $0（本地 Docker） |
| **總計** | **約 USD 20**（台幣 600） |

比買一張顯卡便宜 200 倍。

### 一句話總結

> **LangGraph 不跑模型，它指揮模型。模型住在 API 提供者那邊，你的電腦只負責發 HTTP request。**
