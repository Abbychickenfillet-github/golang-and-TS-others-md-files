# 學習順序與免費部署作品集策略

> 背景：Abby 的付費主機每月 360–600 元，但基本功（HTML/CSS/JS/TS）還沒練熟就要面試，覺得「主機在燒錢、人還沒準備好」很尷尬。結論是 <mark style="background: #BBFABBA6;">面試前完全可以 0 成本完成所有練習與作品託管，先把主機停掉。</mark>

## 重點整理

### 推薦學習先後順序（不要同時開太多戰線）

1. **第一階段 — 根基重紮：HTML → CSS → JavaScript**
   - 前端骨架。<mark style="background: #FFF3A3A6;">JS 的非同步（Async/Await）、閉包（Closure）、DOM 操作不熟，直接寫 TS 或 React 只會更挫折。</mark>
   - 重點：HTML5 語意化標籤；CSS Flexbox / Grid、響應式（RWD）；原生 JS（ES6+）陣列方法、API 串接。
2. **第二階段 — 強型別：TypeScript**
   - TS 是 JS 的「超集（Superset）」，開發時幫你抓錯、定義資料結構。
   - 重點：Interface、Type、Generics（泛型）；把第一階段的 JS 小專案改寫成 TS 版。
3. **第三階段 — 框架整合與作品集：React / Next.js**
   - 有 TS/JS 底子後，看組件與狀態管理會通透很多；此時做的作品才是面試武器。

### 0 元主機替代方案（面試前完全夠用）

| 類型 | 平台 | 成本 | 說明 |
| :--- | :--- | :--- | :--- |
| 靜態前端（HTML/CSS/JS/React） | **GitHub Pages** | $0 | 連動 GitHub 儲存庫，幾下就變線上網站 |
| Next.js / React / TS | **Vercel / Netlify** | $0 | 現代前端部署神品，自動部署、效能快、免費額度大 |
| Node.js / Go 後端 API | **Render / Railway** | $0 / 輕量 | 部署後端 API，免費或極低成本 |
| PostgreSQL 資料庫 | **Neon / Supabase** | $0 | 免費雲端 PostgreSQL 額度，個人作品足夠 |

<mark style="background: #FFB8EBA6;">注意：免費後端方案久未使用會「冷啟動」，第一次開啟需熱身約 50 秒。可在 104 履歷備註「採免費雲端平台，初次連線較慢請稍候 1 分鐘」，面試官能理解，是業界常態。</mark>

### 104 作品集應對戰術

HR / 面試官只在乎「<mark style="background: #FFF3A3A6;">網址能點開、畫面沒壞、GitHub 程式碼漂亮</mark>」，不管主機是自租還是免費平台。

更新節奏：現有作品先搬 Vercel/Render 卡位 104 → 停用付費主機 → 重練基本功與 TS → 用 TS 改寫舊作品重新推上（104 網址自動更新）。

加分寫法：改寫後可在 104 標註「V2 版本：使用 TypeScript 進行全站型別重構，提升程式碼健壯度」——展現重構能力。

<mark style="background: #BBFABBA6;">戰略總結：立刻退租/暫停付費主機卸下扣款壓力 → 一切在本機（VS Code + Live Server）練習 → 每天練習 Push 到 GitHub 當面試證明 → 投履歷前 1–2 週再用 Vercel/Render 部署。</mark>
