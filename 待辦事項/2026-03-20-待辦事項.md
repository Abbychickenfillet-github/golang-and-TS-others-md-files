## BUG: 票券管理頁退款未呼叫綠界 ECPay API

- **嚴重度**: CRITICAL
- **發現時間**: 2026-03-20 03:25
- **狀態**: 待修復
- **錯誤紀錄**: [refund-bypass-ecpay.md](../dashboard/docs/errors/refund-bypass-ecpay.md)

### 問題描述
從票券管理頁（TicketRefundModal）發起退款時，呼叫 `POST /api/v1/orders/{order_id}/refund`，
後端 `OrderService.RefundOrder()` 直接把 refund_record 寫成 `status=success`，
**完全沒有呼叫 `PaymentService.ProcessRefund()` 打綠界退款 API**。

### 證據
- refund_record `RF-20260320-XG6J95` 的所有 ecpay 欄位皆為 NULL
- `ecpay_trade_no`, `ecpay_refund_id`, `ecpay_return_code`, `ecpay_return_message` 全部 NULL
- `ecpay_request_log`, `ecpay_response_log` 全部 NULL
- 但 `status = success`、`processed_at` 有值

### 影響範圍
- 錢沒有實際退回消費者
- DB 卻標記退款成功，造成帳務不一致

### 修復方向
- `OrderService.RefundOrder()` 需要在標記 success 前呼叫 `PaymentService.ProcessRefund()`
- 參考正確實作：`RefundService.ApproveRefund()` (refund_service.go:249)
