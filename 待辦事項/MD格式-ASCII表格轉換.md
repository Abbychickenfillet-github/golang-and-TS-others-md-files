# MD 格式 - ASCII 表格轉換

## 問題

從 Claude Code CLI 複製貼上的 ASCII 表格在 Markdown 文件中會對不齊：

```
┌─────────────┬────────────┬────────────────────┐
│    環境     │    意義    │      部署位置      │
├─────────────┼────────────┼────────────────────┤
│ local       │ 本地開發   │ 你的電腦           │  ← 右邊框線對不齊
└─────────────┴────────────┴────────────────────┘
```

## 原因

ASCII 表格（使用 `┌ ─ ┬ ┐ │ ├ ┼ ┤ └ ┴ ┘` 等字符）依賴**等寬字體**才能正確對齊。

不同的：
- 編輯器（VS Code、Notion、GitHub）
- 字體（有些字體中文和英文寬度不同）
- 渲染器

都會導致對齊錯亂。

## 解決方案

### 使用 Markdown 表格格式

```markdown
| 環境 | 意義 | 部署位置 |
|------|------|----------|
| local | 本地開發 | 你的電腦 |
| staging | 預發布環境 | 伺服器 |
| production | 生產環境 | 伺服器 |
```

**優點**：
- 在任何編輯器/字體中都能正確渲染
- GitHub、VS Code、Notion 都支援
- 不依賴等寬字體

### 轉換方式

1. **手動轉換**：把 `│` 改成 `|`，刪除框線
2. **使用工具**：
   - VS Code 擴充：`Markdown Table Formatter`
   - 線上工具：[Tables Generator](https://www.tablesgenerator.com/markdown_tables)

## 何時用 ASCII 表格？

如果你想保留 ASCII 表格的視覺效果，把它放在 code block 中：

````markdown
```
┌────────┬────────┐
│  環境  │  用途  │
├────────┼────────┤
│ local  │ 開發   │
└────────┴────────┘
```
````

這樣它會以等寬字體渲染，對齊問題會減少（但仍可能因中英文混用而有問題）。

## 總結

| 格式 | 適用場景 | 對齊穩定性 |
|------|----------|------------|
| Markdown 表格 `\| \|` | 文件、README | 穩定 |
| ASCII 表格 `┌─┐` | 終端機輸出、純文字 | 需等寬字體 |
| ASCII 表格 + code block | 想保留視覺效果 | 較穩定 |
