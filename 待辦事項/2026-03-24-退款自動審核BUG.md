# 🔴 BUG: 消費者退款申請不該立即生效

## 問題
消費者在 my-tickets 按「退款」後，票券直接進入「已退款」tab，沒有經過主辦方審核。

## 預期行為
1. 消費者申請退款 → refund_record status=`pending`
2. 主辦方在 /refund-management 看到待審核
3. 主辦方 approve → 呼叫 ECPay 退款 → status=`success`
4. 消費者才看到「已退款」

## 需要檢查
- [ ] `MemberRequestRefund` 是否只建立 pending 記錄？
- [ ] `event.default_approve_refund` 是否觸發了自動審核？
- [ ] 前端 my-tickets 是否把 `pending` 的退款顯示成「已退款」？
- [ ] 前端的退款 subtab filter 邏輯是否正確？
