# 待辦：確認付款後攤位 booking_status 是否正確更新

## 背景

2026-03-31 測試加價購時發現台灣品牌-07 已付款成功，但 `booth.booking_status` 仍為 `available`。
經查是手動修改 DB 時帶入 `\r\n` 換行符造成的髒資料（已清洗）。

但仍需確認：**正常付款流程中，攤位的 `booking_status` 是否有從 `pending` → `booked` 正確轉換。**

## 待確認項目

- [ ] 一般購買攤位（BTH 訂單）：付款成功後 `booking_status` 是否變成 `booked`
- [ ] 加價購（ADDON 訂單）：加價購不應該改變 `booking_status`（已經是 `booked`）
- [ ] ECPay callback 成功時，`VerifyAndProcessCallback` 有沒有觸發攤位狀態更新
- [ ] 訂單取消/退款時，`booking_status` 是否正確回退為 `available`

## 檢查方式

```sql
-- 查看特定攤位的預定狀態
SELECT id, name, booking_status, booking_at, updated_at
FROM booth
WHERE id = '{booth_id}';

-- 查看訂單對應的攤位訂閱
SELECT bos.booth_id, bos.payment_status, b.booking_status
FROM booth_order_subscription bos
JOIN booth b ON b.id = bos.booth_id
WHERE bos.order_id = '{order_id}';
```
