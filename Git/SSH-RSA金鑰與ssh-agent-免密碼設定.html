重點整理

定義：

  • SSH (Secure Shell)：加密的網路傳輸協定，是電腦與遠端伺服器（GitHub/GitLab）之間的「私密通道」。Git SSH 網址長得像 git@github.com:user/repo.git。
  • RSA：非對稱加密演算法，產生一對金鑰：私鑰（Private Key）留在自己電腦、絕不外流；公鑰（Public Key）上傳 GitHub、公開沒關係。
  • 驗證原理：連線時公私鑰自動數學匹配，匹配成功就放行，預設目的就是「免打密碼」。

想要「每次 push 都要密碼」（安全考量）

  • 方法 A（推薦）：給 SSH key 設密碼短語 Passphrase。建立時 ssh-keygen -t rsa -b 4096 會問 Enter passphrase，輸入即可；已存在的免密金鑰可用 ssh-keygen -p -f ~/.ssh/id_rsa 補加密碼。
  • 方法 B：改用 HTTPS 網址 git remote set-url origin https://github.com/user/repo.git，並清掉系統憑證管理器紀錄，每次 push 就會要求輸入 PAT（Personal Access Token）。

為何要有 ssh-agent？

核心：平衡「高安全性」與「便利性」。設了 Passphrase 後若一天 push 50 次就要打 50 次密碼，很痛苦。ssh-agent 是背景執行的「私鑰管理員」：

  • 用 ssh-add 把私鑰放進去，只需輸入一次密碼，之後解密後的私鑰暫存在記憶體，Git 連線時直接找它拿憑證。
  • 安全：硬碟上的私鑰檔依然是加密狀態，就算檔案被偷、沒密碼也解不開。
  • 進階 Agent Forwarding：多跳連線（本機→伺服器 A→GitHub）時，私鑰從頭到尾不離開本機，由本機 agent 回應驗證。
情境 安全性 便利性
私鑰不設密碼 極低（檔案被偷就完蛋） 最高
有密碼、不用 agent 最高 極低（每次都要打）
有密碼 + ssh-agent 最高 高（開機打一次）

重開機也免密碼（託管給作業系統）

  • Mac → Keychain：編輯 ~/.ssh/config 加入 Host * ＋ AddKeysToAgent yes、UseKeychain yes、IdentityFile ~/.ssh/id_rsa，再執行 ssh-add --apple-use-keychain ~/.ssh/id_rsa。（ED25519 金鑰把 id_rsa 換成 id_ed25519）
  • Windows → 內建 OpenSSH Agent（系統管理員 PowerShell）：
Set-Service -Name ssh-agent -StartupType Automatic
Start-Service ssh-agent
ssh-add $env:USERPROFILE\.ssh\id_rsa

設為 Automatic 後，每次重開機系統自動載入金鑰，push 免再打密碼，私鑰在硬碟仍加密。
