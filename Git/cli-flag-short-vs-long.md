# 命令列選項為何有短寫又有長寫（`-v` vs `--verbose`）

本篇重點 a–b，共 2 個。

## (a) 兩種寫法服務不同場景，不是互相取代

| | 短選項 `-v` | 長選項 `--verbose` |
|---|---|---|
| 誰在用 | 互動式手動打指令 | 寫進 script / CI / 文件給別人（或未來的自己）看 |
| 優點 | 打字快 | 一眼看懂用途，不用查手冊 |
| 缺點 | 事後回頭看 script 常忘記代表什麼 | 打字較久，但 shell tab 補全可以緩解 |

## (b) 歷史脈絡

原始 Unix 的 `getopt` 只有單字母短選項；GNU 後來加了 `getopt_long`，補上長選項，**向下相容**原本的短選項，不是取代掉它。所以現在大多數 GNU 風格工具（含 git）都同時支援兩種寫法，讓使用者依場景自己選。
