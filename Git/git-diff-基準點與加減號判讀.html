重點整理

執行 git diff A B 時：前面的 A 是基準點（Source / Old / --- a/），後面的 B 是目標（Target / New / +++ b/）。

符號意義（站在「從 A 變成 B」的角度）：

  • -（紅）：該行在 A 有、B 沒有（相對基準被刪除）。
  • +（綠）：該行在 A 沒有、B 出現了（相對基準被新增）。

口訣：「從前（面）往後（面）看」。想知道「比起 branch1，branch2 改了什麼」就把 branch1 放前面：git diff branch1 branch2。寫反了符號就整個相反。

官方文件佐證（git-scm.com/docs/git-diff）：EXAMPLES 區塊有 git diff topic master「Changes between the tips of the topic and the master branches」，配合 patch 格式 --- a/file（第一參數＝基準）、+++ b/file（第二參數＝目標）。

兩點 .. vs 三點 ...（陷阱）

語法 比較對象 用途
git diff A B 或 git diff A..B A 與 B 各自的最新狀態 直接比兩個分支現況
git diff A...B（三點） A 與 B 的共同祖先 與 B 的最新位置 Code Review：看「我從主線分支出來後，這個 branch 到底改了什麼」

範例（是非／判讀題）

git diff main feature 畫面出現 + console.log('hello') → 代表 feature 比 main 多了這行。
出現 - const oldVar = 1 → 代表 feature 比 main 少了這行。
