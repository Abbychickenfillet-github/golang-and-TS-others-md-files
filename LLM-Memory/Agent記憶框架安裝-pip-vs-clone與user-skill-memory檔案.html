Agent 記憶框架安裝與核心配置檔解析

重點整理

1. Agent 記憶框架（Letta / MemGPT、Mem0）怎麼裝？

  • 90% 場景直接 pip install：只是要在自己專案裡調用記憶功能、要穩定發佈版、方便在 requirements.txt 記版本。
  • 只有這些情況才 git clone：想貢獻代碼修 bug／發 PR、需要改框架核心邏輯（底層 memory 管理）、想跑 tests／examples（pip 包通常不含）。
# Mem0
pip install mem0ai
# Letta（原 MemGPT）
pip install letta

[!warning] 開發提醒 這些框架通常要搭配資料庫（PostgreSQL）或向量資料庫；本地開發前先確認 環境變數（API Keys）已正確配置。

2. pip install vs git clone 核心差異 ⭐

比喻：pip install 像從 App Store 下載 App（直接用）；git clone 是把原始碼整包搬回家（看怎麼寫、甚至改寫）。

特性 pip install [package] git clone [url]
對象 已發布的成品（Wheel/Source Dist） 原始代碼倉庫（Repository）
存放位置 Python 的 site-packages 你指定的工作目錄
修改權限 不易修改，只當 Library 調用 隨意改、可自行 commit
版本控制 用 pip 升降版本 用 git 切分支／回溯
包含內容 僅執行所需檔案 含開發文件、測試、Git 歷程
  • 進階技巧：clone 後若想在專案中使用又能即時生效，用 pip install -e .（可編輯模式），改了 clone 的 code 專案會同步生效。
  • 對 Letta／Mem0 的建議：先 pip install 跑通功能，要深入研究向量資料庫底層邏輯時再 clone。

3. user.md / skill.md / memory.md 差在哪 ⭐

一句話：user.md 是關於「你」，skill.md 是關於「能力」，memory.md 是關於「過去」。 把 AI 想成新助理：user.md＝對老闆的觀察筆記、skill.md＝隨身工作手冊、memory.md＝發生過什麼事的日誌。

檔案 核心問題 變動頻率 範例
user.md（用戶畫像） 你是誰？喜歡什麼？ 低（換工作/改習慣才動） 「全端工程師」「元件用 PascalCase」
skill.md（能力清單/SOP） 怎麼做這件事？ 中（有新技術/流程優化） 「部署 Next.js 步驟」「串 LINE Pay 流程」
memory.md（長期記憶/日誌） 發生了什麼事？ 高（隨對話自動更新） 「專案 DB 已從 MySQL 換成 PostgreSQL」
  • 作用對照：user.md 讓 AI 講話合你胃口、不必每次自我介紹；skill.md 讓 AI 面對複雜任務翻手冊照 SOP 走、不胡搞；memory.md 解決 LLM「健忘」、避免重複問已講過的事。
  • 結合 Obsidian 習慣：user.md＝個人 Readme；skill.md＝你正在建的「程式問答筆記」；memory.md＝你的「開發日誌 Daily Logs」。
