## 1. 三大類別本質差異

| 維度 | 監督式 Supervised | 非監督式 Unsupervised | 強化式 Reinforcement |
|------|-------------------|----------------------|---------------------|
| **訓練資料** | 題目 + **正確答案**（label）| **只有題目，沒答案** | 沒答案，但有**獎勵訊號** |
| **學什麼** | 題目 → 答案的對應 | 資料的內在結構 | 最佳行動策略（policy） |
| **回饋方式** | 直接告訴對錯 | 沒有對錯 | 延遲獎勵（做完一連串動作才知好壞） |
| **目標** | 預測新資料的答案 | 發現資料規律 | 累積最大獎勵 |
| **典型問題** | 這張照片是貓還狗？ | 這 1000 個客戶能分幾群？ | 這步棋怎麼下贏？ |

## 2. 監督式學習 (Supervised Learning)

### 怎麼運作

```
訓練階段：
  給：(x_1, y_1), (x_2, y_2), ..., (x_n, y_n)
       ↑題目      ↑答案
  學：函數 f，使 f(x_i) ≈ y_i

預測階段：
  新題目 x_new → f(x_new) → 預測答案 y_pred
```

### 兩大子類型

| 子類型 | 答案 y 是什麼 | 例子 |
|--------|-------------|------|
| **分類 (Classification)** | 離散類別 | 貓/狗、垃圾信/正常信、A型/B型/C型 |
| **回歸 (Regression)** | 連續數值 | 房價、股價、明天溫度 |

### 代表演算法

| 演算法 | 屬於 | 一句話 |
|--------|------|--------|
| **Linear Regression** | 回歸 | 用一條直線擬合資料 |
| **Logistic Regression** | 分類（名字騙人） | 用 sigmoid 輸出機率 |
| **Decision Tree** | 兩者皆可 | if-else 樹狀規則 |
| **Random Forest** | 兩者皆可 | 一堆 Decision Tree 投票 |
| **SVM** | 主要分類 | 找出最佳分界線（margin 最大） |
| **KNN** ★ | 兩者皆可 | 看最近 K 個鄰居怎麼標 |
| **Neural Networks / Deep Learning** | 兩者皆可 | 用神經網路學任何對應關係 |

### 用例

- **垃圾郵件過濾**：(郵件內容, 是不是垃圾) → 學分類器
- **房價預測**：(房屋特徵, 實際成交價) → 學回歸模型
- **圖像辨識**：(照片, 標籤) → 學 CNN
- **客戶流失預測**：(客戶行為, 是否流失) → 學分類器

### 優缺點

| 優 | 缺 |
|----|----|
| ✅ 結果**可控**——你定義想要什麼答案 | ❌ 標註**貴**——人工標 10 萬筆要幾十萬 |
| ✅ 能算**準確率**評估好壞 | ❌ 受限於**標註者偏見** |
| ✅ 業界主力，工具 / 文件最多 | ❌ 新類別出現要重新標註 + 重訓 |

## 3. 非監督式學習 (Unsupervised Learning)

### 怎麼運作

```
訓練階段：
  給：x_1, x_2, ..., x_n        ← 只有題目，沒答案
  學：資料的某種結構（群、降維後表示、密度分佈等）

應用階段：
  新題目 x_new → 屬於哪群？或：壓縮到低維表示
```

### 主要子任務

| 子任務 | 在做什麼 | 代表演算法 |
|--------|---------|-----------|
| **分群 (Clustering)** | 把資料自動分成幾群 | **K-means** ★、DBSCAN、Hierarchical、GMM |
| **降維 (Dimensionality Reduction)** | 把高維資料壓到低維 | **PCA**、t-SNE、UMAP、Autoencoder |
| **密度估計 (Density Estimation)** | 估計資料分佈 | KDE、GMM |
| **異常偵測 (Anomaly Detection)** | 找跟大家都不一樣的點 | Isolation Forest、One-class SVM |
| **關聯規則 (Association Rules)** | 找「A 出現時 B 也常出現」 | Apriori、FP-Growth |

### 用例

- **客戶分群**：分析消費資料，自動分出「高消費型 / 折扣敏感型 / 嘗鮮型」
- **異常交易偵測**：找信用卡盜刷
- **主題建模 (Topic Modeling)**：對一堆文件自動分主題
- **推薦系統**：找「跟你品味相近的人」（基於 cluster）
- **資料壓縮 / 視覺化**：PCA 降維讓人看得懂高維資料
- **PQ / IVF 等向量索引**：用 K-means 找代表點

### 優缺點

| 優 | 缺 |
|----|----|
| ✅ **不用標註**——海量資料直接餵 | ❌ 結果**不可控**——分出的群可能跟你想的不一樣 |
| ✅ 探索性分析的唯一選擇 | ❌ **沒有客觀對錯**——很難算準確率 |
| ✅ 可能發現人類沒想到的模式 | ❌ K（要分幾群）要自己決定 |
| ✅ 不會被標註者偏見綁住 | ❌ 對初始值敏感（每次跑可能不同） |

> **詳細的優缺點對照**請見 [9. 監督式 vs 非監督式深入對照](#9-監督式-vs-非監督式深入對照)

## 4. 強化式學習 (Reinforcement Learning, RL)

### 🎮 「沒答案，但有獎勵」到底是什麼意思？

#### ⚠️ 先澄清：跟 prompt 沒關係

```
LLM 的人生兩個階段：
  ┌───────────────────┐      ┌──────────────────────┐
  │ 訓練階段           │  ─►  │ 應用階段（你寫 prompt）│
  │ (花幾個月、幾億美金)│      │ (你跟 ChatGPT 對話)   │
  └───────────────────┘      └──────────────────────┘
   ↑ RL 用在這裡             ↑ 跟 RL 已經無關
```

**Prompt** 是訓練完之後，使用者跟模型對話用的。**強化式學習 (RL)** 是**訓練階段**的方法。兩件事是不同階段。寫 prompt 完全不會用到 RL 概念。

#### 用「考試類型」類比三大學習法

| 類型 | 像什麼考試？ | 老師會做什麼？ |
|------|------------|--------------|
| **監督式** | 有標準答案的選擇題 | 每題改完直接告訴你「這題對 / 錯」 |
| **非監督式** | 沒考試，給你一堆資料分類 | 沒老師，自己整理 |
| **強化式** | 玩遊戲打分數 | 不告訴你哪步走錯，**只在遊戲結束時告訴你「贏 +1 / 輸 -1」** |

#### 用「下棋」最容易懂

```
監督式（如果存在）：
  你下第 1 步 → 老師：✅ 對
  你下第 2 步 → 老師：❌ 錯（應該下這裡）
  你下第 3 步 → 老師：✅ 對
  ...每一步都立刻知道對錯

強化式（實際情況）：
  你下第 1 步 → ？ 不知道
  你下第 2 步 → ？ 不知道
  你下第 3 步 → ？ 不知道
  ...
  你下第 80 步 → 將軍！🏆
       ▼
  整局結束 → 拿到 reward = +1（贏了）
       ▼
  AI 反推：「剛剛 80 步整體大概是對的」
       但「哪一步最關鍵」？要靠玩很多很多局慢慢學
```

→ **「沒答案」= 沒人告訴你「某一步該怎麼走」，只能從「整局贏輸」反推**。

#### 為什麼下棋這種事「沒辦法給答案」？

因為**沒人知道每一步的「正確答案」是什麼**：

- **圍棋盤面 10¹⁷⁰ 種可能**（比宇宙原子還多），不可能事先標好「每個盤面該下哪」
- **自駕車「方向盤該轉幾度」**，每個情境都不同，沒法事先標
- **機器人「該抬哪隻腳」**，物理環境千變萬化

→ **沒有現成答案可以教**，只能讓 AI **自己玩很多很多次**，從「贏 / 輸」「分數高 / 低」中學。

#### 遊戲 AI 三個經典例子

##### 範例 1：AlphaGo 下圍棋

```
訓練過程：
  AlphaGo 自己跟自己下了 4000 萬局棋
  每局只知道「最終誰贏」（reward = +1 或 -1）
  從這 4000 萬局的勝負模式中，學到「什麼盤面該下哪」
       ▼
  最後打敗世界冠軍李世乭 (2016)
```

**重點**：沒有人告訴 AlphaGo「這個盤面應該下這裡」——它**自己摸索**出來的。

##### 範例 2：Atari 打磚塊

```
螢幕畫面 → AI 要按 [左 / 右 / 不動]
       ▼
  AI 不知道該按哪個
  亂按 → 球可能掉下去（reward = -1）→ 學到「這情境不該這樣按」
  亂按 → 球反彈打到磚塊（reward = +1）→ 學到「這情境這樣按好」
       ▼
  玩幾百萬次後，AI 學會打到比人類還高分
```

##### 範例 3：自駕車

```
攝影機看到前方有人 → 該怎麼操作方向盤？
  沒人能事先標答案（每個情境都不同）
       ▼
  在模擬環境中試：撞到人 → reward = -100
                  安全閃過 → reward = +1
       ▼
  學到「看到人要怎麼閃」
```

#### RL 的「沒答案」一句話收斂

> **RL 的「沒答案」= 沒人告訴 agent「每一步該怎麼做」，只能從「最後得幾分」反推哪些動作是對的**。
>
> 適合用在「**沒有標準答案、但有明確輸贏 / 分數**」的場景：下棋、玩遊戲、機器人控制、自駕車。

---

### 怎麼運作

```
Agent ──action──► Environment
   ▲                  │
   │                  ▼
   └──reward + state──┘

Agent 嘗試各種 action
→ 環境給 reward（可能延遲）
→ Agent 學「在什麼 state 該選什麼 action 才能累積最大 reward」
```

→ 沒人告訴 agent「正確答案是什麼」，它**自己探索**並從**獎勵訊號**中學習。

### 代表演算法

| 演算法 | 一句話 |
|--------|--------|
| **Q-learning** | 學「(state, action) → 預期累積獎勵」表 |
| **DQN** (Deep Q-Network) | 用神經網路學 Q-function（DeepMind 用來打 Atari） |
| **Policy Gradient** | 直接學 policy（state → action 的機率） |
| **PPO** (Proximal Policy Optimization) | 業界主流，OpenAI / RLHF 用 |
| **AlphaGo / AlphaZero** | RL + 深度學習 + MCTS，下圍棋打敗人類 |

### 用例

- **遊戲 AI**：AlphaGo、Dota AI、星海爭霸 AI
- **機器人控制**：學走路、抓取、自駕車
- **推薦系統**：用 RL 優化長期使用者參與度
- **RLHF (Reinforcement Learning from Human Feedback)** ★：**ChatGPT / Claude 訓練的關鍵步驟**——人類給回饋當 reward 訊號，RL 微調 LLM

### 為什麼 RL 跟 LLM 有關？

```
GPT-4 / Claude 的訓練流程：
  1. Pretraining (自監督式)        ← 學基本語言能力
  2. SFT (監督式微調)              ← 學「該怎麼回答」
  3. RLHF (強化式)                 ← 用人類偏好當 reward 微調
                ↑
        現代 LLM「夠好」的關鍵
```

→ RL 不是只用在遊戲，**每個現代 LLM 都靠 RL 收尾**。

### 優缺點

| 優 | 缺 |
|----|----|
| ✅ 能學「序列決策」問題 | ❌ 需要**大量試錯**（在虛擬環境 OK，現實世界貴） |
| ✅ 不需要事先有「正確答案」 | ❌ Reward 函數很難設計 |
| ✅ 適合「累積目標」場景 | ❌ 訓練不穩定、難 debug |

## 5. 半監督式學習 (Semi-Supervised) — 中間地帶

當你有**少量標註資料 + 大量未標註資料**時：

```
1000 筆有標註 + 100 萬筆沒標註
        ▼
半監督式：用 1000 筆當「種子」，從 100 萬筆中
         自動傳播標籤（label propagation）
        ▼
得到 ~50 萬筆「半自動標註」資料 → 訓練監督式模型
```

**典型方法**：
- **Label Propagation**：類似群裡的點傳染標籤
- **Pseudo-Labeling**：先用少量標註訓練模型 → 給未標註資料打偽標籤 → 重訓
- **Self-Training**：迭代式自我標註

**何時用**：標註成本很高但又有大量原始資料時（很常見）。

## 6. 自監督式學習 (Self-Supervised) — LLM 預訓練的祕密

**自監督式 = 監督式的特殊形式**——標籤**從資料本身自動產生**，不需人工標註。

### 經典例子：BERT 的 Masked Language Modeling

```
原句：The cat sat on the [MASK].
答案：mat                       ← 從原句自動生成的「正確答案」

模型任務：預測 [MASK] 是什麼
```

→ 用「**遮一段，預測被遮的內容**」當訓練任務，**標籤從原始文字自動生成**，不需人標。

### GPT 的 Next-Token Prediction

```
輸入：The cat sat on the
標籤：mat                       ← 下一個詞
```

→ 用「預測下一個 token」當任務，**全網路所有文字都能拿來訓練**。

### 為什麼這個概念這麼重要？

| | 純監督式 | 自監督式 |
|---|---|---|
| 標註成本 | 極高 | **0**（自動生成） |
| 可用資料量 | 標註過的（有限） | **整個 web** |
| LLM 預訓練可行性 | ❌ 不可能 | ✅ **唯一可行的方式** |

→ **自監督式是現代 LLM 能存在的根本原因**。沒有它，GPT/Claude/BERT 都不可能。

### 在分類圖上的位置

```
監督式 ─── 自監督式 ─── 半監督式 ─── 非監督式 ─── 強化式
有標籤              標籤自動生成                  無標籤              無標籤但有獎勵
   ↑                    ↑                          ↑                       ↑
傳統 ML        BERT / GPT 預訓練           少量標註場景             AlphaGo / RLHF
```

## 7. 三大類別的代表演算法總表

| 類別 | 子任務 | 代表演算法 |
|------|--------|-----------|
| **監督式** | 分類 | Logistic Regression、SVM、Random Forest、KNN、Neural Network |
| **監督式** | 回歸 | Linear Regression、Random Forest、Gradient Boosting (XGBoost) |
| **非監督式** | 分群 | **K-means**、DBSCAN、Hierarchical Clustering、GMM |

| **強化式** | - | Q-learning、DQN、Policy Gradient、PPO |
| **自監督式** | - | BERT (MLM)、GPT (NTP)、SimCLR (對比學習) |

## 8. 實務上怎麼選？

### 決策流程

```
你有資料嗎？
  │
  ├── 沒有 → 強化式（從互動中學）
  │
  └── 有
       │
       └── 有標籤嗎？
            │
            ├── 全部都有       → 監督式
            │
            ├── 部分有         → 半監督式
            │
            ├── 完全沒有
            │   │
            │   └── 想做什麼？
            │        │
            │        ├── 分類某新東西    → ⚠️ 需先標註才能監督式（沒標籤就只能非監督式探索）
            │        ├── 找資料結構      → 非監督式（K-means、PCA）
            │        ├── 訓練 LLM/embedding → 自監督式
            │        └── 找異常          → 非監督式（Isolation Forest）
            │
            └── 從資料能自動生成 → 自監督式（NLP 預訓練、對比學習）
```

### 業界常見組合：先非監督探索 → 再監督式精化

```
1. 拿 100 萬筆未標註資料
   ▼ K-means 分 10 群（非監督）
2. 看每群的特性 → 人工為每群命名
3. 從每群挑 100 筆人工標註 → 1000 筆「精挑」標註資料
   ▼
4. 用這 1000 筆訓練監督式分類器
```

→ **省 99% 標註成本** + **拿到可控結果**。

## 9. 監督式 vs 非監督式深入對照

### 成本對照

| 項目 | 監督式 | 非監督式 |
|------|--------|---------|
| 標註成本 | 很高（10 萬筆動輒 30~300 萬元） | 0 |
| 計算成本 | 中（訓練模型）| 中（K-means 也要迭代） |
| 維護成本 | 高（新類別要重標重訓）| 低（新資料直接 fit） |

### 可控性對照

| 項目 | 監督式 | 非監督式 |
|------|--------|---------|
| 結果可預測 | ✅ 你定義要什麼類別 | ❌ 演算法自己決定怎麼分 |
| 結果一致 | ✅ 同樣資料同樣模型 → 同樣結果 | ⚠️ 對初始值敏感 |
| 結果可解釋 | ⚠️ 看模型類型（Decision Tree 易解釋、NN 難）| ❌ 群的「意義」要人解讀 |

### 評估方式對照

| 項目 | 監督式 | 非監督式 |
|------|--------|---------|
| 有客觀指標？ | ✅ 準確率、F1、AUC、RMSE | ❌ 沒有 |
| 怎麼評估？ | 用 holdout / cross-validation | 用 silhouette score、人工檢查 |
| 過擬合可衡量？ | ✅ 看驗證集表現 | ⚠️ 沒明確定義 |

### 一張表收斂

| | 監督式 | 非監督式 |
|---|--------|---------|
| **要標籤** | ✅ | ❌ |
| **目標明確** | ✅ | ❌ |
| **能算準確率** | ✅ | ❌ |
| **可控** | ✅ | ❌ |
| **省成本** | ❌ | ✅ |
| **能用海量資料** | ❌ | ✅ |
| **發現未知模式** | ❌ | ✅ |

## 10. 容易搞混 ⚠️ — KNN vs K-Means

只是名字都有「K」。**根本不同類型的演算法。**

### 🔤 命名解析：mean / nearest 到底是什麼意思？

兩個演算法的英文名常被誤解，先把字面意思釐清。

#### K-means 的「mean」= 平均值，**不是**「方法」

| 英文 | 中文 | 是什麼 |
|------|------|--------|
| **mean** | **平均（值）** | 數學名詞——一堆數字加起來除以個數 |
| ~~method~~ | 方法 | ⚠️ 跟 mean 是**完全不同的英文字**，常被誤聯想 |

**為什麼叫「mean」？** 因為**每個群的中心點 = 該群所有點的平均**：

```
群 A 包含這些點：(1, 2), (3, 4), (5, 6)
        ▼
群 A 的中心 = ((1+3+5)/3, (2+4+6)/3)
            = (3, 4)                     ← 就是「平均值」mean of cluster A
```

演算法的核心動作就是「**反覆把中心移到平均位置**」，所以叫 K-**means**。

**拆字**：
- `K`     = 你要分幾群（你決定的 hyperparameter）
- `means` = 「K 個平均值」（s 是複數，因為有 K 個）
- 合起來：**「分成 K 個群，每群用一個『平均點』當中心」**

**中文標準翻譯**：
- ✅ K-平均演算法
- ✅ K-均值演算法
- ❌ ~~K-方法演算法~~（沒人這樣翻，是錯誤聯想）

#### KNN 的「nearest neighbors」= 最近的鄰居

| 英文 | 中文 | 是什麼 |
|------|------|--------|
| **K** | K 個 | 你決定要看幾個鄰居（例如 K=5） |
| **Nearest** | 最近的 | 距離（通常 Euclidean）最近 |
| **Neighbors** | 鄰居（複數） | 訓練資料中距離新點最近的點 |

**拆字**：
- `K Nearest Neighbors` = **「K 個最近的鄰居」**
- 演算法核心動作：「**看新點周圍最近的 K 個鄰居都是什麼類別**」→ 投票

#### ⚠️ 兩個 K 意義完全不同！

| 演算法 | K 是什麼 |
|--------|---------|
| **KNN** | K = **看幾個鄰居**（K=5 表示參考最近 5 個鄰居） |
| **K-means** | K = **分幾個群**（K=5 表示分成 5 群，產生 5 個中心點） |

→ 同樣寫 `K=5`：
- 在 KNN 是「看 5 個鄰居來投票」
- 在 K-means 是「把所有點分成 5 群」

### 一張表看出差異

| 維度 | **K-Nearest Neighbors (KNN)** | **K-Means** |
|------|-------------------------------|-------------|
| **學習類型** | **監督式**（分類 / 回歸）| **非監督式**（分群） |
| **要不要標籤** | ✅ 需要 | ❌ 不需要 |
| **K 的意思** | 看「最近 K 個鄰居」 | 分成「K 個群」 |
| **訓練階段** | 沒訓練！直接記住所有資料（lazy learner） | 迭代優化群中心（active learner） |
| **預測 / 應用** | 給新點，問「它是哪一類」 | 給一堆點，問「該分成哪幾群」 |
| **輸出** | 新點的類別 / 數值 | 所有點的群編號 + 群中心 |
| **計算貴在哪** | 預測時要算跟所有點的距離 | 訓練時迭代計算群中心 |
| **一句話** | 「**新東西看鄰居**」 | 「**全部一起分群**」 |

### 用例對照

#### KNN 的用例

```
資料：1000 張照片，每張標好「貓 / 狗」
        ▼ 訓練（其實啥都沒做，只是存起來）
新照片來了 → 算它跟 1000 張的距離 → 取最近 5 張（K=5）
       → 5 張裡 4 張是貓 → 投票結果：貓
```

→ **解決的問題**：「給我這個新東西，告訴我它屬於哪一類」

#### K-means 的用例

```
資料：1000 張照片，沒任何標籤
        ▼ 指定 K=3
K-means 自動把 1000 張分成 3 群
       群 A：300 張    群 B：400 張    群 C：300 張
       （演算法不知道哪群是貓、哪群是狗，只知道「這 300 張像」）
        ▼ 人工檢查
你看了發現：A 是貓、B 是狗、C 是兔
```

→ **解決的問題**：「給我這堆東西，告訴我能分成哪幾群」

### 演算法步驟對照

#### KNN 步驟

```
訓練階段：
  把所有訓練資料 (x_i, y_i) 存起來    ← 就這樣，沒了

預測階段（給新點 x_new）：
  1. 算 x_new 跟所有訓練點的距離
  2. 找最近的 K 個
  3. 看這 K 個的標籤 → 多數決（分類）或平均（回歸）
  4. 輸出預測
```

#### K-means 步驟

```
訓練階段：
  1. 隨機放 K 個中心點
  2. 每個資料點歸隊到最近中心
  3. 把中心移到該群所有點的平均位置
  4. 重複 2~3 直到中心不再移動 → 收斂

應用階段（給新點 x_new）：
  算 x_new 跟 K 個群中心的距離 → 歸到最近的群
```

### 為什麼會被搞混？

1. **名字都有 K** ← 最大原因
2. **都會算「距離」** ← 都用歐式距離
3. **都很「直覺」** ← 入門 ML 兩個必學
4. **都跟「鄰近」概念有關** ← KNN 看鄰居、K-means 找中心也是 nearest

### 怎麼記住差異？

> **KNN 的 K = K 個鄰居**（看周圍幾個）
> **K-means 的 K = K 個 mean**（分成幾群中心）

或：

> **KNN：監督式 → 預測一個新點的類別**
> **K-means：非監督式 → 把全部點分群**

## 11. 一句話收斂

> **監督式 vs 非監督式的核心 trade-off**：
> 用「**標註成本**」換「**結果可控性**」。
>
> 業界主流是「**先非監督探索，再監督式精化**」混合策略。
>
> **自監督式**是現代 LLM 能存在的關鍵——**讓資料自己當答案**。
>
> **KNN 跟 K-means** 是分屬監督式和非監督式的兩個演算法，**只是名字有 K** 才被誤會是同一個東西。
