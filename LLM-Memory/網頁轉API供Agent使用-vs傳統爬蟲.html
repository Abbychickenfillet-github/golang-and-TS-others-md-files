重點整理

這類工具在做什麼

像 hermai-cli 這類專案，核心定位是「把任何網頁轉換成供 AI Agent 使用的 API」。本質是進階的網頁自動化／操作引擎，透過 headless browser（無頭瀏覽器，如 Puppeteer / Playwright）模擬真人操作網頁。

適用對象不限於授權商家：
- 公開網頁：一般人瀏覽器看得到的都能轉成 API（比價、抓新聞、監控公開資訊），免憑證。
- 需登入的私有頁面（後台、信箱、個人帳戶）：也可以，但你得自己提供憑證（Cookie / Token / 帳密），讓無頭瀏覽器模擬登入後行為。

服務 Agent 的方式：①結構化輸出——把文字/按鈕/表格轉成乾淨 JSON，AI 一目了然；②動作執行 Action——Agent 發指令（點下一頁、在搜尋框輸入），引擎在後台代為操作。

一般爬蟲就能撈資料,為什麼還要特別寫?

直覺沒錯——單純「撈資料」用 BeautifulSoup / Scrapy / Playwright 就夠了。要特別寫，核心痛點不只是突破反爬蟲，更本質是「給人用的爬蟲」和「給 AI Agent 用的工具」設計邏輯完全不同：

  1. 為 Agent 設計的通用接口標準（常結合 MCP, Model Context Protocol）：傳統爬蟲是客製化、寫死 DOM 結構（div.price、ul>li），網頁一改版代碼就斷。這類工具把網頁動態行為抽象化成 Schema/標準 API Endpoint，Agent 讀 Schema 就知道「要買票得先點 A、填 B、送 C」，像呼叫後端 API 一樣操控網頁，不必為每站手寫代碼。
  2. 處理狀態與雙向操作（Actions），不只唯讀：一般爬蟲多是唯讀（Read）；Agent 需要像真人連續互動：登入→選日期→選人數→過驗證→送出。這類專案在後台幫 AI 管理瀏覽器 Context（Session、Cookie、Token 簽章、重試機制），讓 Agent 能處理複雜的寫入操作（Write Actions）。
  3. 動態適應與反爬蟲防禦：對付 Cloudflare / Datadome 等反機器人牆，底層整合逆向工程、模擬真人鼠標軌跡、處理 per-request-signed（每次請求動態加密簽章）與 bootstrap state——把「繞過防禦、拆動態加密」最髒最痛的活包起來。

一句話總結

維度 傳統一般爬蟲 Hermai-cli 這類專案
服務對象 工程師（撈資料存庫/寫報表） AI Agent（當工具直接呼叫）
主要功能 唯讀為主 雙向（讀＋寫），含點擊/輸入/訂購
網頁改版 結構一變就要重寫 自動生成 Schema，AI 靠語意彈性適應
處理防禦 自己手動解驗證碼/維護 Cookie 底層內建處理動態簽章與 Session 阻擋

傳統爬蟲＝工程師寫給特定任務的「固定劇本」；這類專案＝寫給 AI Agent 的「通用網頁瀏覽控制器」。
