重點整理

核心症狀：明明裝了卻 ModuleNotFoundError

即使已 pip install beautifulsoup4，執行時仍報 ModuleNotFoundError: No module named 'bs4'。

[!warning] 根本原因
這是典型的 Python 環境不匹配：執行程式碼的解譯器（Interpreter），跟你 pip 安裝套件的環境，不是同一個。最常發生在用了虛擬環境、或在 VS Code / Cursor 裡選錯解譯器時。

解決步驟

  1. 確認虛擬環境是否啟用：終端機提示符前應出現 (venv)／環境名稱；沒有就先啟用：.\venv\Scripts\Activate（Windows PowerShell）。
  2. 確認編輯器的 Python 解譯器：VS Code / Cursor 按 Ctrl+Shift+P → Python: Select Interpreter，選到你「裝了 bs4 的那個環境」的路徑。
  3. 在已啟用且正確的環境中重新安裝：pip install beautifulsoup4。
  4. 確認 import 語法：from bs4 import BeautifulSoup（套件名 beautifulsoup4，但 import 時叫 bs4，別打錯）。

[!note] 關鍵概念
套件安裝名 ≠ import 名：安裝是 beautifulsoup4，程式裡卻是 import bs4。

從 VS Code 換到 Cursor 會造成這問題嗎？

會有相關性——因為 Cursor 是獨立編輯器，它自己會記住一份「選定的解譯器」設定，可能跟你在 VS Code 選的不一樣，於是又指到沒裝套件的環境。換編輯器後要再做一次 Select Interpreter。

Anaconda 是什麼

Anaconda 是專為資料科學設計的 Python 發行版（Distribution）——一個「內建工具箱＋環境管理器」的安裝包：

  • 預裝數百個常用套件（NumPy、Pandas、Matplotlib…），不必逐一手動裝。
  • Conda：套件管理＋環境管理系統。類似 pip 但能管 Python 以外的語言；最重要的是能建立多個互相隔離的獨立環境（一個專案用 3.9、另一個用 3.12，套件版本互不干擾）。
  • Anaconda Navigator：圖形介面，點按鈕管理環境、啟動 Jupyter Notebook / Spyder。

Pylance 是什麼，沒它能跑嗎

Pylance 是微軟做的 Python 語言伺服器（Language Server），是 VS Code Python 擴充功能的一部分，提供自動補全、語法檢查、型別提示等。

[!info] Pylance vs 解譯器
沒有 Pylance 程式照樣能跑，完全沒影響。 Pylance 只是讓你「寫」程式更舒服的方向盤與導航儀；真正把 .py 變成可執行指令的引擎是 Python 解譯器（Interpreter）。車子少了導航儀還是能開——只要電腦裝了 Python、且選定環境裡裝了所需套件，程式就能跑。

補充：資料視覺化套件名詞

多數建立在最底層繪圖庫 Matplotlib（Python 最老牌的 2D 繪圖庫）之上。seaborn 是基於 Matplotlib 的高階統計視覺化套件（熱力圖、分佈圖、小提琴圖、回歸圖），預設配色更好看、程式碼更少，且能與 Pandas DataFrame 完美整合。
